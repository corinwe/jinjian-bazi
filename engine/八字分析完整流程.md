## 各步骤输入/工具/输出详表

| 步骤 | 输入依赖 | 做什么 | 调用工具 | 输出 |
|:----|:---------|:-------|:---------|:-----|
| **Step1 排盘** | 年月日时性别姓名 | 计算四柱八字、纳音、藏干 | paipan.py | paipan dict |
| **Step2 建BaZi** | step1的pillars | 组装配件对象 | constants.BaZi | BaZi对象 |
| **Step3 引擎评分** | BaZi对象+birth日期 | 跑36个模块全量计算 | pipeline_v5 → 36个.py | 22个sec_* dict |
| **Step4 加载理论** | 当前要写的§编号 | 加载该§对应的理论规则 | skill_view('bazi-xxx') | 理论规则清单 |
| **Step5 匹配数据** | step3数据+step4规则 | 逐条对比规则vs数据 | 人脑推理 | 验证结论 |
| **Step6 写分析** | step5结论 | 转化为可读文本 | 人脑+文字输出 | 分析段落 |
| **Step7 组合报告** | step6的21个§ | 按模板顺序组合 | write_file | 完整报告 |
| **Step8 推库** | step7报告文件 | 保存到知识库 | git | 推库成功 |

## 我当前流程中的问题点

对照这个流程，我的问题出在：

1. **Step4→Step5 执行不到位**：虽然加载了理论技能，但没有「先列出理论规则清单再逐条匹配」，而是凭记忆直接写。导致出现了「金生土写成土生金」「戊土正印写成偏印」这种低级错误。

2. **Step5→Step6 跳步**：应该先写出「规则→数据→结论」的显式对照，再去润色成自然语言。我经常跳过对照直接写结论，容易出错。

3. **Step6 深度不够**：写出来的分析内容偏浅，比如§16事件表只写了30多条（要求≥70条），§17大运精析有的运只有几句话。

4. **没有校验机制**：写完后没有回头检查Step5的对照表是否全部一致。

## 修复方案

| 问题 | 修复方法 |
|:-----|:---------|
| 跳步（Step4→Step5→Step6） | 每写一个§前，先强制写「规则清单 → 数据对照表 → 结论」三步 |
| 低级错误（十神/生克写反） | 写完后用**step3的引擎原始数据**逐项核对自己的结论，不一致的立刻停 |
| 深度不够 | §16事件表≥70条、§17每运≥200字、总行数≥1500 |
| 无校验 | 出报告前过一遍「引擎数据 vs 报告结论」的一致性检查 |
