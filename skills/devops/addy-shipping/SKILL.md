# Shipping and Launch (生产级发布上线)

## 概述

分阶段部署，持续监控，快速回滚。每次部署都是低风险的，因为：
1. 变更规模小
2. 有功能标志控制
3. 有监控告警
4. 有回滚准备

## 预发检查清单

- [ ] 代码审查通过
- [ ] 所有测试通过
- [ ] CI管道通过
- [ ] 性能评估完成
- [ ] 安全扫描完成
- [ ] 功能标志配置好
- [ ] 监控告警设置好
- [ ] 回滚计划准备好
- [ ] 文档更新（如果需要）
- [ ] 干系人通知

## 分阶段部署

```
Canary (5%) → 扩展 (25%) → 扩展 (50%) → 全面 (100%)
    │            │              │             │
    ▼            ▼              ▼             ▼
  监控响应时间  监控错误率    监控业务指标   完成
```

### 阶段1: Canary (5%)
- 部署到5%的用户
- 监控：无错误率上升
- 持续时间：最小30分钟
- 如果错误率上升 → 回滚

### 阶段2: 扩展 (25%)
- 部署到25%的用户
- 监控：验证指标
- 持续时间：最小1小时

### 阶段3: 扩展 (50%)
- 部署到50%的用户
- 监控：检查业务指标

### 阶段4: 全面 (100%)
- 部署到所有用户
- 监控：持续观察

## 回滚流程

```
1. 检测到问题（监控告警或用户报告）
2. 立即回滚到上一个良好版本
3. 停止管道
4. 调查根因
5. 修复后正常部署
```

## 功能标志生命周期

```
创建 → 默认关闭 → Canary开启 → 逐步扩大 → 全部开启 → 清理代码
  │         │           │           │          │         │
  │         用户不可见  5%用户可见  25%→50%→75%  全部可见  移除标志
```

## 部署后监控（24小时）
- 错误率：无上升
- 延迟：无上升
- 业务指标：正常
- 用户报告：无负面反馈
- 系统资源：正常
