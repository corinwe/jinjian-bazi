# 叙述引擎与标准报告生成管线 v1.0

## 概述
引擎管线输出确定性的21§JSON数据后，通过叙述引擎（narrative functions）将结构化数据转化为命理师口吻的连贯段落，再由标准报告生成器格式化为.md报告。整个过程零LLM、函数调用、确定性。

## 文件结构

```
engine/
├── narratives.py              # 21个 narrative_*() 函数
├── narrative_integration.py   # add_narratives(result) → 注入narrative字段
├── pipeline_v5.py             # run_pipeline() → 调用add_narratives
├── report_standard_gen.py     # 标准报告生成器（引擎JSON→.md）
└── _gen_detail_analysis.py    # detail_analysis规则分析文本（桥梁层）
```

## 数据流

```
用户八字 → pipeline_v5.run_pipeline()
  → 引擎21§确定性计算（身强弱/财星/格局/大运/维度等）
  → _gen_detail_analysis.attach_detail_analysis()（规则分析文本）
  → narrative_integration.add_narratives()（命理师口吻叙述）
  → 完整JSON（含 detail_analysis + narrative 双字段）
  → report_standard_gen.gen_report()（格式化为标准.md报告）
  → 知识库推库
```

## 报告行数

- 引擎 + narrative函数 + 标准报告生成器 → **~380行/报告**
- 旧版 sub-agent 深析报告 → **1700+行/报告**
- 差距原因：narrative函数每节200-500字精炼叙述，旧版用LLM大幅展开
- 要增加行数：扩充narrative函数到每节500-800字 + 增加格式化表格

## 生成命令

```bash
# 1. 跑引擎管线
python3 engine/pipeline_v5.py --name 某人 --gender 男 --year 1980 --month 8 --day 6 --hour 5 --json > /tmp/out.json

# 2. 生成标准报告
python3 engine/report_standard_gen.py /tmp/out.json "姓名" "性别" > 报告.md

# 3. 推库
cd /root/.hermes/weiwuji-knowledge-base
git add -A && git commit -m "📖 报告" && git pull --rebase && git push
```
