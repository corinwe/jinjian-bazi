# FastAPI三层架构实现模式与踩坑实录

## 架构模式

### 三层分离

```
database/       ← 数据层：SQLite/PG，5张核心表+预留字段
api/            ← API服务层：FastAPI异步+线程池
  ├── routers/  ← 路由层：HTTP端点
  ├── schemas/  ← 校验层：Pydantic请求/响应模型
  ├── services/ ← 业务层：编排规则引擎+数据库
  └── repositories/ ← 数据访问层：SQL CRUD
engine/         ← 规则引擎：纯计算，独立子进程
  ├── config/   ← JSON配置文件（藏干/纳音/神煞/称骨等）
  ├── step1_basic.py ← 第一步：11行×4列原始数据
  └── pipeline_product.py ← CLi/Call入口
frontend/       ← 前端：SPA，静态文件
```

### 数据流

```
用户请求 → POST /api/v1/analyze
  → ThreadPoolExecutor (async)
    → AnalysisService.analyze()
      → call_engine() [subprocess, 独立进程]
        → python3 pipeline_product.py --json
          → bazi-engine.py (排盘)
          → step1_basic.py (11×4基础数据)
          → 9个分析模块 (全部代码)
      → 存入数据库
      → 返回结构化JSON
  → Pydantic验证 → HTTP 200
```

## 踩坑实录

### 1. ProcessPoolExecutor vs ThreadPoolExecutor

**错误**：用 `ProcessPoolExecutor` 执行分析任务。

```python
_executor = ProcessPoolExecutor(max_workers=4)  # ❌
_service = AnalysisService()                     # ❌ 包含sqlite3.Connection对象
result = await loop.run_in_executor(_executor, _service.analyze, ...)
```

**错误原因**：`sqlite3.Connection` 对象不能被pickle， `ProcessPoolExecutor` 需要序列化所有参数。

**修复**：改用 `ThreadPoolExecutor`，引擎调用是I/O等待（subprocess），不是CPU密集。

```python
_executor = ThreadPoolExecutor(max_workers=4)  # ✅
# 每次调用创建新的Service实例
def _run_analysis(request):
    svc = AnalysisService()
    return svc.analyze(...)
```

### 2. Pydantic Schema字段传递

**错误**：`AnalyzeResponse` 返回时只传了部分字段。

```python
return AnalyzeResponse(
    analysis_id=result["analysis_id"],
    status="completed",
    basic=result["basic"],
)  # ❌ 没传analysis参数，client收到analysis: null
```

**修复**：显式传递所有必要字段。

```python
return AnalyzeResponse(
    analysis_id=result["analysis_id"],
    status="completed",
    basic=result["basic"],
    analysis=result.get("analysis"),   # ✅
    message=result.get("message"),
)
```

### 3. 引擎输出键名变更

**错误**：假设引擎输出中有旧版键名。

```python
paipan = engine_result.get("paipan", {})
engine_result.get("pillars_raw", {})  # ❌ 这个键不存在！
```

**修复**：所有排盘数据在 `paipan` 键中（year_gan, year_zhi等）。

```python
year_pillar = paipan.get("year_gan", "") + paipan.get("year_zhi", "")  # ✅
```

## 数据库设计要点

### 预留字段模式

```sql
-- 在 analysis_results 表中预留流月/流日字段
CREATE TABLE analysis_results (
    ...
    liu_nian TEXT,  -- 流年（已有引擎支持）
    liu_yue  TEXT,  -- 预留
    liu_ri   TEXT,  -- 预留
    extra    TEXT,  -- 任意扩展JSON
);
```

扩展新维度时：**不改表结构，直接填JSON字段，前端读取展示即可。**

### Repository模式

每个表一个Repository类，继承 `BaseRepository`：

```python
class BaseRepository:
    @property
    def conn(self) -> sqlite3.Connection:
        if self._conn is None:
            self._conn = get_connection()
        return self._conn
```

**注意**：每次调用 `service.analyze()` 时创建新实例，避免连接复用冲突。
