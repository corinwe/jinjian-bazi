# Pipeline v5.0 — 确定性规则引擎架构

> 创建时间：2026-06-26
> 触因：老板指出报告质量不稳定——核心计算正确但补充模块全是硬编码占位符

## 核心变更

| 旧版（v4.0） | 新版（v5.0） | 文件 |
|:------------|:------------|:-----|
| `pipeline_v4.py` | `pipeline_v5.py` | 引擎入口（已上线） |
| `comprehensive.py`(硬编码) | `comprehensive_v2.py`(29个确定性函数) | 事业/子女/健康/外貌/置业/建议 |
| `education.py`(简单) | `education_v2.py`(年柱三档法+文昌双轨) | 学历分析 |
| `marriage.py`(简单) | `marriage_v2.py`(配偶星替代+四大信号) | 婚姻分析 |
| `dimensions.py`(简单) | `dimensions_v2.py`(校准版评分) | 8大维度 |
| `pipeline_product.py` | `pipeline_v5.py` | 引擎CLI入口 |

## 修复的bug

| Bug | 旧版 | 新版 |
|:----|:-----|:-----|
| 纳音为空 | `["","","",""]` | `["石榴木","白蜡金",...]` |
| 大运end_year | `undefined` | `start_year + 9` |
| 四柱表空白 | `pillars`为空 | 完整十神/天干/地支/纳音 |
| 学历硬编码 | 固定字符串 | 年柱三档法+六步排查+文昌检查 |
| 婚姻硬编码 | 永远"中等" | 配偶星定位+四大信号+质量评分 |
| 事业硬编码 | 永远"教育/文化" | 格局定方向+身强弱定级别 |
| 健康硬编码 | 简单五行检查 | 五行过三+七杀断病法 |
| 子女硬编码 | 永远"1-2个" | 十二长生基数法 |

## 引擎调用链

```\nengine_client.py (subprocess)\n  → python3 engine/pipeline_v5.py --name X --gender X --year X --month X --day X --hour X --json\n    → paipan.paipan() 排盘\n    → run_v5() 完整分析:\n       - shen_qiang_ruo (身强弱)\n       - cai_xing (财星)\n       - determine_ge_ju (格局)\n       - compute_da_yun (大运)\n       - analyze_education (学历v2)\n       - analyze_marriage (婚姻v2)\n       - run_comprehensive_engine (综合v2)\n       - DEFAULT_DIMENSIONS (8维度v2)\n    → attach_detail_analysis() ① 规则分析文本（21§全量）\n    → add_narratives() ② **命理师口吻叙述段落（21§全量）【2026-06-27新增】**\n    → JSON输出 (含paipan/basic_data/analysis/result/text)\n```
