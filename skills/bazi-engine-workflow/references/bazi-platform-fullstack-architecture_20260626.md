## 关键踩坑记录

### 1. types.py命名冲突（D级致命）
`engine/types.py` 与 Python 标准库 `types` 模块冲突。当 `engine/` 目录加入 sys.path 后，`from types import BaZi` 匹配到了 Python 内置 `types` 模块。
**修复**: 重命名为 `constants.py`，所有内部 import 改为 `from constants import ...`。
**规则**: 引擎文件永远不要用 `types.py`、`utils.py`、`common.py`、`helpers.py`。

### 2. 日主不计入天干比劫（2026-06-26修正）
身强弱评分循环中包含了日干（日主自己），导致评分被错误抬高。日主是"我"，不能作为生扶自己的力量来源。
**修复**: 天干比劫计分范围 = 年干 + 月干 + 时干（排除日干）。
**案例**: 家主从85.6分->73.6分，修正-12分。

### 3. 引擎路径导入（FastAPI后端）
后端 `analyze.py` 中导入引擎的正确方法：
```python
ENGINE_DIR = os.path.abspath(os.path.join(os.path.dirname(__file__), '../../..', 'engine'))
sys.path.insert(0, ENGINE_DIR)
from pipeline_v4 import run_v4  # 不加 engine. 前缀
from constants import BaZi, Pillar  # 不加 engine. 前缀
```

### 4. 品牌统一（九龙道长→金鉴真人）
所有技能文件中的"九龙道长原始规则"已替换为"金鉴真人原始规则"。
**规则**: 推库前执行 `grep -c "九龙道长" {file}` → 应为0匹配。

## 流年事件断法逻辑链

每个流年事件遵循以下逻辑链：
1. 身强弱 → 定喜忌
2. 流年十神 → 定吉凶方向
3. 流年地支与大运/原局关系 → 定触发方式（冲刑合害）
4. 五行能量变化 → 定程度大小
5. 神煞 → 定性修正
6. 综合评分 + 置信度
7. 输出事件类型 + 描述 + 置信度(0-100%)

6种事件类型：
- 💰 发财: 财星流年+身强+喜用+三合财局/财库冲开
- ⚠️ 灾祸: 七杀攻身+忌神+冲刑+犯太岁+冲大运
- 💍 结婚: 夫妻宫引动+正财/正官到位+印星帮身
- 💼 事业: 官印流年+驿马引动+冲合大运
- 📚 学业: 印星到位+文昌引动
- 🏥 健康: 五行过三+七杀攻身
