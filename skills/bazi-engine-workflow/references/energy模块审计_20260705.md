对照 `constants.py` 的 `BASE_SCORE` / `CANG_GAN_RATIO`、`shen_qiang_ruo.py` 权重体系、`bazi-foundation-analysis` 原始规则逐行审计。

## 审计发现

### P1 缺陷: `sheng_wo` 生克关系逆向（line 109）

```python
# ❌ 当前错误代码:
sheng_wo = WU_XING_SHENG.get(ri_wx, "")
# WU_XING_SHENG = {"木": "火", "火": "土", ...}
# 日主=木时, sheng_wo="火" → 这是"我生"(食伤), 不是"生我"(印)！

# ✅ 正确应为:
sheng_wo = {v: k for k, v in WU_XING_SHENG.items()}.get(ri_wx, "")
# 日主=木时, sheng_wo="水" → 这是"生我"(印)
```

**影响**：`ri_zhu_energy` 子对象的生扶能量计算错误。**万幸：该子对象未被任何下游模块引用**（仅 energy.py 自己的 `__main__` 测试块使用）。

### P2 设计缺陷: 五行能量分布使用扁平权重（未用 BASE_SCORE）

| 位置 | 全引擎标准 (BASE_SCORE) | energy.py 实际值 | 偏差倍数 |
|------|------------------------|-----------------|---------|
| 年支 | 4分 | ~1.9分（本气1.0 + 中气0.6 + 余气0.3） | 年支偏大~2x |
| **月令** | **40分** | **~1.9分** | **月令偏小~21x** |
| 日支 | 12分 | ~1.9分 | 日支偏小~6x |
| 时支 | 12分 | ~1.9分 | 时支偏小~6x |
| 时干 | 12分 | 1.0分 | 时干偏小~12x |

**设计意图分析**：该模块用于"五行过三"疾病判断场景（>3=某五行过旺致病），扁平计数在此场景下有合理性。但输出被用于通用"五行能量分布"用途（pipeline_v5 API 输出），造成数值与 `shen_qiang_ruo.py` 等模块无法对齐。

### P3 规范缺陷: 硬编码权重未引用 CANG_GAN_RATIO

`constants.py` 已有 `CANG_GAN_RATIO = {100: 1.0, 60: 0.6, 30: 0.3}`，但 energy.py 直接硬编码 `+= 1.0` / `+= 0.6` / `+= 0.3`。其他模块（shen_qiang_ruo.py line 163, cai_xing.py line 54）均使用 `CANG_GAN_RATIO.get(ratio, 0)`。

### P3 废弃代码: `ri_zhu_energy` 子对象（lines 124-129）

`ri_zhu_energy` 包含的身强弱判定（含 BUG）完全未被任何下游使用 — pipeline_v5 的身强弱来自 `shen_qiang_ruo.py` 的 `compute_shen_qiang_ruo()`。

### P3 尺度不兼容: `relation_energy` 量级脱敏

`relation_energy` 使用 `NENG_LIANG` 系数（六冲=10, 三刑=15...）量级为5~20，而 `wu_xing_energy` 分值为0.6~4.0。二者尺度相差5~30倍。
