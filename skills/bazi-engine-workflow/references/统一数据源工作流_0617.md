# 统一数据源工作流（2026-06-17新增）

## 架构升级
不再每次重新跑引擎，而是从统一数据源读取确定性数据。
引擎只在以下两种情况使用：
1. 首次录入新八字时
2. 用户确认日期/时辰有误需修正时

## 数据源文件
- 知识库：`/root/weiwuji-knowledge-base/07-国学哲学/八字命格/家族八字核心数据源_v1.json`
- 技能目录：`/root/.hermes/profiles/jinjian-zhenren/scripts/family_bazi_data.json`

## 工作流程

任何时候需要输出/更新八字报告：

1. 从数据源取人
   - 从 family_bazi_data.json 读取目标人物数据
   - 确认数据存在（不存在则先确认八字再追加到数据源）

2. 生成报告
   - 所有数字字段从数据源JSON直接填入模板
   - LLM不计算任何数字，只写分析文字
   - 维度清单逐一覆盖（婚姻/子女/财富/事业/健康/学业/六亲）

3. 输出前验证
   - 自动比对：报告八字 == 数据源.八字
   - 自动比对：报告身强弱分数 == 数据源.身强弱.总分
   - 自动比对：报告财星分数 == 数据源.财星.总分

4. 推库
   - 报告写入知识库
   - 如有数据源变更，同时更新JSON
   - git add/commit/push

口诀：
  数据源是唯一真理，报告数字必须对齐
  引擎只在录入时跑，日常使用从源取
  输出之前先验证，数据源报告两相符
