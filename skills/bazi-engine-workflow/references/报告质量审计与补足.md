# 报告质量审计与补足重写清单

> 编制人：金鉴真人
> 编制时间：2026年06月22日
> 关联技能：bazi-engine-workflow §批量多人处理工作流 ⑥

## 一、审计触发

每次批量生成报告后（≥3人），自动运行审计。

## 二、审计阈值

| 指标 | 合格 | 需补足 | 需重写 |
|:----|:----|:------|:------|
| 总行数 | ≥1500行 | 800~1499行 | <800行 |
| §板块覆盖 | ≥20个§ | 16~19个§ | <16个§ |
| 关键板块(7个) | ≥6个 | 4~5个 | <4个 |
| §16事件表 | ≥70行 | 40~69行 | <40行 |

## 三、关键板块检查清单

7个强制检查板块：
```
§1 一页总览表（25字段四段式）
§2 格局分析（三维度）
§3 身强弱详解（含评分明细）
§4 喜用神详解（含用神窗口）
§8 财富分析（含九龙道长五级对照）
大运精析（≥8步，每步含流年简析）
§18 三决断（6要素断语格式）
```

## 四、补足策略矩阵

| 现状 | 补足方案 |
|:----|:---------|
| 行数<800 | 整份重写（delegate_task重新生成） |
| 800~1200行·缺后半 | 追加§11~§21（学历/婚姻/子女/健康/大运/事件表/人生建议） |
| 1200~1499行·缺深度 | 每个§扩充分析（加案例/加表格/加白话解读） |
| 缺关键板块 | 用patch在对应位置插入该板块 |
| 缺§16事件表 | 追加≥70行事件总表（按大运分段·9类事件全覆盖） |
| 缺§18三决断 | 生成6要素格式的三决断段落 |

## 五、重写时delegate_task context模板

```

八字: {四柱}

忌: {五行}>...
大运: {大运序列}

```

## 六、验收命令

```bash
# 行数
wc -l {报告路径}

# §覆盖
grep -c "^## §" {报告路径}

# 关键板块
grep -c "§1" {报告路径}
grep -c "§8" {报告路径}
grep -c "大运" {报告路径}
grep -c "三决断" {报告路径}
grep -c "事件总表\|全生命周期" {报告路径}
```
