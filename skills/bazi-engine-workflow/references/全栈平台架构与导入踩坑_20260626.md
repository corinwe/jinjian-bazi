# 全栈平台架构与导入踩坑记录（2026-06-26）

## 1. Python `types` 命名冲突

**问题**：引擎 `types.py` 与 Python 标准库 `types` 模块冲突。
当 `engine/` 目录加入 sys.path 后，`from types import BaZi` 匹配到了 Python 内置 `types` 模块。

**修复**：`mv types.py constants.py`，所有内部 import 改为 `from constants import ...`。

**涉及文件**（全部改 import）：
- `engine/constants.py`（原 types.py）
- `engine/shi_shen.py` 
- `engine/shen_qiang_ruo.py`
- `engine/cai_xing.py`
- `engine/ge_ju.py`
- `engine/da_yun.py`
- `engine/dimensions.py`
- `engine/energy.py`
- `engine/liu_nian.py`
- `engine/pipeline_v2.py`
- `backend/app/routers/analyze.py`

**防再犯**：永远不用 `types.py`、`utils.py`、`common.py`、`helpers.py` 命名文件。

## 2. 引擎路径导入（FastAPI）

`backend/app/routers/analyze.py` 中导入引擎的正确方式：

```python
ENGINE_DIR = os.path.abspath(os.path.join(
    os.path.dirname(__file__), '..', '..', '..', '..', 'engine'))
# 从 routers → app → backend → platform → +engine
if ENGINE_DIR not in sys.path:
    sys.path.insert(0, ENGINE_DIR)

# 直接导入（不加 engine. 前缀，因为 ENGINE_DIR 就是引擎根目录）
from pipeline_v2 import run_v2
from constants import BaZi, Pillar
```

**不要用** `from engine.pipeline_v2 import run_v2` — 这会在 `engine/engine/pipeline_v2.py` 路径下找，不存在。

## 4. 前端注意事项

- SPA使用原生JS（无框架依赖），页面切换用 `display:none/block` + `navigate()` 函数
- 所有API请求走 `api()` 函数，自动携带JWT token
- 分析表单默认填充子源八字示例（庚申辛巳甲午丙寅）
- 报告渲染使用 `renderReport(data, reportId)` 函数
- 响应式设计：`@media (max-width: 600px)` 切换单列布局
