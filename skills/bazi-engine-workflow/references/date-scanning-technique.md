## 故事（凤案例·2026-06-23）

凤的八字被搞错了3次：

| 版本 | 八字 | 日主 | 日期 | 来源 |
|:----|:-----|:----:|:----:|:-----|
| ❌ 旧数据源 | 戊午甲子**辛亥**己亥 | **辛**金从弱 | 12月15日 | 旧版未校准 |
| ❌ v1.0报告 | 戊午甲子**己酉**乙亥 | **己**土 | 12月13日 | 引擎跑错 |
| ✅ **正确** | 戊午甲子**庚戌**丁亥 | **庚**金身弱 | **12月14日** | 老板纠正 |

根因：1978年12月13/14/15三天，日柱分别是己酉/庚戌/辛亥，连续三天日主分别是己土/庚金/辛金。差1天整份报告报废。

## 技术：日期扫描法

当出生日期不确定或八字有矛盾时，扫描相邻日期定位正确日柱。

```python
from hermes_tools import terminal

# 步骤1 — 确定扫描范围
# 从用户给的前后3-5天开始扫描
birth_year = 1978  # 用户给的年份
birth_month = 12   # 用户给的月份

for day in range(10, 20):  # 扫描前后5天
    r = terminal(
        f"python3 .../bazi-engine.py {birth_year} {birth_month} {day} 21 0 11 女 测试 --json 2>/dev/null "
        f"| python3 -c 'import sys,json; d=json.load(sys.stdin); print(d[\"四柱\"][\"日柱\"], d[\"日主\"])'",
        timeout=15
    )
    print(f"  {birth_month}月{day}日: {r['output'].strip()}")
```

## 口诀

日期不对别硬算，日柱可能差一天
前后三天扫一遍，看谁日主匹配对
再问用户看特征，二月冬还是夏
引擎官网双确认，一致才能算正确
