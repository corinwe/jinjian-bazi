# 🚨 引擎大运年份bug修复记录（2026-06-25）

## bug描述
引擎文本输出大运年份时，用了 `int(qi_yun_age)` 截断起运年龄小数部分：
```python
# 旧代码（有bug）
b_year = r['birth'].year  # 1980
s_year = b_year + int(qi_yun_age)  # int(0.37)=0 → 1980 ❌
e_year = b_year + int(end_a)       # 1990 ❌
```

导致起运0.37年的八字，大运从1980年开始算（正确应为1981年）。

## 修复
文本输出改用 `calc_da_yun_with_age()` 函数的年份数据：
```python
# 新代码（正确）
da_yun_age_data = calc_da_yun_with_age(r)
d = da_yun_age_data[i]
s_year = d['起始年份']  # 1981 ✅
e_year = d['终止年份']  # 1990 ✅
```

同时修复了 `calc_da_yun_with_age()` 中的年龄计算：
- 旧：`start_age = round(qi_yun_age + i * 10, 1)` → 0.37岁起运
- 新：`start_age_base = math.ceil(qi_yun_age)` → 起运年龄向上取整 → 1岁起运


## 根因
文本输出函数（print部分）和JSON输出函数（calc_da_yun_with_age）用了两个不同的大运年份计算逻辑。文本输出是手动算的（用了int截断），JSON输出用了正确的起运日期计算。修复后两者统一使用calc_da_yun_with_age。
