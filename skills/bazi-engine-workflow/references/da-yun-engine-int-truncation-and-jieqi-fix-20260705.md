# 大运引擎int截断Bug与节气天数硬编码修复记录

> **编制:** 金鉴真人
> **触发:** 2026-07-05 老板指出大运计算有误（起运年接近1981但引擎算出1980）
> **状态:** ✅ 已修复并推库 `7a1f2c3`
> **影响模块:** `da_yun.py`, `constants.py`, `pipeline_v5.py`, `bazi-must-run-engine.sh`

## 一、发现的两个致命Bug

### Bug ①: int截断导致大运年龄区间全错

**文件:** `engine/da_yun.py`

```python
# ❌ 旧代码（有Bug）
qi_yun_age = round(1.1 / 3, 2)  # = 0.37 ✅
qi_yun_age_int = int(qi_yun_age)  # = 0  ← ❌ 截断了小数

# 大运年龄用int计算:
start_age = qi_yun_age_int + i * 10  # = 0 + 0 = 0 ❌ 应为0.37
end_age = qi_yun_age_int + (i + 1) * 10 - 1  # = 0 + 10 - 1 = 9 ❌ 应为10.37
```

| 影响 | 旧值 | 正确值 |
|:----|:-----|:-------|
| 第1步起始年龄 | **0岁** | **0.4岁** |
| 第1步终止年龄 | **9岁** | **10.4岁** |
| 第1步起始年份 | **1980年** | **1980年**（小数部分0.37<0.5不进位） |
| 起运年份判定阈值 | 0.7（不合理） | 改为0.5（符合四舍五入逻辑） |

**根因分析:** 开发者误以为 `int(qi_yun_age)` 能保留精度，实际上 `int(0.37)=0`，所有大运的起始年龄都少算了0.37岁。同时 `end_age = start_age + 10 - 1` 公式也错了——10年跨度的终止年龄应为 `start_age + 10`（开区间），而不是 `start_age + 9`（闭区间）。

### Bug ②: 节气天数硬编码1.1天

**影响所有pipeline文件:**

| 文件 | 行号 | 代码 |
|:----|:----:|:------|
| `pipeline_v5.py` | 135 | `run_v5(bazi, ..., qi_yun_days=1.1)` |
| `pipeline_v4.py` | 30 | `run_v4(bazi, ..., qi_yun_days=1.1)` |
| `pipeline_v3.py` | 50 | `... qi_yun_days=1.1` |
| `pipeline.py` | 24 | `... qi_yun_days=1.1` |

1.1天 ≈ 离家主最近的节气(立秋)仅1天，对家主恰好正确。
但对其他所有人，节气距离完全不同，用1.1全错：

| 人物 | 生日 | 节气距离 | 硬编码起运 | 正确起运 | 误差 |
|:----|:----|:---------|:----------|:---------|:-----|
| 家主 | 1980-08-06 | 到立秋~1天 | 0.37岁 | 0.33岁 | **OK** |
| 主母 | 1987-07-20 | 到立秋~18天 | 0.37岁 | **6.0岁** | ❌ **差5.6岁** |
| 子源 | 2011-05-31 | 到立夏~25天 | ~0.7岁 | **8.33岁** | ❌ **差7.6岁** |

## 二、修复方案

### 修复①: int截断 → 浮点数计算

```python
# ✅ 修复后
start_age = round(qi_yun_age + step * 10, 1)   # 0.4, 10.4, 20.4, ...
end_age = round(qi_yun_age + (step + 1) * 10, 1)  # 10.4, 20.4, 30.4, ...
start_year = birth_year + int(qi_yun_age + step * 10)  # 1980, 1990, 2000, ...
```

同时需要 `constants.py` 中 `DaYun.start_age` 的类型从 `int` 改为 `float`。

### 修复②: 节气天数硬编码 → 自动计算

新增 `compute_qi_yun_days()` 函数（位于 `da_yun.py`）：

```python
def compute_qi_yun_days(birth_year, birth_month, birth_day, month_zhi, is_shun) -> float:
    """
    根据出生日期和月支，自动计算到下一个/上一个节气的天数
    基于简化节气表（±1天误差）
    """
```

节气表采用月支序号映射：

```python
JIE_QI = {
    1: (2, 4),   # 寅→立春·2月4日
    2: (3, 6),   # 卯→惊蛰·3月6日
    3: (4, 5),   # 辰→清明·4月5日
    4: (5, 6),   # 巳→立夏·5月6日
    5: (6, 6),   # 午→芒种·6月6日
    6: (7, 7),   # 未→小暑·7月7日
    7: (8, 7),   # 申→立秋·8月7日
    8: (9, 8),   # 酉→白露·9月8日
    9: (10, 8),  # 戌→寒露·10月8日
    10: (11, 7), # 亥→立冬·11月7日
    11: (12, 7), # 子→大雪·12月7日
    12: (1, 6),  # 丑→小寒·1月6日
}
```

关键逻辑：
- **顺排（阳男阴女）**: 到**下一个节气**的天数 = 月支的下一个序号的节
- **逆排（阴男阳女）**: 到**上一个节气**的天数 = 月支对应的节

### 修复③: 大运步数从8步改为动态

```python
# ❌ 旧: range(8)
# ✅ 新: 按100岁寿命计算所需步数
max_steps = max(8, int((100 - qi_yun_age) / 10) + 2)
max_steps = min(max_steps, 12)
```

### 修复④: pipeline_v5参数传递

```python
# ❌ 旧: run_v5(bazi, birth_year, birth_month_lunar, qi_yun_days=1.1)
# ✅ 新: run_v5(bazi, birth_year, birth_month, birth_day, qi_yun_days=None)
```

`qi_yun_days=None` 触发自动节气计算。同时需要传递 `birth_day`。

### 修复⑤: 门禁脚本调用对齐

```bash
# ❌ 旧: full_result = run_v5(bazi, birth_year=$YEAR, birth_month_lunar=$MONTH)
# ✅ 新: full_result = run_v5(bazi, birth_year=$YEAR, birth_month=$MONTH, birth_day=$DAY)
```

## 四、铁律（写大运前必过）

```yaml
□ 节气天数: 必须基于实际节气表计算，禁止硬编码1.1
□ 起运年龄: 用浮点数 (days/3)，禁止 int() 截断
□ 大运年龄: 用浮点 start_age/end_age，禁止 int 丢失精度
□ 起运年份: 小数>=0.5进1，禁用旧阈值0.7
□ 步数: 动态计算(最大100岁)，禁止固定8步
□ 一致性: compute_da_yun 的返回值必须与 pipeline 输出对齐
□ 门禁: bazi-must-run-engine.sh 必须传递 birth_day 参数
```
