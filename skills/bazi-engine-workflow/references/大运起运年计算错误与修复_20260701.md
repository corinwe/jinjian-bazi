### 1.4 引擎代码int()截断Bug（2026-07-07发现）

| 问题描述 | 根因 | 修复 |
|:---------|:-----|:-----|
| 家主起运0.33岁→`int(0.33)=0`→甲申算成1980~1989 | `da_yun.py`用`int(qi_yun_age + step*10)`截断浮点 | 改用浮点月份偏移计算 |

**引擎修复详情（da_yun.py）:**

旧公式（含Bug）:
```python
start_year = birth_year + int(qi_yun_age + step * 10)
# int(0.33)=0 → 甲申start_year=1980 ❌
```

新公式:
```python
start_age_float = qi_yun_age + step * 10
start_age_int = int(start_age_float)
start_month_offset = int(round((start_age_float - start_age_int) * 12))

effective_month = birth_month + start_month_offset
year_carry = 0
while effective_month > 12:
    effective_month -= 12
    year_carry += 1

start_year = birth_year + start_age_int + year_carry

# Q4进位: 起运在10~12月 → 只占当尾巴 → 跳到下一年
if effective_month >= 10:
    start_year += 1
```

**原始理论依据：** "一天为四个月"（不是"一天=0.33年"用int截断）。浮点年需还原成月份再从出生月偏移计算整年区间。

**Q4进位规则：** 当偏移后月份≥10（第4季度），该年只占最后2~3个月，大运主要覆盖次年起的10年，故显示年份+1。

## 二、铁律（已固化到 bazi-platform-harness 铁律⑤）

```
强制公式: (结束年 - 开始年 + 1) = 10

写大运前四步校验:
1. 读数据源 — 从JSON精确提取起运年龄
2. 算年份范围 — 起运年龄 + 10n = 第n+1运的结束年龄
3. 交叉验证 — 大运干支与引擎排盘输出一致（顺排/逆排）
4. 最终检查 — 每行大运的结束年-开始年+1 = 10
```

## 四、验证清单（每次写报告前必过）

### 4.1 基础大运验证

```yaml
□ 第1步：从JSON读起运年龄 → 不是凭记忆写
□ 第2步：确定顺排/逆排（阳男阴女顺，阴男阳女逆）
□ 第3步：计算每个大运的起始/终止年份（起运年 + 10n）
□ 第4步：逐行验证 (结束年 - 开始年 + 1) = 10
□ 第5步：确认当前年份所在大运正确
□ 第6步：确认流年表中每行的大运列正确
□ 特别检查：未成年人的当前大运（如子源15岁→壬辰运）
```

### 4.2 财富报告大运额外验证（2026-07-01新增）

```yaml
□ 大运年龄区间连续性检查：前一运结束年龄+1=后一运开始年龄
  （如丙戌22-31→丁亥32-41✅，丁亥30-40❌）
□ 起运年龄从头计算：禁用从旧报告直接复制大运数据
  必须算节气天数→除以3→取整
□ 文昌处理：尊重老板最终决定权
  老板说「不用补」→ 报告改为「无需补」
  老板说「补上」→ 加文昌补法
□ 10年跨度的3种检查：
  ① 年份跨度：(end_year - start_year + 1) = 10
  ② 年龄跨度：(end_age - start_age + 1) = 10  
  ③ 当前年份必须落在正确的大运区间内
□ 大运列修正后，分析文本需同步协调（2026-07-01新增教训）：
  当大运列从庚戌→辛亥后，以下分析文本必须同步更新：
  - 「庚戌伏吟」→ 流年庚戌+大运辛亥≠伏吟，需改为正常分析
  - 「申戌拱金」→ 流年戊申+大运辛亥=申亥相害，需改为正确组合
  - 「酉戌相害」→ 流年己酉+大运辛亥=正常，需重新评估
  铁律：改大运列只是第一步，必须同步协调分析文本
```

### 4.3 文昌铁律（2026-07-01固化）

引擎文昌检查只提供参考，老板的最终决定权在文昌处理中为最高优先级：
- 引擎说「需补」→ 老板说「不用补」→ 改报告为「无需补」
- 引擎说「需补」→ 老板说「补上」→ 加完整文昌补法
- 引擎说「自带」→ 老板说「不用补」→ 一致，不动
- ❌ 禁止用引擎数据反驳老板的文昌决定

**口诀：文昌补不补，老板说了算；引擎做参考，决定权归主。**
