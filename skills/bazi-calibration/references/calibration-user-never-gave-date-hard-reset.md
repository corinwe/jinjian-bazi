# 用户说「我没给过你」硬重置七步法

> 编制人：金鉴真人
> 编制时间：2026年06月11日 23:30
> 版本：v1.0
> 来源：主母生辰硬重置实战（误记庚申日，用户指出从未给过此日期）

## 触发条件

| 用户语句 | 含义 | 优先级 |
|:---------|:-----|:------:|
| 「我没给过你这个日期」 | 我记忆中的日期不来自用户 | 🔴 |
| 「你从哪里推出来的？」 | 推导过程可疑 | 🔴 |
| 「我没说过……」 | 数据来源与原始输入不符 | 🔴 |
| 「那不是我的意思」 | 误解了用户意图 | 🟡 |

## 七步流程

### 第0步：立即停止
当前分析→立刻终止输出，不输出任何基于怀疑数据的结论

### 第1步：确认用户原始输入
只基于用户新确认的数据，不使用任何历史/记忆数据

### 第2步：全量清除相关记忆
搜索memory中所有相关数据，全部标记存疑，只保留用户新确认的

### 第3步：从零重新排盘
农历→公历用代码计算→跑bazi-engine.py→显示结果让用户确认

### 第4步：逐一排除候选
用家族合参多维度比较法评分，排除理由必须基于八字理论

### 第5步：更新所有依赖数据
更新memory→家族总表→重写报告(v6.0起)→旧报告加勘误声明

### 第6步：推知识库
git add/commit/push

### 第7步：永久固化到技能库
写入bazi-calibration踩坑记录 + family-birthtime-disambiguation.md

## 主母案例复盘

Session A：排盘发现庚午/辛巳都不完美，硬凑庚申为答案
Session B~C：基于错误庚申日写报告
Session D~E：固化错误到memory
Session F：用户指出「我没给过你这个日期」→硬重置触发

## 教训
1. 不要自己编造日期——用户给的日期就是全部候选范围
2. 不要「硬凑」命理吻合——庚申=家主权庚申看似有缘，但用户没给
3. memory的「校准标注」可能本身就是错的
4. 用户的否定直接覆盖所有历史记忆
