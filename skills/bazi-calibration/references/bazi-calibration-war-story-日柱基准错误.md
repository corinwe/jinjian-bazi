# 踩坑记录：日柱基准错误（1900-01-01≠甲子日）

> **日期**：2026-06-06
> **严重程度**：🔴 致命 — 导致3份报告重写，浪费数小时
> **涉及命主**：主母「成」

## 错误经过

1. 我收到主母生辰（农历1987年6月25日午时=公历1987-07-20）
2. 用代码排盘时，用了错误的基准日 **1900-01-01=甲子日**
3. 计算出的日柱 = 庚申日（实际=庚午日）
4. 基于错误的日柱写了一整份报告（身强72分、双正官格、庚坐申禄等）
5. 用户指出6月25日=庚午日，我查万年历确认正确

## 根因分析

| 原因 | 说明 |
|:----|:------|
| **直接原因** | 误记"1900-01-01=甲子日"为常识，未经验证即使用 |
| **潜在原因** | 没有"锚点验证"的强制流程——排完日柱后没有用已知日期交叉验证 |
| **系统原因** | 技能文件中日柱计算的基准标注为"甲子日"，没有人纠正过 |

## 教训

1. **1900-01-01 = 甲戌日**（不是甲子日！），这是经过5个锚点验证的事实
2. **每次排盘必须用代码计算日柱**，禁用任何人脑估算
3. **每次排完日柱后，必须用至少一个已知锚点验证结果**
4. 已用5个锚点验证基准正确性（见 calendar-anchor-points.md）
