## 一、触发条件

当用户要求"重算""V6""完整分析""全部维度"等关键词时，启动V6流程。

## 二、流程概览

```
用户请求（例：重算XX的八字V6）
    ↓
① 加载SKILL.md + 相关references
    ↓
② 获取/确认命主基本数据（八字、已知事件、职业、家庭）
    ↓
③ 读取之前版本报告（如有）作为基线
    ↓
④ 用delegate_task启动并行子代理（多人同时分析）
       ├─ 每人一个独立任务
       ├─ 每个任务包含：完整八字数据 + 已知事件 + 报告模板
       └─ 每个任务自行写入独立文件
    ↓
⑤ 验证各报告内容质量
    ↓
⑥ 推送到魏无记知识库
```

## 三、V6报告八大结构

```
第一部分：命格命理总论
  ├─ 八字排盘（四柱/藏干/十神/纳音/十二长生）

  ├─ 用神层级论（酉>申>辰丑>未戌）
  └─ 命格评分（含六大平衡评估）

第二部分：学业与事业
  ├─ 分年龄段分析（每步大运）
  ├─ 已知事件八字印证
  └─ 当前职位的格局解读

第三部分：行业与财富等级（V6新增核心）
  ├─ 五级财富体系判定（巨富/大富/中富/小康/贫穷）
  ├─ 行业五行匹配分析（完整五行行业表）
  ├─ 合作模式建议（身强用财官食/身弱用印比）
  ├─ 发财时机表（大运流年逐段）
  ├─ 正财vs偏财分析
  └─ 用神层级验证

第四部分：健康分析
  ├─ 五行对应身体
  ├─ 重点关注部位
  └─ 各运健康预警

第五部分：子女与家庭
  ├─ 子女星/子女宫分析
  ├─ 缘薄因素排查
  ├─ 婚姻幸福度评分
  └─ 家庭总体评价

第六部分：盲断三件已发生未提及事件（核心）
  ├─ 断事铁律执行（反推时间线→基线→信号分级）
  ├─ 每件事标注：年份+事件描述+信号强度(1-10级)
  ├─ 信号强度≥6才描述重大事件
  └─ 无强烈信号则按正常轨迹描述

第七部分：逐流年分析（2026-2035年）
  每一年必须包含：
  ├─ 📋 总览表：年份/年龄/全年基调(✅⚠️❌)+天干地支分析+大运互动
  ├─ 天干地支的十神转化（对大运的影响）
  ├─ 新知识与大运的互动（财富等级/合作模式/行业适配）
  ├─ 📅 关键月份分析（2-4个重点月份）
  ├─ 📊 五维度评分：事业/财富/健康/家庭/子女（★1-5）
  └─ 💡 策略建议（年度核心建议）

第八部分：一生总结
  ├─ 命主一生口诀（四句诗）
  ├─ 关键转折点时间线
  ├─ 综合评分表
  ├─ 用神速查表
  └─ 综合寄语
```

## 四、并行子代理技术要点

```python
# 用于delegate_task的prompt结构
{
    "context": "完整命主数据 + 已知事件 + 体系规则 + 报告模板",
    "goal": "生成XXX的V6完整八字命理分析报告，写入文件 /tmp/xxx_v6_report.md",
    "toolsets": ["terminal", "file"]  # 仅需写入文件和简单命令
}
```

**关键要点**：
1. 每个子代理独立运行，不共享状态
2. 每个子代理需要完整的context（包含所有规则和已知信息）
3. 子代理只需要write_file和terminal工具即可
4. 完成后检查写入文件的行数验证质量

## 五、V6相比V5的核心升级

| 维度 | V5 | V6 |
|:---|:---|:---|
| 财富分析 | 粗略身弱财旺论 | 五级财富体系+合作模式+行业选择 |
| 盲断事件 | 仅验证已知事件 | 盲断3件未提及事件+信号强度分级 |
| 逐流年 | 每一年简单分析 | 每年8维度（基调/干支/大运/月份/五维评分/策略） |
| 知识体系 | 金鉴实战Day1+Day2 | 新增第四课财运+进阶09断财 |
| 报告格式 | V5固定模板 | V68大结构+盲断+财富等级+合作模式 |

## 六、常见陷阱与对策

| 陷阱 | 对策 |
|:---|:---|
| 子代理context太大超过token限制 | 精简为仅关键数据+report模板，去除口水话 |
| 子代理重复计算身强弱 | 在context中直接给出已确认的身强弱结果 |
| 盲断事件凭空硬凑 | 强制执行断事铁律三步法，信号<6级不描述 |
| 推知识库时路径错误 | 确认知识库绝对路径 /tmp/weiwuji-knowledge-base/ |
| 品牌名称写错 | 正确名称：**魏无记**（不是唯无极） |
