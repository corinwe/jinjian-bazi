# 人物档案目录清理规范

> **编制人：** 金鉴真人
> **编制时间：** 2026年06月17日
> **触发案例：** 主母成目录从20个文件清理至1个正确版本

---

## 为什么需要目录清理

一个家族成员的人物目录可能随时间累积多个版本文件：
- 不同时辰的备选版本（巳时/午时/戌时等）
- 不同校准阶段的历史版本
- 身强vs身弱的平行试验版本
- 不同命名规范的文件（金鉴真人_开头的旧版 vs 名字_开头的新版）

若不清理，下次分析时可能：
- ❌ 更新了错误版本（如更新了v9.0庚申日版而非v11.0庚午日版）
- ❌ 从旧版复制过时的生辰数据
- ❌ 混淆多个并行版本

---

## 清理触发条件

任一条件满足即执行清理：

```yaml
□ 人物目录有≥5个文件（版本过多）
□ 确认了最终八字（不再需要备选时辰版本）
□ 老板指示「固定下来」
□ 人物有多个版本且最新版已被老板确认
```

## 清理标准流程（6步）

### Step 1 — 确认最终正确版本

```yaml
□ 列出目录下所有文件：ls -la *.md
□ 按时间排序：最新的=最可能的正确版？未必！
   （可能最新的是错误版，如v10.0庚申日版被错误更新）
□ 确认方法：
   ① 读取前5行确认八字、日主、出生时间
   ② 与记忆/家族总表中的正确八字对比
   ③ 运行的引擎排盘验证
□ 最终正确版 = 与老板确认一致的版本
```

### Step 2 — 清理文件：仅保留正确版本

```yaml
# 创建归档目录
mkdir -p ../归档_YYYYMMDD/{人物名}_旧版/

# 移动所有错误版本到归档
for f in *.md; do
  if [ "$f" != "正确版本文件.md" ]; then
    mv "$f" "../归档_YYYYMMDD/{人物名}_旧版/$f"
  fi
done
```

### Step 3 — 确认目录仅剩正确版

```yaml
ls -la *.md
# 应该只有1个文件
# 读取文件头确认八字正确
```

### Step 4 — 更新文件版本号

确认文件内版本号与文件名一致：
- 如文件名是 v11.0，文件头部也要是 v11.0
- 标题中的描述与最终确认一致（如"最终确认版"）
- 清理 §1 中的「备选」「未确认」等临时标注

### Step 5 — 更新Memory

```yaml
将人物正确八字写入memory：
  名称：{人物名}最终确认
  内容：农历X年X月X日X时=公历YYYY-MM-DD HH:MM→XX柱(从弱/身强X分)
  标注：XXX版本已废弃归档
```

### Step 6 — 推库

```bash
cd /root/weiwuji-knowledge-base
git add -A
git commit -m "0617 {人物名}目录清理: 仅保留vX.Y正确版"
git pull --rebase
git push
```

## 常见问题

### Q: 旧版归档后需要删除吗？
A: 不需要。归档保留旧版便于历史追溯，但不应出现在主目录中。

### Q: 如果老板后来又要求看旧版怎么办？
A: 归档目录中完整保留，可直接查阅。

### Q: 归档后的文件名需要改名吗？
A: 建议不改名，直接按原名移入归档即可。如有多个冲突版本，可在文件名末尾加「已废弃」标注。
