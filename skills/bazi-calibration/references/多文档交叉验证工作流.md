# 多文档 vs 现有技能 交叉验证流程

> 创建于：2026-06-05（吸收7份九龙课程文档时首次使用）

## 🚨 致命数据准确性陷阱（本场会话踩坑）2026-06-05

> **踩坑**：家主父亲出生时间，用户已明确说过「农历1949年八月初九午时」，但我从历史session中提取了错误版本（闰七月十二/公历9月4日），导致用户纠正。
> **根因**：历史session中混入了未经验证的推算结果，未做交叉验证就直接使用。
> **教训**：对于人物生辰数据，每次使用前必须主动向用户复述确认。

```yaml
【强制流程】使用任何人物八字前：
  □ 步骤1：主动向用户复述：「确认一下XXX的八字是XXXX，对吗？」
  □ 步骤2：用户回复"对"→使用；用户纠正→以用户最新说法为准，更新记忆
  □ 步骤3：不能从session历史中"拼凑"数据，必须以用户最后一次明确陈述为准
  □ 步骤4：复述时一定要写农历/公历日期，方便用户核对记忆中的生日
```

## ⚠️ 任务规模警告（本场会话踩坑）2026-06-05

> **踩坑**：尝试用 `delegate_task` 并行生成4份完整八字分析报告（每份200-400行），4个子代理全部因超时中断。
> **教训**：全生命周期八字分析的输出体量远超子代理合理负载。大任务必须分拆。

```yaml
【强制规则】委托子代理输出八字分析报告时：
  □ 单人单次：最多委托1人/次（不要4人并行）
  □ 优先单次输出：在父agent中逐个执行，而非并行委托4人
  □ 如必须并行：最多并发≤2，且每个任务明确限制输出行数（≤300行）
  □ 对于"重断"类任务（需应用全部新知识的深度分析），推荐逐个输出
```

## 🔧 单行大文件处理技巧（本场会话实用）2026-06-05

AI语音转写的课程文档常为**单行长文本**（几十KB~上百KB一行），`read_file`按行读取会失效。

```python
# 方案A（推荐）：用patch在时间戳前插换行符
# 找出"说话人 1"这类标记，用patch的replace_all模式插入换行
# 操作：找"说话人 1" → 替换为"\n说话人 1"（开启replace_all）
# 之后文件变为多行，read_file可正常分页读取

# 方案B（terminal python）：
python3 -c "
with open('file.txt', 'r', encoding='utf-8') as f:
    content = f.read()
# 用句号或时间戳分割
lines = content.replace('。', '。\n')
with open('file_split.txt', 'w') as f:
    f.write(lines)
"
```

## 适用场景

收到多份新文档（如课程录音、论文、技术文档）需要与现有技能库做全量吸收时。

## 六步流程

### 第1步：全量提取（并行化）

对每份文档提取纯文本：
- docx → zipfile + xml.etree.ElementTree
- pdf（可解析）→ PyMuPDF (fitz)
- pdf（扫描件）→ ocr-and-documents skill

提取后保存为 .txt，标记主题+字数。

效率技巧：使用 `delegate_task` 并行处理，最多并发4个子代理。

### 第2步：摸清现有技能体系

在开始对比前，加载所有相关领域的 SKILL.md：
```yaml
# 八字命理领域
bazi-fortune-analysis  → 命运/财富/婚姻/子女
bazi-liunian-analysis  → 流年/大运/节气
bazi-health-psychology → 健康/心理
bazi-calibration       → 校准/报告/规范
bazi-auto-verify       → 自动验证
bazi-report-template   → 模板
bazi-destiny-analysis  → 深度分析（与fortune重叠）
bazi-master-agent      → 路由
```
目的：全面了解已有知识，避免重复。

### 第3步：逐文档精读 + 知识点提取

对每份文档完整读取（注意单行大文件问题），逐段提取：
- 命理口诀
- 方法论流程
- 量化规则
- 案例中隐藏的方法论
- 与已有知识不同的说法

输出格式：每个知识点一行「【知识点】原文摘要 + 提炼」

### 第4步：系统对比 + 三分类标记

将新知识点与已有技能逐项对比，标记为三类：

| 标记 | 含义 | 处理 |
|------|------|------|
| ✅ **已有** | 现有技能已覆盖 | 跳过（验证一致性）|
| ❌ **新知识** | 现有技能未覆盖 | 评估P0/P1/P2后补充 |
| ⚠️ **冲突** | 说法矛盾 | 记录差异原因，保留两种视角 |

### 第5步：防遗漏扫描

对每份新文档，逐项检查以下常见遗漏点：

```
□ 是否有能量倍数/量化规则？
□ 是否有墓库/入库/入墓规则？
□ 是否有十神×事件对应表？
□ 是否有穿害/刑冲特殊应事？
□ 是否有从格/特殊格局规则？
□ 是否有生多为克/分担原理？
□ 是否有实战案例展示的方法论？
□ 是否有变性与生克反转规则？
□ 是否有空亡的特殊影响？
□ 是否有36岁分界/节气应事？
□ 是否有「与日元喜忌无关」的独立规则？
```

### 第6步：优先级标记 + 更新技能

```
P0（必须立即补）：能量倍数错误/墓库规则错误/矛盾冲突
P1（尽快补）：全新的断事规则/方法论细节
P2（记录即可）：边角案例细节/特殊现象
```

更新顺序：
1. 补对应子skill的SKILL.md（用 `patch`）
2. 或新增 references/ 文件
3. 更新版本号和编制时间
4. 更新 master-agent 的索引墙
5. 更新 memory
6. 推知识库（git）

## 输出产物

1. 交叉验证报告（差距与新增点.markdown）
2. 更新的 SKILL.md 文件
3. 新的 references/ 文件
4. 知识库存档文档
