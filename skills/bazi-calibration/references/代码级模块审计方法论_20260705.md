# 代码级模块审计方法论 v1.0

> **来源**: 2026-07-05 cai_xing.py 深度审计实战
> **前置技能**: `bazi-wealth-analysis`（或对应域的 skill）
> **与报告级审计的区别**:
> - 报告级审计 → 检查输出数字是否合理、章节是否一致、事件是否自洽
> - **代码级审计** → 检查引擎代码是否**正确实现了 skill 中的规则**

```yaml
第0步: 加载对应 skill

第2步: 理解依赖
  从 import 找到 constants / 数据类型 / 调用链路

```

## 六步审计检查清单（通用版）

对任何评分/计算模块，按以下顺序逐项验证:

### ① 基础分检查
确认每个位置的基础分与 skill 定义一致:
- 年干/月干/年支/月令/日支/时干/时支 各是多少？
- 代码中的数值与 skill 表完全一致？

### ② 藏干百分比检查
确认 ratio 映射正确:
- 本气 100% → 1.0
- 中气 60%  → 0.6
- 余气 30%  → 0.3
- 代码引用的是 `CANG_GAN_RATIO` 还是硬编码了错误值？

### ③ 五行/十神定向检查
确认判断逻辑与 skill 一致:
- "我克者为财" — 代码用的是什么公式？
- 检查: `WU_XING_KE[ri_wx] == cg_wx` 是否正确？（日主所克五行 == 目标五行）
- 是否有将劫财/比肩/官杀 误判为 财星/食神/印 的 bug？
- 逐日主五行验证（金→木、木→土、水→火、火→金、土→水）

### ④ 特殊位置权重检查
- 月令藏干是否用了 **40分** 基础分？（不是12分！）
- 年支基础分是4分不是8分？（注意不要和年干混淆）

### ⑤ 特殊规则处理检查
skill 中是否有特殊格局规则未被代码覆盖？
- 从弱格特殊处理（§3）: 身弱0→50分、财星得令加40分
- 从强格特殊处理
- 假从弱/从旺处理
- 辰戌丑未变性规则

### ⑥ 数据集完整性检查
- 所有计算的中间值是否存入 dataclass？（不仅仅是汇总到 total）
- `cai_xing_explain` / 输出函数 是否展示了所有分项？
- 下游 pipeline 输出（`pipeline_v5.py`）是否映射了所有字段？
- 总分是否 = 所有分项之和？（算术一致性铁律）

## 实战发现总结（来自2026-07-05 cai_xing.py审计）

### BUG 模式1: 局部变量不入库
年干(8分)和月干(12分)被算入总分，但 `CaiXingDetail` 没有对应字段
→ 分项之和(0+0+...) ≠ 总分(20.0) → 致命算术矛盾
**修复**: 先在 dataclass 加字段 → 再在 compute 函数赋值 → 再在 explain/pipeline 输出

### BUG 模式2: Explain/输出未覆盖全分项
`cai_xing_explain` 只输出5行，但 total 包含7项
→ 下游开发者和前端看不到完整明细
**修复**: explain 和 pipeline 输出必须全量映射，与 dataclass 字段一一对应

### BUG 模式3: 特殊格局规则未实现
skill 中明确写了从弱格规则，但 compute 函数签名无 `sqr_label` 参数
→ 从弱格时财星分数不会自动调整
**修复**: 方案A（推荐）— 在 compute 函数添加 sqr_label 参数，内部判断
        方案B — 在 pipeline 调用处后处理
