# 服务器网络配置参考

本文档记录了在部署小程序时，服务器端需要检查和配置的关键网络信息。

## 服务器公网IP

- **IP地址**: `43.162.90.39`
- **用途**: 小程序前端配置的 `apiBaseUrl` 必须指向此地址，以便用户手机能访问到部署在服务器上的API服务。
- **验证命令**:
  ```bash
  # 查看公网IP
  curl ifconfig.me
  # 或
  ip addr show eth0 | grep inet
  ```

## API服务器端口监听配置

### 关键检查点
部署后若出现“加载失败”，必须按顺序检查：
1.  **进程是否存在**: `ps aux | grep api-server`
2.  **监听地址是否正确**: `netstat -tlnp | grep :3000`
    - **错误状态**: 显示 `127.0.0.1:3000` 或 `localhost:3000`，表示仅本地可访问。
    - **正确状态**: 显示 `0.0.0.0:3000` 或 `:::3000`，表示监听所有网络接口，允许公网访问。
3.  **本地连通性**: `curl http://localhost:3000/api/v1/health`
4.  **公网连通性**: `curl http://43.162.90.39:3000/api/v1/health` （从服务器自身或另一台机器测试）

### 配置修正
如果监听地址错误，需修改API服务器启动代码（如 `api-server.js`）：
```javascript
// 错误：仅本地访问
// app.listen(3000);

// 正确：允许公网访问
app.listen(3000, '0.0.0.0', () => {
  console.log('API服务器运行在: http://0.0.0.0:3000');
});
```

## 云服务商安全组配置

这是导致“网络请求失败”的最常见原因。即使服务器配置正确，云服务器的虚拟防火墙（安全组）可能默认阻止非标准端口。

### 必须添加的入站规则
- **协议**: TCP
- **端口范围**: `3000` （或你的API服务端口）
- **来源**: `0.0.0.0/0` （或根据需求限制IP段）
- **策略**: 允许

### 配置位置（示例）
- **腾讯云**: 云服务器控制台 -> 实例 -> 安全组 -> 编辑规则
- **阿里云**: ECS控制台 -> 实例 -> 安全组配置 -> 配置规则

## 小程序前端配置对应修改

服务器IP和端口配置好后，小程序前端必须同步修改：

```javascript
// 在 app.js 或全局配置文件中
App({
  globalData: {
    // 开发环境（仅本地）
    // apiBaseUrl: 'http://localhost:3000/api/v1',
    
    // 生产环境（指向服务器公网IP）
    apiBaseUrl: 'http://43.162.90.39:3000/api/v1',
  }
})
```

## 快速诊断脚本

在服务器上创建诊断脚本 `check-network.sh`：

```bash
#!/bin/bash

IP="43.162.90.39"
PORT="3000"

echo "1. 检查API进程..."
ps aux | grep api-server | grep -v grep

echo "\n2. 检查端口监听..."
netstat -tlnp | grep :$PORT

echo "\n3. 测试本地连接..."
curl -s -o /dev/null -w "%{http_code}" http://localhost:$PORT/api/v1/health && echo " 本地连接成功" || echo " 本地连接失败"

echo "\n4. 测试公网连接..."
curl -s -o /dev/null -w "%{http_code}" http://$IP:$PORT/api/v1/health && echo " 公网连接成功" || echo " 公网连接失败"

echo "\n5. 检查安全组提醒..."
echo "请登录云控制台，确认安全组已开放TCP端口 $PORT"
```

## 总结排查流程

1.  **内通外不通** -> 检查API服务器监听地址（应为 `0.0.0.0`）。
2.  **内外都不通** -> 检查API进程是否运行。
3.  **配置正确仍不通** -> **99%是云服务商安全组未配置**，必须登录控制台添加规则。
4.  **小程序仍报错** -> 检查小程序 `apiBaseUrl` 配置是否已更新为公网IP。
