# 新知识学习物理协议（金鉴真人·2026-07-01固化）

> **触发条件：** 老板丢给我任何新知识——文档、音频、视频、链接、图片、截图
> **物理铁律：** 以下步骤一条不能少，不能跳过，不能简化

---

## 🚨 物理铁律：逐字逐句精读（v1.4升级）

> **来源：** 2026-07-04 老板纠正「逐字逐句精读啊」— 无论是自己读还是派子Agent读，都必须逐字逐句，扫读=没读。

### 适用场景全覆盖

```yaml
本铁律覆盖所有读取内容的场景：
  □ 亲自 read_file — 必须读完所有行
  □ 派 Sub-Agent（delegate_task）— goal中必须明确写
  □ vision_analyze 看图 — 逐字读图中文字
```

### 铁律一：亲自读

```
□ 用 read_file 分多次（offset+limit）确保全覆盖
□ 每次 limit=500，分N批读到文件末尾
□ 完成后自检：确认读取范围覆盖了文件总行数
□ 遇到内容密集的段落，减小 limit 值（如200行一次）
```

### 铁律二：派 Sub-Agent 读

```
□ goal 中必须明确写「逐字逐句精读」「读完全部N行」
□ 原文行数必须直接写在goal中（如「共7510行，分15次」）
□ 必须写「不要跳」「不要跳过任何行」
□ 返回后验证报告深度（行数/章节数与原文匹配）

✅ 正确写法：
  goal="【逐字逐句精读】音频X（228k字）\n必须：\n✅ 读完全部7510行！不要跳！"

❌ 错误写法（子Agent会扫读）：
  goal="提取音频X的知识点" — 子Agent只看首尾，跳过中间大量行
  goal="先读取前100行确定主题" — 这是扫读，不是精读
```

> **关联技能：** 子Agent派发模板见 `skill_view('bazi-task-dispatch')` 陷阱⑨

### 铁律三：验证方法

```
□ 报告行数是否与原文合理匹配？
□ 原文关键段落是否在报告中都有对应？
□ 是否有明显被跳过的行数范围？
```

学习 ≠ 看过

### 第0步：发现新知识 → 触发门禁

当老板说「你看这个」「你学这个」「你读这个」，或者发送任何形式的内容时：

```
□ 自动识别知识类型：文档/音频/视频/链接/图片/截图
□ 自动加载本学习协议
□ 自动进入第1步
```

**类型处理规则：**

| 类型 | 处理方式 |
|:----|:---------|
| **图片/截图** | 用 vision_analyze 逐字读取图片中的文字，不可扫一眼就过 |
| **文档/链接** | 用 read_file/web_extract 获取完整内容，不可只看摘要 |
| **音频** | 用 audio-transcribe 技能完整转写，不可快进/跳听 |
| **视频** | 逐帧精读关键截图，不可只看进度条 |

## 第1步：逐字逐句精读（强制·不能跳过）

```
规则：每个字、每个标点、每个表格、每张图都必须看
禁止：扫读、跳读、只读标题
```

**执行方法：**
```
□ 逐行读完整内容
□ 标记不理解的地方（不能跳过，必须问）
□ 记录每条规则的原文依据（行号/时间戳）
□ 识别核心概念（这条知识在讲什么）
□ 识别规则边界（什么情况下适用/不适用）
□ 识别与现有知识的冲突（有冲突必须问，不能自己猜）
```

### §1.1 领域核查（v1.1新增）

**铁律：标题/文件名/用户描述 ≠ 实际内容。** 必须快速验证后再深读。

**典型陷阱：** 标题说「八截风水」实际讲「洗髓功」；系列课名为A，本节课讲B；赠送课/彩蛋内容独立于主课。

```yaml
【核查流程】
① 快速读前100行 → 自问「这真的是在讲标题/文件名所指的内容吗？」
② 如果发现偏离 → 在报告中标注「标题vs实际」对照表
③ 知识分析必须基于实际内容，而不是标题
④ 文件名/报告名可保留原标题（便于追踪溯源）

【对比现有体系时的处理】
  如果实际内容与现有体系完全无关（不同领域）：
  ✅ 标注为【新发现 100%】
  ✅ 在报告中明确说明「无重叠/可平行共存」
  ❌ 不要强行套入现有体系分类
  ❌ 不要硬找不存在的关系
  ❌ 不要试图把养生功法归类到八字命理

【口诀】
  标题只是引路人，百行读完知真身
  分析只听实际话，不从标题论乾坤
```

## 第2步：用自己的话复述（检验是否真懂）

**规则：不能用原文的表述，必须用自己的语言重新组织**

```
□ 这条知识的核心是什么？（一句话说清楚）
□ 它和已有知识的关系是什么？（是补充？是修正？是新领域？）
□ 它背后的底层逻辑是什么？（为什么是这样，不是那样）
```

**合格标准：** 能用大白话讲给不懂的人听

---

## 第3步：举一反三（检验是否能应用）

**规则：用这条知识至少举3个例子，验证理解**

```
示例框架：
  示例1 — 简单应用（直接套用规则）
  示例2 — 边界情况（规则在什么情况下不适用）
  示例3 — 逆向思考（反过来会怎样）
```

**合格标准：** 能回答「如果XX情况，用这条规则会得出什么结论？」

---

## 第4步：用Checker验证（独立对抗性审查）

**规则：派独立的Checker Agent审查第1-3步的产出**

```python
delegate_task(
    goal="Checker验证：我学到的{知识主题}是否正确理解",
    context=f"""
    原始知识: {原文}
    我的理解: {复述}
    我的例子: {举一反三}
    
    检查清单：
    [ ] 复述是否准确反应了原文意思？（不是自己臆想的）
    [ ] 是否有遗漏的重要内容？
    [ ] 是否有过度解读（说了原文没说的）？
    [ ] 举一反三的例子是否合理？
    [ ] 边界情况是否考虑完整？
    
    输出：PASS 或 REJECT
    如REJECT，列出具体问题
    """,
    toolsets=['terminal', 'file']
)
```

**循环规则：**
- Checker PASS → 进入第5步
- Checker REJECT → 回到第1步，修正理解后再来
- 最多循环3次，超过后自动上报老板

---

## 第5步：老板查缺补漏（最终把关）

**规则：主动向老板汇报学习成果，不可跳过**

```
汇报格式：
┌─────────────────────────────────────┐
│ 📖 学习汇报：{知识主题}               │
│                                       │
│ 原文核心：{一句话总结}                 │
│ 我的理解：{用自己的话复述}              │
│ 举一反三：{3个例子}                    │
│ Checker验证：PASS/REJECT              │
│                                       │
│ 请问老板我理解得对吗？还漏了什么？       │
└─────────────────────────────────────┘
```

**老板反馈后：**
- 老板说「对」→ 固化到知识库+技能库
- 老板纠正 → 修正理解 → 回到第4步重新验证
- 老板补充→ 合并补充内容 → 再次汇报

## 第6步：物理固化（知识入库）

**规则：学习通过后，必须物理存下来**

```
□ 推库到 knowledge-base（如果是新知识）
□ 更新 skill（如果是现有技能的补充）
□ 更新 memory（如果是经常用的规则/铁律）
□ 老板如果纠正过 → 把「纠正的过程」也存下来（防止再犯）
```

## §7 第三方内容入库评估（v1.0 2026-07-04新增）

> **触发条件：** 拿到非权威/第三方来源的内容（其他讲师课程、网络文章、公众号、学生笔记等），需要判断"要不要入库"
> **核心区别：** 本协议的§1-§6 假设内容是权威且值得学的；§7 处理的是"不确定值不值得入库"的内容。

### §7.1 完整评估流程

```yaml
第0步 — 加载本协议
第1步 — 完整精读（复用§1-§2的逐字精读和复述）
第2步 — 与现有知识库全量交叉比对
  对每个知识点标记三类：
  ✅ 已有（被现有体系完全覆盖）
  📌 已有但表述不同（核心一致，表达方式/例子不同）
  🆕 新发现（现有体系缺失的规则/方法论/视角）

  操作手法：
  □ 加载对标技能（如 bazi-foundation-analysis）
  □ 逐知识点对比，不跳步
  □ 用表格记录每个知识的「状态」「来源」「与已有体系的差异」

第3步 — 入库价值量化判断
  【指标1】重叠率 = (✅ 数量) ÷ 总知识点数
  【指标2】新发现率 = (🆕 数量) ÷ 总知识点数
  【指标3】内容纯度 = 有效知识点字数 ÷ 总字数（排除互动/推广/闲聊）
  【指标4】权威性评分：
    高 🟢 — 经典原文/权威机构/经实战验证
    中 🟡 — 从业者经验总结/有实战案例
    低 🔴 — 个人观点/推广性质/无验证

第4步 — 领域门禁 → 价值判断矩阵

**⚠️ 必须先过领域门禁，再进矩阵：**

```yaml
【领域门禁规则——在跑矩阵前先判断】

if 内容领域 == 现有知识体系领域:
  → 进入下方「价值判断矩阵」正常计算
  → 重叠率/新发现率基于同一领域计算

elif 内容领域 != 现有知识体系领域（如八宅风水 vs 八字命理）:
  → 【不进入矩阵】
  → 直接使用 §1.1 跨领域规则：
    ✔ 标注为【新发现 100%】但说明属不同领域
    ✔ 在报告中明确说明「体系A vs 体系B，无重叠/可平行共存」
    ✔ 入库建议固定为：选择性摘取（只摘有参考价值的部分）
    ✔ 无需计算重叠率/新发现率百分比（百分比在此场景无意义）
  → 输出文档的⑫入库价值判断中注明：「跨领域内容，不适用矩阵」
```

**价值判断矩阵（仅适用于同领域内比较）：**

  | 新发现率 | 重叠率 | 建议 |
  |:--------:|:------:|:-----|
  | ≥30%     | ≤60%   | 全文入库（有价值新内容多） |
  | 10-29%   | 60-85% | 选择性摘取（标记🆕部分入库） |
  | <10%     | >85%   | 不入库，或作为参考/教学案例存档 |
  | 🔴权威性 | 任意   | 入库时标注「第三方·权威性待验证」 |
  | 跨领域   | 跨领域 | 不进入矩阵，见上方领域门禁规则 |

第5步 — 输出结构化评估文档
  推荐输出结构（参考本次实践的13章节格式）：
  ┌──────────────────────────────────────┐
  │ ① 讲师背景与课程架构                  │
  │ ② 核心哲学框架                       │
  │ ③-⑩ 按知识点分类的详细提取            │
  │ ⑪ 与现有体系的交叉对比（重要！）       │
  │ ⑫ 入库价值判断（含量化指标+建议）     │
  │ ⑬ 提取总结（一句话结论+标签）          │
  └──────────────────────────────────────┘

第6步 — 存储决策
  ✅ 全文入库 → 放入 `00-原始素材/` 并标注来源+评估结论
  ✅ 选择性摘取 → 只保存摘取的参考片段到 `references/` 或打上「第三方参考」
  ❌ 不入库 → 只在memory中记录结论，不占用知识库空间
  ⚠️ 非权威来源的知识，必须标注来源类型：
    「讲师：XXX」「来源：网络课程」「权威性：中等」
```

### §7.2 第三方内容的典型特征

| 特征 | 对入库的影响 |
|:----|:------------|
| **教学法创新（颜色编码/评分简化等）** | 有价值参考，但不改变理论知识本身 |
| **与权威源高度重叠（90%+）** | 不入库或只存摘要，避免知识库膨胀 |
| **商业化推广/互动抽奖** | 排除（不计入有效知识点） |
| **个人经历/感悟** | 不入库（非普适性知识） |
| **新奇/有趣的视角（如三起三落/少年运晚年运）** | 选择性摘取，标注来源 |

### §7.3 实战示例

> **案例A（同领域，高重叠）：** 刘宇敦老师周易讲课（84k字音频转写）
> **评估结论：**
> - 有效知识点占比 ~40%（其余为互动/推广/重复）
> - 与金鉴八字体系重叠率 ~90%
> - 新增有效知识点：十二长生在人生阶段应用（少年运/中年运/晚年运概念）
> - **入库建议：中等偏低 — 选择性摘取，不建议全文入库**
> - 输出文件：`2026-07-04_刘宇敦老师_周易讲课_知识点提取.md`

> **案例B（跨领域，不同体系）：** 九龙道长·八宅环境能量学高级篇（135k字粤普混合转写）
> **领域判定：** 八宅风水 ≠ 八字命理（不同术数分支）
> **评估结论：**
> - 有效知识点占比 ~60%
> - 与金鉴八字体系：领域不同，不适用重叠率计算
> - 处理方式：走领域门禁 → 跳过矩阵 → 标注【新发现 100%属八宅领域】
> - **入库建议：选择性摘取 — 标注「第三方·权威性中等」，只摘实操方案**
> - 输出文件：`2026-07-04_粤语混合课程_知识点提取.md`

### §7.4 错误防范

**常见陷阱：**
1. ❌ 把教学法创新当成理论知识入库 → 颜色编码、100分制是教法，不是传统理论
2. ❌ 高估新发现率 → "好像没见过" ≠ "体系里没有"，先仔细搜再下结论
3. ❌ 忽视来源权威性 → 第三方讲师的案例数据不可用于校准已有体系
4. ❌ 强行套入现有体系 → 如果内容属不同领域（养生功法≠八字命理），标注为【新发现 100%】并行共存

**口诀：**
  第三方内容别着急，交叉比对先做起
  教学创新非理论，新发现要仔细辨
  重叠太高不入库，选择性摘取不浪费
  权威标注不能少，输出文档结构清
