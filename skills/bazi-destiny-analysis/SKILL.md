# 八字命理深度分析 Skill

## 核心能力

1. **八字排盘与格局判定**：严格按月令透干规则判定正八格/杂气格/特殊格局
2. **日主强弱分析**：金鉴真人原始评分规则（月令印40分/天干+地支印比计分）
3. **五行十神分析**：藏干十神逐字推导，确认正偏阴阳无误
4. **大运流年推演**：推算大运并标注关键流年
5. **事象深度分析**：财富/事业/婚姻/子女/健康/灾祸逐项基于引擎数据展开

## 🚨 格局判定规则（每次写§2前必须过此规则·完整体系·2026-07-07重构）

> **铁律**：先通过bazi-paipan-sop Phase 4.2的7步检查清单，再写§2。
> **完整规则**：详见 bazi-fortune-analysis §6（格局判定·完整体系）

### 主格判定（月令透干法）

```
月支本气是否透干？（对比年/月/日/时四柱天干）
  ├─ 是 → 以本气十神定为正八格 ✅
  └─ 否 → 月支中气是否透干？
      ├─ 是 → 以中气十神定为正八格 ✅
      └─ 否 → 月支余气是否透干？
          ├─ 是 → 以余气十神定为正八格 ✅
          └─ **全部不透 → 杂气格** ⚠️
```

### 成格条件校验（新增·必须执行）

定主格后，检查该格局的成格条件：
- 身强弱条件是否满足？（参见§6.9格局VS身强弱表）
- 十神位置和比例是否适当？
- 有无刑冲合害破坏格局？
- 不满足条件时注明「格局条件未完全满足」

### 辅格判定

```
辅格一：日支本气十神 = 内在性格/脾气
辅格二：月干十神 = 外在表现/社会角色
辅格三：暗合关系 = 隐藏性格
```

### 实例对照

|:----|:----|:------|:---------|:-----|:-----|:---------|
| 庚申 癸未 辛亥 辛卯 | 未 | 己(偏印)/丁(七杀)/乙(偏财) | 己丁乙全不透 | **杂气格** | 食神透干·自坐伤官 | 杂气格·食神透干·自坐伤官 |
| 丁卯 丁未 庚午 壬午 | 未 | 己(正印)/丁(正官)/乙(正财) | 丁火透月干 | **正官格** | 杀印相生成格条件未全满足 | 正官格·自坐正官 |

### 常见错误

```yaml
❌ 错误1：把月干的十神当格局名
   例：月干癸水(食神) → 叫"食神配印格"
   ✅ 正确：必须看月令藏干是否透出，月干≠月令透出

❌ 错误2：月令不透时硬安一个正八格名
   例：未月己不透 → 叫"偏印格"
   ✅ 正确：全部不透就是杂气格

❌ 错误3：不检查成格条件直接叫特殊格局名
   例：有七杀+正印但身强 → 叫"杀印相生格"
   ✅ 正确：杀印相生必须身弱（条件①）

❌ 错误4：混用食神/伤官
   例：癸水对辛金是食神（阴生阴），不是伤官
   ✅ 正确：逐字推导十神，不凭记忆
```

## 📋 标准分析流程（配合bazi-paipan-sop执行）

### Phase 0：数据准备
1. 加载技能：skill_view('bazi-engine-workflow')（引擎规则）
2. 加载技能：skill_view('bazi-report-template')（模板规则）
3. 加载技能：skill_view('bazi-calibration')（校准规则）
4. 确认引擎数据就绪（/tmp/{姓名}_engine.json）

### Phase 1：格局判定（严格按照上方规则）
1. 查月令地支 → 确认藏干
2. 查四柱天干 → 确认哪些藏干透出
3. 透干 → 正八格名；不透 → 杂气格
4. 引擎ge_ju字段仅作参考，以手动验证为准

### Phase 2：身强弱+喜用神
1. 从引擎result提取sec_3_shen_qiang_ruo
2. 确认喜用/忌神与身强弱逻辑一致
3. 身强喜克泄耗（财官食），身弱喜生扶（印比）

### Phase 3：大运分析（8步至约80岁）
1. 从引擎提取sec_17_da_yun_detail.list
2. 只取前8步（至约80岁），去掉81岁后的
3. 标注最佳/最差/现行大运

### Phase 4：事象分析（按模板§§展开）
1. 财富：财星评分+补财库方案（充库/蓄库/开库）
2. 婚姻：夫妻宫+配偶星+特征+桃花窗口
3. 子女：子女星+子女宫+出息程度+性别倾向
4. 性格：五重人格+十神底色
5. 文昌：仅≤25岁未成年人输出改进方案

### Phase 5：报告生成
1. 按bazi-report-template精简版（800~1000行）
2. 禁止废话，每句话有引擎数据支撑
3. 大运只到80岁
