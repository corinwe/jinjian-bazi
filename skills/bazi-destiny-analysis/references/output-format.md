## 图表规范

### 必须包含的图表

| 图表名称 | 类型 | 用途 |
|---------|------|------|
| 五行力量分布图 | 柱状图/饼图 | 展示五行百分比 |
| 大运走势图 | 折线图 | 展示各阶段运势高低 |
| 十神格局图 | 表格/雷达图 | 展示十神分布 |
| 财富路径图 | 面积图/阶梯图 | 展示财富积累趋势 |

### 图表设计要求
- 使用matplotlib生成
- 配色方案：低饱和度，统一色调
- 字体：中文使用支持CJK的字体
- 分辨率：至少150 DPI
- 保存格式：PNG

### 五行配色标准
| 五行 | 推荐颜色 |
|-----|---------|
| 木 | #4CAF50（绿） |
| 火 | #F44336（红） |
| 土 | #FF9800（橙） |
| 金 | #FFC107（金） |
| 水 | #2196F3（蓝） |

## 校准标记规范

### 校准状态标记
在报告中使用以下标记区分校准状态：

| 标记 | 含义 | 使用场景 |
|-----|------|---------|
| 【初判】 | 未经校准的初始判断 | 第一轮分析 |
| 【验证】 | 已验证准确的预测 | 校准后确认 |
| 【修正】 | 校准后修正的内容 | 校准后更新 |
| 【待验】 | 尚未发生，待验证 | 未来预测 |

### 校准记录格式
每次校准后，在报告开头添加校准记录：

```
【校准记录】
第N次校准 | 日期：YYYY-MM-DD | 提供事件：N个
整体吻合度：X% | 格局调整：X处 | 用神调整：X处
```

## 多轮校准报告迭代

### 迭代规则
1. **第一轮**：基于命盘做初判，标记为【初判】
2. **第二轮**（用户提供事件后）：做事件验证，标记验证结果，修正偏差，标记为【验证】/【修正】
3. **第三轮**（用户提供更多信息）：进一步校准，更新预测
4. **第N轮**：持续迭代，每次更新校准记录

### 报告合并原则
- 每轮校准生成独立报告
- 最新报告包含所有历史校准记录
- 最终报告为"完全校准评估"，汇总所有轮次

### 报告命名规范
| 轮次 | 报告名称 |
|-----|---------|
| 第一轮 | {姓名}八字命理综合详析报告 |
| 第二轮 | {姓名}八字命理验证细化报告 |
| 第三轮 | {姓名}命格等级与事业财富预测 |
| 第四轮 | {姓名}事业财富校准与最佳发展路径 |
| 最终轮 | 完全校准评估{核心主题} |

## 输出格式选择

### Markdown格式（默认）
- 适用：快速输出、在线阅读
- 优点：格式简洁，易于转换
- 包含：完整文字+图表图片引用

### Word文档格式（docx）
- 适用：正式报告、打印交付
- 要求：使用C# + OpenXML SDK生成
- 结构：封面→目录→正文→封底
- 规格：A4纸张，标准页边距

### 生成Word文档的步骤
1. 使用matplotlib生成所有图表（PNG格式）
2. 使用Playwright + SVG生成封面/封底背景
3. 使用C# + OpenXML SDK构建文档：
   - 封面：浮动背景图+标题
   - 目录：自动目录域代码
   - 正文：标题+段落+表格+内嵌图表
   - 封底：浮动背景图+结语
4. 编译验证：使用`./scripts/docx build`命令

### 注意事项
- CJK文本处理：中文引号使用`\`转义
- 表格宽度：GridColumn与TableCellWidth必须匹配
- 图片路径：使用绝对路径
- 字体设置：必须设置EastAsia字体
