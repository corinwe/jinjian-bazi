# 金鉴真人·八字报告文件完整性恢复工作流

> 创建日期：2026-06-25
> 触发条件：报告文件中出现重复§、缺失§、§顺序混乱、多PIPELINE-SIG、文件结构崩溃
> 适用场景：多次ALLNEW追加生成导致的文件损坏

---

## 诊断信号清单

检查以下信号，有≥2个则走「完整恢复流程」：

- [ ] `grep -n "^## §"` 返回的§编号有重复
- [ ] 总§数量 > 21（含重复）
- [ ] §17只有<100行（被截断）
- [ ] PIPELINE-SIG > 1个
- [ ] 有「后略」「因篇幅限制」等截断标记
- [ ] §顺序跳跃（如§12后跟§14，中间缺§13）
- [ ] grep "编制人" 返回≥2行
- [ ] `wc -l` 比上一版本突然减少或异常增大

## 完整恢复流程（9步）

### Step 1 — 备份当前文件
```bash
cp {FILE} ${FILE}.bak.$(date +%H%M%S)
```

### Step 2 — 从git找回最近清洁版本
```bash
git log --oneline --follow "{FILE}" | head -5
# 选择ALLNEW或最长版本的commit
git show {COMMIT_HASH}:{FILE} > /tmp/original.md
wc -l /tmp/original.md
```

### Step 3 — 诊断：解析当前文件§结构
```bash
# 检查§覆盖
grep -n "^## §" {FILE} | awk -F: '{print $2}' | sort -t' ' -k2
# 检查重复
grep "^## §" {FILE} | sort | uniq -c | sort -rn
# 检查PIPELINE-SIG
grep -c "#PIPELINE-SIG" {FILE}
```

### Step 4 — 去重：保留每个§的首次出现
```bash
awk '/^## §[0-9]/ && seen[$0]++{next} !/^#PIPELINE-SIG/ || NR==1' \
  /tmp/original.md > {FILE}.dedup
# 验证
grep -n "^## §" {FILE}.dedup
```

### Step 5 — 检查缺失§和特殊问题
- 检查§17是否被截断（仅含17.1或<100行）
- 检查§18-§21是否存在
- 检查§12/§14/§15/§16是否有深度版内容在文件末尾未被合并

### Step 6 — 合并补充内容
对于有深度版的§（如§12[0]简版+§12[1]深度版）：
1. 用原始版本中的简版§做基础
2. 从深度版提取补充内容（去掉重复的## §标题行）
3. 用Python组装：sec12_base + '\n\n' + sec12_depth_content

### Step 7 — 重新生成被截断的§
§17被截断时，全新生成17.2-17.10（每运≥250字运象分析+关键年份）：
- 17.2 乙酉（11~20岁）：偏财+比肩 · 中学求学
- 17.3 丙戌（21~30岁）：正官+正印 · 职场黄金期
- 17.4 丁亥（31~40岁）：七杀+伤官 · 最苦最炼
- 17.5 戊子（41~50岁）：正印+食神 · 稳重上升
- 17.6 己丑（51~60岁）：偏印+偏印 · 巅峰收获
- 17.7 庚寅（61~70岁）：劫财+正财 · 知识变现
- 17.8 辛卯（71~80岁）：比肩+偏财 · 安享晚年
- 17.9 壬辰（81~90岁）：伤官+正印 · 思维活跃
- 17.10 癸巳（91~100岁）：食神+正官 · 长寿收官

### Step 8 — 组装最终文件（Python示例）
```python
header = lines[:N]
sec1_to_11 = '\n'.join(lines[16:754])
sec12 = base_sec12 + depth_sec12  # 去重标题行
# ... 同理每个§
final = header + sec1_to_11 + sec12 + ... + sig
write_file(path, final)
```

### Step 9 — 最终验证
```bash
echo "§覆盖: $(grep -c "^## §" {FILE})/21"
echo "重复: $(grep "^## §" {FILE} | sort | uniq -d | wc -l)"
echo "PIPELINE-SIG: $(grep -c "#PIPELINE-SIG" {FILE})"
echo "编制人: $(grep -c "^**编制人" {FILE})"       # 期望=1（仅文件头部）
echo "后略: $(grep -c "后略\\|因篇幅" {FILE})"
echo "品牌: $(grep -c "九龙道长" {FILE})"           # 期望=0
grep -n "^## §" {FILE}
```

**注意：** 编制人如果≥2，用patch删除文件中间出现的那几行。

## 快速修补模式（小问题专用）

只适用于局部问题（如某§内容不够但结构完整），跳过Step 2-4：

1. 用 `patch` 在对应§末尾插入补充内容
2. 用 `patch` 删除多余PIPELINE-SIG
3. 验证：`grep -c "#PIPELINE-SIG" {FILE}` 应为1
4. 验证：`grep "^## §" {FILE} | sort | uniq -d` 应返回空

---

## 关键陷阱（避免掉坑）

1. **awk去重陷阱**：重复§被删除后，§17可能只剩标题行（因为原版§17在重复区域有内容，首次出现只有标题）。去重后必须检查§17行数。
2. **Python index偏差**：lines数组的index比grep行号少1。grep行N = 数组index N-1。
3. **patch删除过量**：多行匹配时用最独特的上下文（如前3行锚定），不要依赖短字符串匹配。
4. **合并深度版时去掉§标题**：深度版内容以`## §N ...`开头，合并前必须去掉标题行，否则会产生重复§。
