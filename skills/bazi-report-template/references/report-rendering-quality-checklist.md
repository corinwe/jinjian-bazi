# 报告渲染质量检查清单（2026-06-27 · 老板实际PDF验证后新增）

> **触发场景**：老板收到实际PDF报告后，发现3个渲染质量问题。这些问题不是引擎数据错误，而是前端渲染层的bug。

---

## 检查清单（每份报告输出前/每次前端渲染修改后）

### ✅ PDF导出质量

- [ ] PDF页码不重复（第1页和第2页内容不同）
- [ ] PDF内容完整（无截断、无空白页）
- [ ] PDF文件大小合理（图片型PDF通常在5-15MB/5页）
- [ ] 中文文字可被OCR正确识别（用`pdftoppm + tesseract`验证）

### ✅ 数据展示一致性

- [ ] 没有 fallback 文本「暂无XX数据」与真实数据同时出现
- [ ] 所有引擎返回的数据都有对应的展示位置
- [ ] 有数据的§不展示「暂无」占位符

### ✅ 中文文案质量

- [ ] 没有「窗口岁」「XX分岁」类生造词
- [ ] 没有Python dict格式泄漏（`{'类型': '七杀'...}`）
- [ ] 所有数字/分数嵌入到完整句子中（不单独展示「64分」）
- [ ] 文案自然流畅，读起来像人写的

### ✅ 视觉质量

- [ ] 暗金配色一致（gold #c9a84c, 背景 #0d0d1a）
- [ ] 字体层次分明（标题/副标题/正文/辅助文字）
- [ ] 卡片间距合理
- [ ] 移动端适配（<480px 宽屏不溢出）

---

## 快速验证命令

```bash
# 1. 生成PDF并检查页数
# （手动操作：点击「下载PDF」）

# 2. OCR检查文本质量和字数
pdftoppm -png -r 200 report.pdf /tmp/ocr_check
for i in 1 2 3 4 5; do
  tesseract /tmp/ocr_check-${i}.png /tmp/ocr_out-${i} -l chi_sim 2>/dev/null
  echo "第${i}页字数: $(wc -c < /tmp/ocr_out-${i}.txt)"
done

# 3. 检查明显错误词
cat /tmp/ocr_out-*.txt | grep -c "窗口岁"    # 应为0
cat /tmp/ocr_out-*.txt | grep -c "暂无"      # 如果数据已填，应为0或合理次数
cat /tmp/ocr_out-*.txt | grep -c "{'"        # JSON格式泄漏，应为0
```
