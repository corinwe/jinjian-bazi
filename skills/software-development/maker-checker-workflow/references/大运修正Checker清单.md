# 大运修正 · Checker验证清单

> 2026-07-01 从立报告3轮修复教训中提炼
> 用于任何八字报告大运数据的Maker/Checker验证

## 何时使用

任何涉及八字报告**大运/起运数据修正**的任务，必须走Maker/Checker分离工作流。

## 层级一：大运表验证（基础）

- [ ] 起运年龄是否正确？（节气天数÷3）
- [ ] 大运干支是否正确？（阳男阴女顺排，阴男阳女逆排）
- [ ] 大运年份是否与起运年龄一致？
- [ ] 年龄范围是否连贯？（前一步的end_age+1=下一步的start_age）

## 层级二：关键发财窗口验证

- [ ] 发财窗口的年龄引用是否与大运表一致？
- [ ] 发财窗口的年份引用是否与大运表一致？

## 层级三：流年分析文本验证（最容易遗漏！）

⚠️ **重要**：大运表的修正只是第一步，流年分析文本中的大运引用往往深层嵌入。

- [ ] 流年表中每一行的"大运"列是否正确？
- [ ] 流年分析文本中提到的"X大运"是否与所在年份匹配？
- [ ] 分析文本中的"大运+流年"组合分析是否基于正确的大运？
- [ ] 涉及"三合/六合/半合/冲害"的分析是否基于正确的大运干支？
- [ ] 标注为"交运年/交接年"的年份是否确实在大运交接的年份？

## 层级四：破财预警表验证

- [ ] 破财预警表的大运列是否正确？
- [ ] 破财预警分析文本中的大运引用是否正确？

## 层级五：补财库/化解步骤验证

- [ ] 补财库步骤中提到的大运/年龄引用是否正确？
- [ ] 所有提到"某岁某大运"的表述是否同步更新？

## 常见遗漏模式（立报告教训）

| 错误模式 | 根因 | 修复方法 |
|---------|------|---------|
| 流年表大运列改了但分析文本没改 | Maker只改了表没改文 | Checker要逐行读分析文本 |
| "三卯"类组合名称不对 | 大运变了，合冲组合名变了 | 检查所有依赖大运地支的组合名称 |
| "X运将尽"类表述 | 还在用旧大运名 | 搜索所有"大运"关键词 |
| 补库步骤中的年龄引用 | 起运变了年龄参考点也变了 | 搜索所有"岁"关键词 |
