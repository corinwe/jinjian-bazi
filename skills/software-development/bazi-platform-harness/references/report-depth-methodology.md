# 引擎扩展法 — 深度报告内容生成方法论

> 来源：2026-06-26 老板纠正：「你扩充不是瞎编啊，一定是按照规则把每一段的内容细化」
> 根因：generate_deep_report.py 写了通用描述文本，没有基于规则数据展开
> 正确做法：扩展引擎分析模块的输出，而不是在格式化层写通用文本
> 老板确认：当被问「扩引擎模块还是格式化层加描述」，选「选一」

## 核心原则

不要在格式化层写通用命理描述。所有分析文字必须源自引擎模块的规则计算。

错误的做法：
```
generate_deep_report.py 中写：
  "此运为黄金时期，各方面顺风顺水"  ← ❌ 通用套话
```

正确的做法：
```
career_v2.py 的 analyze_career_full() 输出：
  "【格局定方向】格局偏印格，事业方向宜走冷门研究/分析师路线。独特，适合技术岗。"
  "【恶神制化定级别】七杀有制，化为权威和执行力，能担当重任。"
  ← ✅ 基于规则计算，有具体数据支撑
```

## detail_analysis 模式

每个引擎分析模块的主函数在返回结构化数据的同时，新增 detail_analysis 字段：

```
return {
    "career_direction": direction,
    "career_grade": career_grade,
    ...
    "detail_analysis": "\n".join(detail_parts),
}
```

生成逻辑：
1. 使用模块内已有的判定变量
2. 每条分析对应一条规则
3. 标注规则来源
4. 不要写没有数据支撑的通用描述

## generate_deep_report.py 使用策略

1. 优先使用各模块 detail_analysis 字段
2. fallback到从各节常规字段提取数据展开
3. 不添加引擎没有计算的分析文字

## 报告深度扩展三大技术（2026-06-27实战沉淀·generate_deep_report.py→1649行）

当需要将报告从不足1500行扩展至1500+行时，使用以下三种技术（按优先级排序）：

### 技术一：各维度规则引证与计算依据（优先于通用描述）

在每个主要维度（§3身强弱/§8财星/§10事业/§11学历/§12婚姻/§14健康等）后追加规则引证段：

```
## 各维度规则引证与计算依据

**§3身强弱评分依据**：
月令本气印=40分（素材20行1038）·月令中/余气印=0分·比劫全算（素材09行89）
燥土条件版：天干丙/丁引化→当火看不计分；天干壬/癸灭火→当土看计分（素材05行337+09行309）

**§8财星评分依据**：
年干=8分·月干=12分·时干=12分·年支=4分·月令=40分·日支=12分·时支=12分
藏干比例：本气100%·中气60%·余气30%
财星=只含正财和偏财（不含劫财）·日主克者为财
```

规则：每行必须有【规则→数据映射】，不写通用命理原理。

### 技术二：大运级别规则引用行

每步大运的运象分析追加一行规则依据，格式：

```
规则依据：{大运名}运天干{字}={喜/忌}·地支{字}={喜/忌}→评分{N}/10
```

确保每个分数都有具体的干支喜忌依据可追溯。

### 技术三：数据配置速查表（报告末尾）

报告末尾追加命主核心数据速查：

```
## 数据配置速查

八字：{bazi_str}
引擎：金鉴真人确定性规则引擎v5.0（{行数}行Python）
模块：shen_qiang_ruo/cai_xing/ge_ju/da_yun/dimensions等{N}个
测试：{N}条自动化测试全通过（100%）
验证：引擎+官网双通道验证
```

### 禁止的扩展方式

❌ 写通用命理描述（「此运顺风顺水」）→ 没有数据绑定
❌ 写通用附录（「全生命周期速查表」「通用风水建议」）→ 没有个性化
✅ 正确：先扩引擎模块 → 引擎输出更丰富的数据 → detail_analysis自然变厚 → 行数增长

口诀：引擎不出的数据，报告层也编不出来
      先扩引擎再展报告，不写占位不凑数
      规则引证加行数，数据速查收报告
      三大技术用完后，1500行不愁
