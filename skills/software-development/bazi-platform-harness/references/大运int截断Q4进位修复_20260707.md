# 大运起算年份：int截断 + Q4进位 修复纪实

> **触发：** 2026-07-07 老板指出魏启令大运年份从1980~1989开始是错的
> **根因：** `da_yun.py` 第181行 `birth_year + int(qi_yun_age + step * 10)` 用int()截断小数
> **修复：** 改用浮点月份偏移 + Q4(10~12月)自动进位
> **状态：** ✅ 引擎已修，技能库已更新

## 一、Bug定位

**文件：** `projects/bazi-platform/engine/da_yun.py` 第181行

**旧代码（bug）：**
```python
start_year = birth_year + int(qi_yun_age + step * 10)
```

**问题：** `int(0.33 + 0 * 10) = int(0.33) = 0` → start_year=1980

但按原始理论规则：
- "一天为四个月" → 1天÷3×12=4个月
- 起运时间 = 1980-08-06 + 4个月 = 1980年12月
- 大运从1980年12月开始，但1980年只占最后3周尾巴
- 第一个完整年度是1981年，显示为1981~1990

**新代码（修复）：**
```python
start_age_float = qi_yun_age + step * 10
start_age_int = int(start_age_float)
start_month_offset = int(round((start_age_float - start_age_int) * 12))

effective_month = birth_month + start_month_offset
year_carry = 0
while effective_month > 12:
    effective_month -= 12
    year_carry += 1

start_year = birth_year + start_age_int + year_carry
# Q4进位：起运在年底(10/11/12月) → 只占尾巴，跳到下一年
if effective_month >= 10:
    start_year += 1
```

## 二、Q4进位规则

传统规则：起运时间 = 出生时间 + 折算时间。"一天为四个月"。

如果折算后的起运时间落在Q4（第4季度，10~12月），该年只占最后不到3个月，大运的主要覆盖区间是次年起的10年，所以显示年份+1。

### 适用条件
- 仅当 `effective_month >= 10`（即起运在10/11/12月）
- 不影响正常起运（1~9月起运的不进位）

### 验证
| 人 | 出生月 | 起运偏移 | 有效月 | Q4? | 起始年 |
|:--|:------|:---------|:------|:---:|:------:|
| 魏启令 | 8月 | +4月=12月 | 12月 | ✅ | 1981 ✅ |
| 立 | 5月 | +4月=9月 | 9月 | ❌ | 2015 ✅ |
| 主母 | 7月 | +6月=1月 | 7月+6=13→1月(carry=1) | ❌(1月) | 1994 ✅ |
| 子源 | 9月 | +6月=15月→3月(carry=1) | 3月 | ❌ | 2020 ✅ |

2. **铁律㉔（引擎禁用int）**：da_yun.py中所有年份计算已用浮点偏移
3. **验证工作流第3关**：已更新为浮点月份偏移公式，删除了旧的 `int(age) + (1 if frac>=0.5)` 公式
4. **报错示例修正**：`错误2`从"int(0.33)=1980正确"改为"int(0.33)=0→1980❌"
