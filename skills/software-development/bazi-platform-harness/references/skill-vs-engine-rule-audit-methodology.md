# Skill vs Engine 模块级规则审计方法论

> **来源：** 2026-07-06 health_v2.py 深度审计实战
> **用途：** 当需要对一个特定引擎模块（如 health_v2.py）逐条对照其 skill（如 bazi-health-psychology）时使用
> **与铁律⑧的关系：** 铁律⑧是全量引擎管道审计（模块间引用链），本方法是**单模块的技能 vs 代码逐规则审计**

---

## 📋 审计触发条件

当以下场景出现时使用本方法论：

| 场景 | 例子 |
|:----|:-----|
| 老板说「深度审计健康/婚姻/财富引擎」 | health_v2.py vs bazi-health-psychology |
| 发现引擎输出和skill规则不一致 | 引擎五行旺衰判断缺具体疾病描述 |
| 新规则加入skill后，排查引擎是否已实现 | 公众号新规则→查对应代码 |
| 全量审计前，先做单模块摸底 | 先审health再审marriage |

---

## 🔬 审计三步法（2026-07-06 实战验证）

### Step 1: 资料收集

```
□ skill_view('bazi-xxx-analysis') → 加载skill全部规则
□ read_file(engine/xxx_v2.py) → 加载引擎全部代码
  └─ 如果文件>1000行，分段读取（offset/limit）
```

### Step 2: 关键词搜索验证

对 skill 中的每一条规则，用多组关键词在代码中搜索确认：

```python
# 模式 A: 具体术语搜索（用于验证特定规则是否实现）
search_files("妇科|肿瘤|欠水|自杀|色欲", path="engine/xxx.py")
search_files("乙巳|甲午|甲子|乙亥|乙丑|癸丑", path="engine/xxx.py")
search_files("骨折|血光|抑郁|失眠|增生|结节", path="engine/xxx.py")

# 模式 B: 功能块搜索（用于找到哪个函数处理了某类规则）
# 如果关键词找不到，说明该规则没有对应代码实现

# 模式 C: 结构搜索（查找表/字典/函数名）
search_files("WEAK_SYSTEMS|LIUCHONG|GANZHI_DISEASE", path="engine/xxx.py")
```

**搜到的 ≠ 已实现** — 必须验证搜索结果是否真的是该规则的代码逻辑：
- 「乙巳」出现在 `LIUNIAN_YUCE` 流年表 → 不是干支疾病表22组（容易误判！一定要看上下文）
- 「妇科」出现在日支位置描述 → 只是标签文本，不是检查逻辑
- 「肿瘤」出现在情绪器官表描述中 → 不是「土主瘤肿」规则

### Step 3: 三态分类 + 优先级

每条规则标记为：

| 状态 | 条件 | 行动 |
|:----|:-----|:-----|
| ✅ 完全实现 | 代码逻辑匹配skill规则，有明确函数/表处理 | 检查是否有反模式（只实现了表定义没检查逻辑） |
| ⚠️ 部分实现 | 代码有类似功能但细节不同，或只有表没有检查函数 | 标注差异点和缺失部分 |
| ❌ 未实现 | 关键词搜索无结果，或搜索到的内容是其他用途 | 区分：引擎内应实现 vs 外围规则（手相/面相） |

优先级分级：

| 等级 | 定义 | 例子 |
|:----|:-----|:------|
| P0 紧急 | **引擎应实现但缺失**，影响分析准确性 | 干支疾病表22组、妇科肿瘤、自杀命 |
| P1 重要 | 缺失但影响面较小，或部分实现但不完整 | 五行旺衰具体描述、WEAK_SYSTEMS后三项 |
| P2 优化 | 有实现但可完善，或边界规则 | 木旺情绪化解建议、病伤对应精确化 |

## ⚠️ 已知审计陷阱

### 陷阱1: 「名称相似≠功能相同」
- ❌ `LIUNIAN_YUCE` 表含「乙巳」→ 这不是干支疾病表22组
- ✅ 干支疾病表22组是一张全新的 `GANZHI_DISEASE` 表（乙巳→肝炎、甲午→肝炎...）

### 陷阱2: 「搜到词就以为实现了」
- ❌ 搜到「妇科」在 YIZHA_MAP → 只是位置描述，不是「用水欠水→妇科病」逻辑
- ✅ 必须查看上下文，确认是检查逻辑还是被动文本

### 陷阱3: 「表有定义→检查函数就有逻辑」
- ⚠️ WEAK_SYSTEMS 定义了4项高危特征，但只有「心血管🚨」有代码检查
- ✅ 必须验证：定义的每一条是否都有对应的检查函数

### 陷阱4: 「代码运行了→shen_label就被使用了」
- 2026-07-05教训：shen_label传入字典仅用作输出元数据，未被分析逻辑使用
- ✅ 验证引用链：入口参数 → 哪些函数接收了它 → 函数体内是否真的用了它做条件判断

以下为本次审计发现的关键关键词验证结果：

| 验证目标 | 搜索词 | 结果 | 结论 |
|:---------|:-------|:-----|:------|

| 心血管高危 | 身强水旺 + 无火调候 | check_wuxing_energy_imbalance | ✅ 已实现 |
