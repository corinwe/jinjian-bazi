产出物清单：

- [ ] 非功能性需求（性能/安全/可扩展性）
- [ ] 成功标准 / 验收条件
- [ ] 边界条件（不包括什么）
- [ ] 依赖和前置条件
- [ ] 任务分解（每个任务<2小时）

## 架构设计阶段（Phase 1）

产出物清单：
- [ ] 架构图（组件/数据流/部署）
- [ ] API契约（请求/响应格式）

- [ ] 分页和过滤（列表端点）
- [ ] 速率限制和认证
- [ ] 向后兼容（Hyrum法则）

## 增量开发阶段（Phase 2）

**加载技能：** addy-incremental-impl, addy-tdd, vercel-react-best-practices

### TDD强制流程
- [ ] RED：先写会失败的测试
- [ ] GREEN：写最少代码让它通过
- [ ] REFACTOR：清理实现
- [ ] 验证：所有测试通过

### 增量实现检查
- [ ] 薄垂直切面（< 2小时）
- [ ] 系统保持功能性
- [ ] 包括测试
- [ ] 功能标志控制（如需要）
- [ ] 回滚友好

## 测试验证阶段（Phase 3）

**加载技能：** addy-tdd, addy-testing-browser, bazi-auto-verify, bazi-wushidun-verify

### 全量验证门禁
- [ ] lint检查通过
- [ ] 类型检查通过
- [ ] 单元测试通过
- [ ] 集成测试通过
- [ ] 八字排盘验证通过（bazi-pipeline.sh --verify）
- [ ] 21§报告完整（无待填字段）
- [ ] 财星§1 vs §8一致性检查通过
- [ ] 五鼠遁/五虎遁验证通过

### Bug修复流程（Prove-It模式）
- [ ] 先写复现Bug的测试
- [ ] 测试失败（确认Bug存在）
- [ ] 实现修复
- [ ] 测试通过（确认修复有效）
- [ ] 运行完整测试套件（无回归）

## 代码审查阶段（Phase 4）

**加载技能：** addy-code-review, addy-security

### 五轴审查
- [ ] 正确性：代码实现功能
- [ ] 可读性：命名/控制流/组织
- [ ] 架构：模块边界/依赖方向
- [ ] 安全：输入验证/认证/SQL注入/XSS
- [ ] 性能：N+1/复杂度/资源使用

### 规模控制
- [ ] 变更 < 300行
- [ ] 单文件 < 500行
- [ ] 单文件变更 < 200行

## 部署上线阶段（Phase 5）

**加载技能：** addy-cicd, addy-shipping, addy-git-workflow

### 预发检查清单
- [ ] 代码审查通过
- [ ] 所有测试通过
- [ ] CI管道通过
- [ ] 功能标志配置好
- [ ] 回滚计划准备好
- [ ] 监控告警设置好

### 分阶段部署
```
Canary(5%) → 扩展(25%) → 扩展(50%) → 全面(100%)
    │            │              │             │
  30分钟观察    1小时观察      1小时观察     持续观察
```
