# 九九投资分析标准流程

## 触发条件
当老板要求分析个股、持仓、投资策略、或行业趋势时使用。每次分析按以下7个角色依次输出，确保全面覆盖。

## 两个应用场景

### 场景A：深度分析（老板主动要求分析某只标的时）
使用完整的7角色框架，每个角色详细展开，适合单只标的研究。

### 场景B：晨报速评（每日晨报「潜力标的推荐」板块）
压缩为轻量版，每只标的3-5句：
1. **基本面（1句）：** 利润增速/PE/市占率/赛道地位
2. **技术面（1句）：** 趋势方向/支撑位/阻力位
3. **新闻面（1句）：** 核心催化剂/风险事件
4. **多空（各1条）：** 看多逻辑 vs 看空逻辑
5. **操作（2线）：** 策略A（进取建仓）+ 策略B（稳健等待）

> 晨报场景不需要详细展开7个角色，但要确保每个视角都覆盖到。详见 `references/market-watchlist-full.md`（在 jiujiu-daily-morning-briefing 技能下）的标的池轮换规则。

---

## 分析流程（7角色框架）

### 1️⃣ 基本面分析师
- 公司主营业务、商业模式、护城河
- 最新财报数据：营收、净利润、EPS、增长率
- PE、PB、ROE等估值指标
- 行业地位与竞争优势
- **结论：基本面评级（优秀/良好/一般/差）**

### 2️⃣ 技术分析师
- 当前股价、近期走势
- 支撑位 / 阻力位
- 均线趋势（5日/20日/200日）
- 成交量变化
- **结论：技术面评级（强势/中性/弱势）**

### 3️⃣ 新闻情绪分析师
- 近期重大新闻和公告
- 市场情绪（正面/中性/负面）
- 行业政策环境
- 竞争对手动态
- **结论：情绪面评级（乐观/中性/悲观）**

### 4️⃣ 多头研究员（看多逻辑 🔵）
- 列出3-5个看多理由
- 每个理由附数据支撑
- 潜在催化剂

### 5️⃣ 空头研究员（看空逻辑 🔴）
- 列出3-5个看空理由
- 每个理由附数据支撑
- 主要风险点

### 6️⃣ 交易员（操作建议）
- **入场策略：** 当前是否适合买入/补仓/卖出
- **建议价格区间**
- **仓位建议：** 占总仓位比例
- **目标价：** 短期/中期/长期

### 7️⃣ 风控官（风险管理）
- **最大风险敞口**
- **建议止损位**
- **建议止盈位**
- **仓位过重警告**（如果某只股占比过高）
- **组合分散度评估**

---

## 最终决策（综合判断）
综合以上7个角色分析，给出：
- **总体评级：** ⭐⭐⭐⭐⭐ 买入 / 增持 / 持有 / 减持 / 卖出
- **核心逻辑：** 一句话总结
- **优先行动：** 老板今天该做什么

## 重要原则
1. **多空平衡：** 不能只讲好听的，风险和机会同等重要
2. **数据驱动：** 每个观点必须有数据支撑，不靠感觉
3. **长线思维：** 老板是长线投资者，避免短期噪音干扰
4. **可操作：** 最终要给老板明确的行动建议，不做模棱两可的分析

## 常见陷阱 ⚠️

### 🔴 股票代码混淆（已踩坑）
- 老板的美股是 **ECL（艺康/Ecolab）**，不是 **ICLR（ICON/益康）**
- 发音相近的代码要特别小心，**务必先用浏览器确认**股票全名和公司简介
- ECL = 工业水处理/清洁/AI数据中心液冷；ICLR = 临床CRO
- 同理，检查所有代码：MSFT ✓、9988.HK ✓、1810.HK ✓、3690.HK ✓、1211.HK ✓

### 🔴 日期/时间线混淆（已踩坑）
- 老板时区 = **北京时间（UTC+8）**
- 永远先确认当前北京时间再说话
- 阿里财报：港股收盘后（北京时间下午4点后）发布
- 美股交易时间是北京时间晚上到次日凌晨
- 不要在没有确认当前时间的情况下推断"明天"、"今天"

### 🔴 数据源不可用 & 替代方案

**Yahoo Finance 常见问题：**
- Yahoo Finance API 可能被限流（`yfinance RateLimited` / `Edge: Too Many Requests`）
- 直接 `curl query1.finance.yahoo.com` 加上 `| python3` 会被安全扫描拦截（curl_pipe_shell）
- yfinance Python 包也会被限流（IP 级别）

**✅ 已验证的替代方案：**

| 市场 | 方案 | 可靠性 |
|:----|:-----|:------:|
| 港股 9988.HK / 1810.HK / 3690.HK | Google Finance 浏览器访问 `quote/<ticker>:HKG` | ✅ 已验证 |
| 美股 MSFT/NVDA/ECL/AVGO/AMD | Google Finance 浏览器访问 `quote/<ticker>:NASDAQ`（或 :NYSE） | ✅ 已验证，含盘后价 |
| A股 002594.SZ / 000333.SZ | **Tencent API**：`curl -s 'qt.gtimg.cn/q=sz002594,sz000333'` | ✅ **最快最稳** |
| A股 OTC/粉单 | Google Finance 浏览器搜索替代代号（BYDDF、MIDUY 等） | ⚠️ 不一定有 |

**港股/美股 Google Finance 提取方法：**
1. 用 `browser_navigate(url='https://www.google.com/finance/quote/<TICKER>:MARKET')` 
2. 从 snapshot 中找 `StaticText "$xxx.xx"` + 紧邻的 `StaticText "+x.xx%"` 
3. 盘后价格标记为 `After hours`，通常在收盘价下方
4. 注意：Google Finance 的 SZ 代码格式为 `002594:SZ` 但实际不支持 A股

**A股 Tencent API 解析：**
- URL：`https://qt.gtimg.cn/q=sz002594,sz000333`（前缀 `sz` = 深交所, `sh` = 上交所）
- 返回格式：`v_code="51~name~code~current~prev_close~open~...~date~change~change_pct...~"`（波浪号分隔）
- 关键字段索引（0-based）：(3)当前价, (4)昨收, (30)日期, (31)涨跌额, (32)涨跌幅%
- 示例 Python 解析：`parts = line.split("~"); current=parts[3]; prev=parts[4]; chg=parts[31]; chg_pct=parts[32]`

**数据源优先级（推荐顺序）：**
1. Tencent qt.gtimg.cn（A股首选，极快无限制）
2. Google Finance 浏览器导航（港股/美股首选，含盘后）
3. Yahoo Finance（最好用，但易被限流，备用方案：使用不同 User-Agent 或间隔重试）
4. Bing/Google 搜索新闻（作为交叉验证）

### 🔴 图片处理（已支持OCR提取）
- 老板可能发持仓截图（JPEG/PNG）
- **✅ 已支持**：用 OCR 提取图片中的文字数据
- 安装：`apt-get install tesseract-ocr tesseract-ocr-chi-sim -y && pip install pytesseract`
- 用法：缩小图片到宽度600 → `pytesseract.image_to_string(img, lang='chi_sim+eng')`
- 已验证：港股富途/美股/A股三券商持仓截图可准确识别（2026-05-13）
