# 排盘引擎架构铁律（2026-06-26新增）

> 根因：本会话中我写了 buggy 的 paipan.py 替代已验证的 bazi-engine.py，导致月柱全部算错一位。

---

## 核心原则

### ✅ 永远使用 bazi-engine.py

`bazi-engine.py`（位于 `/root/weiwuji-knowledge-base/07-国学哲学/八字命格/scripts/bazi-engine.py`）是已验证的排盘引擎：
- 使用 `ephem` 天文库精确计算节气时间（分钟级精度）
- 月柱按节气分界（立春/惊蛰/清明/立夏/芒种/小暑/立秋/白露/寒露/立冬/大雪/小寒）
- 支持真太阳时修正
- 输出JSON格式，含四柱、纳音、藏干、十神、大运、起运年龄

### ❌ 禁止自己写 paipan.py

自己写的排盘引擎有三个已知bug陷阱：
1. **月柱计算** — YUE_ZHI映射偏移 + real_month公式错误（节气前/后分支计算均bug）
2. **日柱计算** — 基准日偏移，导致日柱错误
3. **边界处理** — 立春前后年柱切换、12月/1月边界回绕

### 调用方式

```python
import subprocess, json

BAZI_ENGINE = "/root/weiwuji-knowledge-base/07-国学哲学/八字命格/scripts/bazi-engine.py"

def run_paipan(name, gender, year, month, day, hour):
    shichen_idx = hour // 2  # 0子 1丑 2寅 3卯 4辰 5巳 6午 7未 8申 9酉 10戌 11亥
    cmd = [
        "python3", BAZI_ENGINE,
        str(year), str(month), str(day),
        str(hour), "0", str(shichen_idx),
        gender, name, "", "--json"
    ]
    result = subprocess.run(cmd, capture_output=True, text=True, timeout=30)
    return json.loads(result.stdout)
```

### 评分引擎

评分引擎位于 `/root/bazi-platform/engine/` 目录：
- `shen_qiang_ruo.py` — 身强弱评分（九龙道长原始规则）
- `cai_xing.py` — 财星分数计算

这些评分引擎已经过验证，**不要重写**。如果有校准差异，先检查规则版本差异（如年支比劫是否计算），再决定是否修改引擎代码。

### pyc缓存清除（重要！）

修改 engine/*.py 后：
```bash
rm -rf /root/bazi-platform/engine/__pycache__/
pkill -f "main.py"
# 重新启动服务器
```

否则旧pyc缓存会导致修改不生效。
