### 知识点提取方法论（老板亲授·固化·每次扫描必用）

> ⚠️ **核心方法**：凡是**断言式判断句**，不针对个例而针对一类情况，即使只提了一句两句，就是核心知识点。

**识别特征**：
```
✅ 是知识点的句式：
  - "如果...则..."（条件式判断）
  - "...代表..."（定义式判断）
  - "...为..."（归属式判断）
  - "男命...女命..."（分类式判断）
  - 句式不包含具体人名/具体八字
  - "也就是..." / "意味着..." / "所以..." 推导出的规则
  - "另外..." / "还有一个事..." / "对了..." 引出补充规则

❌ 不是知识点的句式：
  - 只针对某个人（"他是辛金日主"）
  - 描述具体案例事件
  - 过渡/闲聊/推销内容
```

### 铁律一：逐行通读，不是逐段读


### 铁律二：一句两句也是知识点（强化版·本会话教训固化）

```yaml

  只要符合"断言式判断句"特征，不管这个规则在文中出现多少次

六大高遗漏类型（本会话实战教训）：

    ✅ 正确：口诀必须是完整列表→天干四冲+地支六冲全部列全
  
  类型D — 案例中蕴含的原则：
    ❌ 错误：只记了"胆囊炎"没记"肝和胆没有痛觉细胞→不疼→疼就是大病"
    ✅ 正确：案例中的推理链每一步都是独立知识点
  
  类型E — 随口补充的规则：
    ❌ 错误：导师说"还有一个事啊"然后讲偏印性冷淡→当闲聊跳过
    ✅ 正确：导师专门说"还有一个事"→一定是重要补充→必须提取
  
  类型F — 一两句的跨模块引用：
    ❌ 错误：疾病文档中提了"杀印相生格是伟人格"→只在疾病笔记里记了
    ✅ 正确：跨模块引用→物理更新到格局模块的skill文件

  类型G — 结构式知识遗漏（本会话新增·最致命）：
    ❌ 错误：原文有完整的"宫位年龄逐字分龄法"体系（每个字9年·7字计龄·日干不计），
           但只关注了单个断言句，没把完整的结构式知识抽象出来
    ✅ 正确：读完文档后问自己——这篇文档有没有一个完整的框架/体系/方法论？
           如果有，应该单独抽象并记录为一个独立的知识模块，不能只拆成零散断言句

    【检查方法】读完每份文档后，停一分钟问自己：
    ① 这篇文档的核心框架是什么？
    ② 有没有一个完整的体系/流程/方法论贯穿全文？
    ③ 如果去掉所有案例和口水话，剩下的骨架是什么？
    ④ 这个骨架是否完整记录到了知识库/技能中？

    【本会话教训】16-四柱八字课程教学与案例分析_2026-05-30.txt
     原文完整记录了逐字分龄法：一个字大概代表9年，年干1~9岁，年支10~18岁，
     月干19~27岁，月令28~36岁，日支37~45岁，时干46~54岁，时支55岁以后
     同时还有：日干不计年龄。7字计龄的结构式知识。
     我只提取了零散的宫位知识，没把这套完整的逐字分龄法抽象出来。

案例分析（本会话教训—核心复盘）：
  ✅ 原文一处："食神落在了这，这里落食神的人，对行哎之事都会特别喜欢"
     之前漏了 → 只提取了偏印落配偶宫=性冷淡，没提取食神=欲望强

  ✅ 原文："纳音有水不算"

  ✅ 原文："丁癸冲、丙壬冲...天干有四冲"
```

### 铁律四（防遗漏扩展）：用户说的也要验证——不当墙头草再强化

> ⚠️ **核心教训**：用户之前告诉我藏干计分是\"本气100%、中气30%、余气10%\"，\n> 我盲目接受了，没跟课程原文验证。后来用户自己纠正——正确应是\"中气60%、余气30%\"。\n> 用户亲口说：「如果我说的是错的，你要告诉我，我错了，而不是说完全听我的」

```yaml
【强制规则】
  用户给的任何信息（包括藏干计分、规则口诀、特殊案例等）
  必须用课程原文/知识体系交叉验证后再采纳
  不能因为\"是老板说的\"就直接接受

  三步验证法：
  ① 用户说了什么？记下来
  ② 对照已有知识体系（课程原文/SKILL.md/知识库）→是否一致？
  ③ 一致 → 采纳 | 不一致 → 告知用户\"这里跟课程原文不同，您看哪个为准？\"

【实战案例·本会话教训】
  ❌ 错误：用户说\"中气30%\"→我直接记成30%→实际应为60%
  ✅ 正确：用户说\"中气30%\"→我去查课程原文\"中气是60%，12乘以0.6是约7分\"→发现矛盾→告知用户

【口诀】
  老师说的也要验，课程原文是准绳
  不当墙头草，只认真知识
```

```yaml
每次吸收新知识后，必须确认三个体系全部更新：

【知识库】— 理论存档
  □ 知识库理论文档已更新（路径：01-理论体系/）
  □ 速查卡/速查表已更新（如有）
  □ 原始素材已归档（00-原始素材/编号排序）

【技能库】— 分析执行引擎
  □ 本模块的skill文件已更新（如bazi-health-psychology/SKILL.md）
  □ 跨模块知识点已物理更新到目标skill文件（grep确认存在）
  □ 对应的reference文件已同步更新
  □ 对应的自动化工作流已同步更新

【自动应用体系】— ⭐ 两个子系统都要检查 ⭐

     □ 新知识是否影响验证流程？（流程步骤/判断规则/门禁逻辑）

【口诀】
  知识提取三联动，库+技+应用不能少
  库是理论存档处，技是执行引擎魂
```

## 扫描流程

```
1. 逐行通读全文，标记所有断言式判断句
2. 逐个判断：是否已存？归属哪个模块？

6. 推库
```

> 这是必修步骤，不可跳过。跳过 = 遗漏知识点。

## 通用检查项（必过）

```
□ 1. 这份文件/资料是否被完整读取了每一段？
    	⚠️ 从头读到尾，不跳段。文档末尾的案例往往藏着遗漏。
	
□ 2. 是否有能量倍数/量化规则？
    	例：三合多少倍、六合多少倍、有引化vs无引化
    	检查点：三会→三合→半三合→六合→拱合→三刑→自刑→冲→害→破→透干→通根
	
□ 3. 是否有辰戌丑未/墓库/入库vs入墓规则？
    	检查点：40分阈值、五行变性、生克反转、与日元无关、位置规则、刑冲打开/关闭/损坏
	
□ 4. 是否有十神×事件级别对应？
    	检查点：七杀/伤官/枭印/劫财/灾煞/血刃 各4级事件
	
□ 5. 是否有穿害/刑冲的特殊应事？
    	检查点：子未=皮肤病、酉戌=牙疼、寅巳=突发病 等
	
□ 6. 是否有从格/特殊格局规则？
    	检查点：从强恒定20分、从弱恒定50分、假从弱、真从弱、化气格、伤官伤尽格
	
□ 7. 是否有生多为克/分担原理？
    	检查点：5倍阈值、5倍漂1个木但漂不动2个
	
□ 8. 是否有实战案例展示的断事流程？
    	检查点：是否有Step by Step的断事示范
	
□ 9. 是否有空亡的特殊影响？
    	检查点：减半规则、双向影响（库+空=减半好事，墓+空=减半坏事）
	
□ 10. 是否有36岁分界/节气应事规则？
     	检查点：36岁前后应灾轻重不同、节气交割前后3天应事
```

## 遗漏高发区（特别关注）

```yaml
① 文档末尾的案例：最容易遗漏，因为前面理论看完了容易松懈
② 学员互动中的回复：导师被问到的问题往往是最关键的知识点
③ 看似"废话"的过渡句：导师经常在闲聊后爆出金句
④ 被自己否定的说法：如"不对，我看错了" → 纠正后的才是正确的
⑤ 倍数表边缘的规则：如自刑、六害、六破的倍数容易被遗漏
⑥ 一句话带过的特例规则：如"纳音有水不算分"这种一句话但很关键的规则
⑦ 对比性知识：如"偏印=性冷淡 vs 食神=欲望强"这种对比才完整
⑧ 否定式知识：所有包含"不算""不能""不要""区别"的句子
```
