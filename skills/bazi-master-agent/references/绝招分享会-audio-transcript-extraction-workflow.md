## 与公众号文章提取的核心区别

| 维度 | 公众号文章 | 绝招分享会（音频转写） |
|:----|:----------|:---------------------|
| 内容密度 | 高——每句都是干货 | 极低——90%是闲聊/互动/广告 |
| 噪声类型 | 极少 | 开场白、直播间异常、闲聊、提问互动、广告推销 |
| 提取策略 | 逐句全记录 | 先遍历快速标记，再提取规则 |
| 输出比例 | ~90%沉淀 | ~10-20%沉淀 |
| 文件格式 | .html/.txt | .docx |
| 工具链 | curl + 浏览器提取 | python-docx |

## 五步提取流程

### Step ① — 批量文件提取

所有文档统一用 python-docx 提取到 /tmp/：
```python
from docx import Document
doc = Document("{path}")
text = "\n".join([p.text for p in doc.paragraphs if p.text.strip()])
```

### Step ② — 快速预览分类

读前15-20行判断内容领域：
- 过三关预测 → bazi-fortune-analysis
- 化解/阵法/法术 → bazi-remission-methods
- 健康/疾病 → bazi-health-psychology
- 神煞/基础 → bazi-foundation-analysis
- 婚姻/感情 → bazi-marriage-analysis

### Step ③ — 逐段精读+并行提取

使用 delegate_task 并行处理，每个子agent精读1-3篇，按技能域分组输出。

**过滤规则（必须剔除的内容）：**
- ❌ 直播间异常：打不开/掉线/转播设置
- ❌ 广告推销：课程推广/证书考试/产品推销
- ❌ 闲聊互动：喊人/寒暄/等同学/开玩笑
- ❌ 纯叙述无规则：真人真事背景描述（无规则可提炼）
- ❌ 重复冗余：同一个事说两遍以上

**保留规则（必须提取的内容）：**
- ✅ 条件式断言：「XX了要XX」
- ✅ 操作步骤：「先做X，再做Y」
- ✅ 禁忌提醒：「XX不行」「不能XX」
- ✅ 验证案例：「某某用了有效」
- ✅ 口诀/速查表
- ✅ 原理说明：「因为XX所以XX」
- ✅ 需要多少天/多少次/什么时间

**输出格式：**
```
| 序号 | 类别 | 规则/方法 | 详细说明 | 对应技能包 |
|:----:|:----|:----------|:---------|:----------|
```

### Step ④ — 跨技能沉淀

同一条规则可能跨多个技能包。按主域+副域分配：
- 主域：最直接的应用技能包
- 副域：在§合入建议中标注「此规则与xx技能交叉，可引用」

规则超过15条时，按子分类拆多个表格（如：化解类/神煞类/健康类）。

### Step ⑤ — 全量审计+推库

完成后必须回答4个审计问题：
1. **所有技能包都更新了吗？** → 列出涉及/更新/跳过的完整表格
2. **理论知识沉淀了吗？** → 规则写入对应SKILL.md
3. **物理自动化体系更新了吗？** → 脚本/规则文件检查
4. **推送仓库了吗？** → git commit + push 状态

## 实战教训（2026-06-30·9篇）

### 教训① — 子agent写错路径
子agent可能把内容写到知识库路径而非技能库路径。
**解决：** 完成后 `diff` 两个路径，不一致则 `cp` 过去。

### 教训② — 背景法/佩戴法归属
天合地合吸引法同时属于bazi-remission-methods（方法）和bazi-marriage-analysis（应用场景）。
**规则：** 方法本身归remission，场景归marriage，交叉引用。

### 教训③ — 过三关绝招的特殊性
文档4《周易预测过三关绝招讲解》的核心价值是「不用八字就能断事」。
这类内容归bazi-foundation-analysis的过三关节（§27/§29.18），不要独立成技能包。
