## ⚠️ 致命教训：立的财库误判（触发CHECKER设计）

**问题：** 立（甲木日主，八字辛卯癸巳甲戌庚午）的财富报告v1.1中，财库标注从正确的"有（日支戌偏财库）"被错误"修正"为"无（戌为火库/食伤库，非财库）"。

**根因三重：**
1. **§10.1表理解不全**：戌同时出现在"财才墓库(土库)"和"食伤库(火库)"两列。对于甲乙木（财=土），戌=财才墓库。只看火库属性漏了土库属性。
2. **缺乏交叉验证**：旧版循环引擎只有learn→verify→apply 3段，没有report vs skill的校验环节
3. **过度修正**：v1.0原本写对了，v1.1被"修正"为错误

**为什么旧循环没发现？** 旧版只验证"新规则是否写入skill"，不验证"skill已有规则是否在报告中正确应用"。

## ⚠️ 致命教训：印枭墓库规则全错（2026-07-02老板校准）

**问题：** §10.1表中庚辛金印库写为丑(土库)，壬癸水官杀库写为未(土库)。实际土的墓库=戌（原始素材"戌=火库+土库"）。

**错误表（修正前）：**
```
庚辛金 | 丑(金库) | 未(木库) | 戌(火库) | 丑(土库)❌ | 辰(水库)
壬癸水 | 辰(水库) | 戌(火库) | 未(土库)❌ | 丑(金库) | 未(木库)
```

**正确表（修正后）：**
```
庚辛金 | 丑(金库) | 未(木库) | 戌(火库) | 戌(土库)✅ | 辰(水库)
壬癸水 | 辰(水库) | 戌(火库) | 戌(土库)✅ | 丑(金库) | 未(木库)
```

**戌的完整分布（每种日主至少2列）：**
- **甲乙木**：财才墓库(土库) + 食伤库(火库)
- **丙丁火**：比劫库(火库) + 食伤库(土库)
- **戊己土**：比劫库(土库) + 印库(火库)
- **庚辛金**：官杀库(火库) + **印库(土库)** ⚠️ 原来写丑
- **壬癸水**：财才墓库(火库) + **官杀库(土库)** ⚠️ 原来写未

**口诀：** 戌是土库不是猜，生我者印我克财；庚辛见戌印有库，壬癸见戌官自来。

## ⚠️ 5种墓库都要检查，不只是财才墓库

**老板校准（2026-07-02）：**
1. **比劫墓库** = 蓄财库①·合作得财（素材11行365：日时柱有库=财库或比劫库）
2. **财才墓库** = 蓄财库②·直接蓄财（效果最直接）
3. **官杀库** = 权力/管理
4. **印库** = 证书/合同（印枭=生我者的墓库）
5. **食伤库** = 赚钱动力源（不是蓄财，但决定赚钱能力）

**强制规则：** CHECKER必须检查全部5种墓库，不能只看财才墓库一个维度。

## 🔬 CHECKER协议

### 核验算法

```python
for each report:
    ① 提取八字(4柱) + 日主(如甲木)
    ② 日主→日主组(甲乙木)
    ③ 查§10.1表 → 该日主的5种墓库
       - 比劫库, 财才墓库, 官杀库, 印库, 食伤库
    ④ 逐库: 取地支→检查八字中是否有该地支→检查报告是否提及
    ⑤ 不一致 → 拦截! 输出问题清单
       一致   → 通过
```

### 戌的双重属性验证（常见陷阱）

戌同时出现在多于1列，需要特别注意：

```python
# 错误：只查一列（立报告的v1.1错误只查了食伤库）
if 戌 in 八字 and 戌 == 食伤库:
    print("食伤库 ✅")  # 正确但漏了财才墓库!

# 正确：遍历所有5列
for ku_type in [比劫库, 财才墓库, 官杀库, 印库, 食伤库]:
    if 戌 == 表[日主][ku_type]:
        print(f"{ku_type}=戌 ✅")
```

## ⛔ 反模式

| 反模式 | 为什么错 | 正确做法 |
|:-------|:---------|:---------|
| 手动精读→手动对比→手动写报告 | 没有自动化，不可重复，易漏 | 先跑循环引擎再执行输出 |
| 只看新规则，不扫受影响报告 | 报告数据不一致 | 循环引擎自动扫描所有报告 |
| 只更新skill不更新报告 | 报告内容过时 | 循环引擎自动列出所有需更新报告 |
| checker只查财才墓库一个维度 | 漏了比劫库(蓄财)+食伤库(动力源)+印库+官杀库 | 5种墓库全量检查 |
| checker只验证新规则 | 已有规则在报告中的错误应用漏检 | CHECKER读skill表→推导→对比报告 |
| 把戌只当一种库 | 戌同时在2+列出现，只看一种会漏判 | 检查全部5列 |
| 印库=丑(庚辛金)或未(壬癸水官杀) | 土库=戌，庚辛金印库=戌，壬癸水官杀库=戌 | 原始素材"戌=火库+土库" |
| 规则体系改了但报告没更新 | 四系统不同步，报告内容与规则不一致 | 四系统同步更新：规则+技能+自动化+报告 |
| 知识库存docx | 知识库应只存原始格式.md | 只存.md，转格式只用于飞书发送 |
