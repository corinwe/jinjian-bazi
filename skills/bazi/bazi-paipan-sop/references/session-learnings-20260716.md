> 本轮从老板反复指出的"唯一数据源""物理机制""端到端验证"等架构问题出发，建立起数据层+业务层分离的物理约束体系。

## 1. 体用定义终版（不可再改）

```
体 = 日主 + 印 + 比劫 + 食伤（食神+伤官全部归体）
用 = 财 + 官（正财偏财正官七杀，只有这两个）
做功 = 体去追求用的方式
```

**口诀**：体是我和我的工具，印比食伤都在体；用是追求的目标，财官才是真用途。

## 2. 天干财官必须计入用（新增Bug!）

**问题**：report-generator.py只把藏干财官计入用，漏了天干财官。
- 子源八字: 年干辛正财8分 + 月干癸正官12分 + 时干癸正官12分 = 32分漏了
- 导致用从46.4分降到14.4分，体用比从2.62变成6.78
**修复**：天干十神也要检查是否在yong_keys中，年干8分/月时干12分加入用

## 3. 大运顺逆看年干阴阳，非日干（Bug!）

**问题**：大运扩展函数用日干定顺逆，应该是年干。
- 魏启令(年干庚阳+男→顺排) → 之前用日干辛阴判为逆排
**修复**：`is_shun = (YY_MAP[nian_gan]=='阳' and gender=='男') or (YY_MAP[nian_gan]=='阴' and gender=='女')`

## 4. 大运只出8步，不是10步

**问题**：默认扩展到10步(到100岁)，老板说8步(到80岁)足够。
**修复**：extend_dayun()默认count=8

## 5. 数据源架构（老板校准·核心）

**架构**：
```
引擎CLI → engine.json → bazi-data-source.py → datasource.json(23字段)
                                                    ↓
                                          ┌─ 盲派体系(体用规则)
                                          └─ 九龙体系(身强弱/喜用神)
```

**物理约束链**（按顺序）：
```
① pre_llm_call (inject-context.sh) → 自动检测BAZI_DATASOURCE
② pre_tool_call (precheck.py) → 检查BAZI_DATASOURCE + /tmp/.bazi_verified
③ report-generator.py → def module(DS)——没有DS参数拒绝执行
④ post_tool_call (verify-format.py) → 数字对齐验证
```

**数据源字段**（23个）：
8字段(年干年支月干月支日干日支时干时支) + 四柱 + 藏干 + 藏干十神 + 十神 + 纳音 + 空亡 + 日主 + 日主五行 + 性别 + 八字 + 身强弱 + 起运年龄 + 神煞(8种) + 8大运

## 6. 报告不是数据堆砌

**之前**：报告中只打印DS字段值（"体=122.8分 用=30.0分"）
**现在**：DS数据+规则=分析结论（"身强64分→喜火/水/木→忌土/金"）
要求：每份报告包含具体分析文本，不仅是数据表格

## 7. 口诀条件校验

**典型案例**：「四柱皆带财，库在日时来，食伤能生财，不富没天理」
- 家主八字年柱（庚申）无财 → 四柱不满足 → 不能直接引用
- 应写：三柱有财，年柱无财，但日时有库+食伤生财路线清晰
**通用规则**：任何口诀/断语引用前，逐条核对其条件是否满足本八字。

## 8. 大运真假规则 — 藏干也算有

原局申中戊(余气30%) → 大运戊子天干戊为真（非假）
原局亥中甲(中气60%) → 大运甲申天干甲为真（非假）
**规则**：只要藏干（本气/中气/余气）存在对应的天干，该天干在大运出现即为真。
