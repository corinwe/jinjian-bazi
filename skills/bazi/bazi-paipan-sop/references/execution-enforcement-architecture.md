> **核心问题：** 写了规则但不执行。根因：SOUL.md被塞入过多SOP步骤（铁律A~G），稀释了人格定义；且无工程手段强制工具调用验证。

## 物理审计流程（每次执行任务后的标准验证）

```
第1步：文件有这句话吗？  → grep -c "关键词" 文件
第2步：规则被执行了吗？ → 跑一次真实用例验证
第3步：输出中能验证吗？ → grep -c "关键词" 输出
第4步：从输出反推回文件 → 每个输出数据能否追溯到引擎JSON？

口诀：写下→审计→说OK，三步少一不算完
      文件有不算有，审计过了才是真
```

## 常见失执行模式（本会话教训）

| 失执行模式 | 表现 | 修复 |
|:----------|:-----|:-----|
| 写了规则但跳过 | 铁律A~G在SOUL.md中但写报告时没看 | 移出SOUL.md到HERMES.md（职责分离） |
| 不读JSON凭记忆写 | 藏干十神写错、生克关系写反 | 铁律F：每§前先`python3 -c "import json; ..."` |
| 强制内容漏了 | 婚姻子女年份表没放 | 铁律G：先搭骨架+占位符再填肉 |
| 输出前不做grep | 品牌名残留 | 铁律E：输出前grep扫查 |
| 不验证工具返回值 | 以为写完了实际没写 | tool_use_enforcement: true |
