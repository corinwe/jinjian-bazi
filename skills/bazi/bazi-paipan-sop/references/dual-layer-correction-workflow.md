# 双验修正工作流：理论层 + 实践层

## 问题来源

2026-07-09 老板指出「月令只看本气对日元生助」。查技能文件只有印的规则，缺比劫。但**引擎代码同样**缺这个规则——月令比劫循环了所有藏干包括中气。

老板一句反问暴露了问题：「有补充到你的身强弱技能，以及强制加载的物理应用体系当中吗？」

## 教训

当老板指出一个理论规则错误时，**不能只查技能文件**——必须同时查引擎代码，因为两者经常不一致。

```
过去的行为模式:
  老板指出规则 → 查技能文件 → 修技能 → 汇报完毕 ❌
  
正确的行为模式:
  老板指出规则 → 并行查技能文件 + 引擎代码 → 修两层 → 重跑 → 汇报 ✅
```

## 为什么两层会分离

| 层 | 文件 | 问题 |
|:---|:-----|:------|
| 理论层 | skill/SKILL.md | 「印在月令本气才计算生助」— 只写了印 |
| 实践层 | engine/shen_qiang_ruo.py | 月令比劫全计分，循环所有藏干 |

两层不同步的原因：
1. 技能文件更新时下意识以为引擎已经一样
2. 引擎代码更新时没有同步更新技能文件
3. 两者独立维护，缺乏交叉校验

## 双验修正流程

```
Step 1 — 确认问题
  老板指出规则 → 不辩解，说「明白我查」
  
Step 2 — 双线并行搜索
  线A（理论层）: search_files 技能目录 → 找该规则的所有写法
  线B（实践层）: search_files 引擎目录 → 找该规则的代码实现
  
Step 3 — 对比差异
  对比A和B的结果 → 列出所有不一致处
  特别注意：同一规则名在不同文件中是否有不同理解

Step 4 — 同时修复
  修复技能文件 → 补充缺失的规则
  修复引擎代码 → 修改错误的实现

Step 5 — 双向验证
  引擎代码修完 → 跑真实案例验证
  技能文件修完 → 跑端到端分析确认规则被正确应用

Step 6 — 报告修正
  更新知识库中所有受影响的报告
  git commit 双库（工程库 + 知识库）
```

## 检查清单

```
□ 技能文件中该规则的描述完整吗？
□ 引擎代码中该规则的实现与技能描述一致吗？
□ 所有相关报告已更新？
□ 受影响的案例已重跑验证？
□ 双库已git commit？
```

## 适用范围

此流程适用于所有「规则修正」类任务：
- 身强弱规则修正
- 十神判定规则修正
- 格局判定规则修正
- 财星/婚姻/子女/健康等任何事象规则修正
- 大运/流年计算规则修正

任何修正都必须过理论和实践两层。
