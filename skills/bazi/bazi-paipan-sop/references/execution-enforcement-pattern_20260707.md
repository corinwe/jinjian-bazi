# 执行强制模式（2026-07-07会话沉淀）

> **来源**：2026-07-07全链路审计会话。老板反复纠正"写了规则但不执行"的问题。
> **核心问题**：规则写到了文件里，但写报告时没有物理执行——凭记忆替代了引擎数据。
> **解决方案**：职责分离 + 物理检查点 + 工具返回强制校验。

## 总结的三个核心教训

### 1. 职责分离：SOUL管人格，HERMES管SOP

```
之前：所有规则都堆在SOUL.md → "知道"但是"不执行"
现在：
  SOUL.md = 身份+风格+3条执行铁律 + "必须遵循HERMES.md SOP"
  HERMES.md = 项目级铁律 + 铁律A~G(具体执行标准)
  SOP = 具体6 Phase执行流程
```

原理：人格定义（你是谁）和项目流程（你要做什么）不能混在一起。

### 2. 物理检查点：每§写完先验JSON再打勾

```
之前：写报告→直接写完→推库（不检查数据来源）
现在：每写一§→读JSON确认数据一致→打勾→下一§
```

每个§的 ✅ 完成标记不是"写完了"，是"写完了且数据与JSON一致"。

### 3. 工具返回强制校验

config.yaml 的 `tool_use_enforcement: true`（已从auto升级）

系统会在每次工具调用后检查返回数据，确认执行成功才允许下一步。阻断"脑补执行成功"的问题。

## 老板铁律（每次写报告前默念）

```
① 引擎数据不用，那你跑它干啥？
② 每§之前读JSON，从JSON里取十神
③ 写一个§，验一个§，匹配了才能打勾勾
④ 文件有不算有，审计过了才是真
```
