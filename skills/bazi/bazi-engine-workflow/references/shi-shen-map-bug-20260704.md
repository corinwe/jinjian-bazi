# SHI_SHEN_MAP 食神/伤官阴阳映射颠倒 Bug (2026-07-04)

## 错误现象

辛金日主 × 月干癸水 → 引擎输出「伤官」（正确应为「食神」）
辛金日主 × 时干壬水 → 引擎输出「食神」（正确应为「伤官」）

## 根因

`projects/bazi-platform/engine/constants.py` 第95行注释和第102-103行 SHI_SHEN_MAP 中
食神/伤官的阴阳映射完全反了：

```python
# 错误（修复前）:
("我生", "异"): "食神",   # ❌ 异阴阳=伤官，不是食神
("我生", "同"): "伤官",   # ❌ 同阴阳=食神，不是伤官

# 正确（修复后）:
("我生", "同"): "食神",   # ✅ 同阴阳=食神（阴生阴/阳生阳）
("我生", "异"): "伤官",   # ✅ 异阴阳=伤官（阴生阳/阳生阴）
```

## 影响范围

所有通过 `get_shi_shen_for_gan()` / `get_shi_shen_for_cang_gan()` 获取十神的代码
均受影响。具体表现：
- 阴生阴（如辛→癸）本应=食神，错误输出=伤官
- 阴生阳（如辛→壬）本应=伤官，错误输出=食神

## 已发现的影响

### 案例A：家主魏启令（辛金日主·月干癸水）
- 八字：庚申 癸未 辛亥 辛卯
- 月干癸水被标为「伤官」97处，实际应为「食神」
  - 辛阴生癸阴=同阴阳→食神，引擎错误输出为伤官
- 格局也从「伤官配印」降级为「偏印格+食神泄秀」
- 修复：生成v2.0报告（2026-07-04·1,505行）

### 案例B：主母成（庚金日主·时干壬水）
- 八字：丁卯 丁未 庚午 壬午
- 时干壬水被标为「伤官」87处，实际应为「食神」
  - 庚阳生壬阳=同阴阳→食神，引擎错误输出为伤官
  - 「伤官见官」格局判断全错，应为「食神见官」(食神温和VS正官自律，不是冲突是互补)
- 修复：生成v2.0报告（2026-07-05）

### 模式总结
```
错误方向：同阴阳被我生 → 标成了「伤官」
          异阴阳被我生 → 标成了「食神」
          └─ 这就是 SHI_SHEN_MAP 中 ("我生","同") 和 ("我生","异") 完全互换的结果

对阴日主（辛金）：月干癸水(同阴阳=食神) 被标为伤官 → 格局降级
对阳日主（庚金）：时干壬水(同阴阳=食神) 被标为伤官 → 伤官见官误判
```
任何日主为「阳金/阴金/阳水/阴水」且月干/时干/藏干出现「我生」关系时，十神都错了。

## 🚨 引擎数据不得被LLM绕过铁律（2026-07-05新增）

> **致命教训**：本会话中，主母报告v1.0（庚午日）的**空亡本应为戌亥**（庚午日→甲子旬→空亡戌亥），
> 引擎 `_get_kong_wang()` 函数正确返回了「戌亥」，但LLM报告生成器**没有使用引擎的空亡输出**，
> 而是【自己独立计算】了空亡，写成了「辰巳」——全错！
>
> 子源报告v1.0（丙戌日）的旧版空亡也被LLM独立计算为「戌亥」，
> 正确应为「午未」（丙戌→甲申旬→空亡午未）。
>
> **根因**：LLM认为「我懂空亡怎么算」，直接绕过了引擎的结构化输出。
> 这和食神/伤官Bug是不同的问题——引擎数据是对的，但LLM没用它。

### 🚨 强制规则

```yaml
【强制流程】每次生成报告时：

Step 0 — 从引擎获取完整数据
  bash projects/bazi-platform/scripts/bazi-must-run-engine.sh -n {姓名} -g {性别} -y {年} -m {月} -d {日} -h {时}
  → 结果包含：四柱/纳音/文昌/引擎评分(身强弱/财星/格局/喜用/大运)

Step 1 — 使用引擎数据作为唯一来源
  ✅ 空亡 → 从引擎输出的日柱推空亡（日柱某旬→某空亡）
    检查方法：日干推空亡（甲子旬→戌亥空...）
    或直接用引擎 `day_pillar.kong_wang` 字段（如果有）
  ✅ 十神 → 从引擎 `engine_scores` 或手动验证后写入
  ✅ 身强弱/财星/起运 → 直接从 `engine_scores` 提取
  ✅ 大运序列 → 直接从 `engine_scores` 提取
  
  ❌ 禁止：LLM自行推算空亡（如「丙戌日→甲申旬→戌亥空」→❌应该是午未）
  ❌ 禁止：LLM自行推算十神（如「辛生癸=伤官」→❌应该是食神）
  ❌ 禁止：LLM自行推算身强弱/大运

Step 2 — 双重验证（推库前必须过）
  □ 空亡核对：日柱→六甲旬→空亡地支。写下来，与报告中的空亡对照
  □ 十神核对：每个天干写出来「日生X=同/异→食神/伤官」
  □ 喜忌核对：身强弱→喜克泄耗or喜生扶→喜用/忌神是否正确

口诀：引擎数据是第一，空亡十神直接取
      LLM不要自己算，绕过引擎出大错
      空亡验证靠六甲，十神验证靠阴阳
      推库之前过两遍，引擎输出对一对
```

## 🚨 重要教训：引擎Bug修复后的全量报告再生规则

当引擎常数/核心逻辑被修复后，**所有已生成的报告都必须重新生成**，不只是当前正在看的那一份。

**2026-07-04/05教训：** 引擎SHI_SHEN_MAP修复后，只重新生成了家主报告，主母报告还是老的错误数据。
老板发现「主母也不是伤官，是食伤」时才意识到v1.0报告还用的是旧数据。

**正确流程：**
```python

    ↓

    ├─ 所有 commit < 修复commit（如750fc37）的报告
    └─ 所有日主有「我生」十神的报告
    ↓

    ├─ 用delegate_task并行重新生成每个报告
    └─ 每个报告必须指定正确的引擎数据作为source of truth
    ↓
Step 3: 验证+删除旧版
    ├─ 检查新报告的十神正确性（grep "伤官" 不应该出现在我生同的位置）
    └─ 旧版v1.0删除，不残留
    ↓
Step 4: 推库
```

## 十神阴阳铁律

```
生我者 = 印    : 同阴阳=偏印, 异阴阳=正印
我生者 = 食伤  : 同阴阳=食神, 异阴阳=伤官  ← 这是最容易被搞反的！
克我者 = 官杀  : 同阴阳=七杀, 异阴阳=正官
我克者 = 财    : 同阴阳=偏财, 异阴阳=正财
同我者 = 比劫  : 同阴阳=比肩, 异阴阳=劫财
```

记忆口诀：**「正异偏同」**——正印/正官/正财=异阴阳，偏印/七杀/偏财=同阴阳；
但**食伤例外**：食神=同阴阳，伤官=异阴阳（食神温和故从同，伤官激烈故从异）。
