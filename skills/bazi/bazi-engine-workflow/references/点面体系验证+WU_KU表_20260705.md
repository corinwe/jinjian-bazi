# 点面体系验证方法论 + WU_KU表双确认

## 一、点面体系验证（老板核心方法论）

**修一个点→延伸到整个面→延伸到整个体系**

### 执行步骤
```
修了Bug X
    │
    ▼
Step 1 — 搜索"点"：Bug X 是否在其他文件中有相同实现？
    Example: step1_basic.py 的十神Bug和shi_shen.py的十神Bug完全一样
    → 搜索所有 .py 文件中的 same_yy 相关条件
    → 搜索所有十神判定逻辑（我生/克我/生我/同我/我克）
    │
    ▼
Step 2 — 搜索"面"：修改的模块被哪些其他模块调用？
    谁调了这个函数？
    谁传参给它？
    谁接收它的输出？
    → 用 search_files 搜索函数名
    │
    ▼
Step 3 — 搜索"体系"：修改涉及的规则在其他事象模块中是否一致？
    Example: 修了财富的冲库逻辑 → 检查置业/灾祸/流年模块是否也用了冲库规则
    → 用跨模块覆盖矩阵检查
    │
    ▼
Step 4 — 用真实八字跑全流程验证
    → 选一个能暴露Bug的八字
    → 跑 pipeline_product.py 或 test_full_suite.py
```

### 验证清单
```
□ 同文件内：修改的函数前后调用是否正确？
□ 跨文件：相同逻辑的其他实现是否也需要改？（⚠️常见陷阱）
□ 上游：调用方是否传了正确的参数？
□ 下游：接收方是否用了正确的输出字段？
□ 体系：这个规则在其他模块中是否使用了同样的变量/表？
```

## 二、WU_KU表 双确认清单

**问题来源**：wealth_v2.py 的 WU_KU 表中，金行和水行各有1个错误，但 bazi-wealth-analysis skill 的描述是正确的。代码未跟进skill的更新。

### WU_KU 表（五行×五种墓库）

| 五行 | 比劫库 | 财库 | 官杀库 | 印库 | 食伤库 |
|:----|:-------|:-----|:-------|:-----|:-------|
| 木 | 未 | 戌 | 丑 | 辰 | 戌 |
| 火 | 戌 | 丑 | 辰 | 未 | 戌 |
| 土 | 戌 | 辰 | 未 | 戌 | 丑 |
| 金 | 丑 | 未 | 戌 | **戌**(非丑) | 辰 |
| 水 | 辰 | 戌 | **戌**(非未) | 丑 | 未 |

### 五行的印库公式
```
金行印库：庚辛金 → 印=戊己土 → 戊己土库=戌 ✅
水行官杀库：壬癸水 → 官杀=戊己土 → 戊己土库=戌 ✅
```

### 每次修改WU_KU表后的检查
```
□ 金印库 = 戌（不是丑）
□ 水官杀库 = 戌（不是未）
□ 木财库 = 戌（木克土为财→土库在戌）
□ 火财库 = 丑（火克金为财→金库在丑）
□ 土财库 = 辰（土克水为财→水库在辰）
□ 金财库 = 未（金克木为财→木库在未）
□ 水财库 = 戌（水克火为财→火库在戌）
```
