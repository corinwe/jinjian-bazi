# 产品架构铁律：规则引擎（确定性代码）+ 网站/API，不是LLM一问一答

> **2026-06-26老板亲自确认 · 来源：本会话对话**
> **老板原话：「你的报告质量还是不行，一会儿稳定一会儿不稳定」「我们是商业化产品，必须任何一个八字进来都能按照标准格式以及实际规则，匹配客户八字的各方面信息，逐一对比得出报告结论」**

## 三种方案对比

| 方案 | 说明 | 稳定性 | 成本 | 状态 |
|:---|:----|:------:|:---:|:----:|
| ❌ LLM一问一答 | 大模型直接写报告 | ⭐⭐ | 0.1-0.5元/次 | **已否决** |
| ✅ 规则引擎+API+前端 | Python代码+FastAPI+SPA | ⭐⭐⭐⭐⭐ | 0.001元/次 | **已确认采用** |
| ⚠️ 规则引擎+LLM Hybrid | 方案2+LLM润色 | ⭐⭐⭐⭐⭐ | 略增 | **未来扩展** |

## 核心铁律

### 1. LLM零推理

```
规则引擎做所有确定性计算（排盘/身强弱/财星/格局/喜用神/大运/8维度）
  → 输出结构化JSON
  → 前端直接渲染
  → LLM仅做可选的润色翻译（JSON→自然语言）
```

**绝对禁止：**
- ❌ LLM直接写完整报告（质量不稳定）
- ❌ Subagent delegate_task出报告（不靠谱）
- ❌ LLM自行计算身强弱/财星分数（会被算错）

### 2. Subagent不可靠原则

老板原话：「他们（subagent）不靠谱」

| 场景 | 可用否 | 说明 |
|:----|:-----:|:-----|
| 确定性代码已完成，仅需JSON→自然语言翻译 | ✅ | 安全 |
| 报告模板已固化，仅需填充数据 | ⚠️ | 注意格式一致 |
| 数据已在context中传递，subagent不需计算 | ✅ | 数据封闭 |
| subagent自行计算八字/分数/大运年份 | ❌ | 绝对禁止 |
| subagent自行对规则做判断 | ❌ | 绝对禁止 |
| subagent查询知识库做推理 | ❌ | 绝对禁止 |

### 3. 产品形式

```
┌─────────────────────────────────────┐
│           用户输入层                  │
│   (网站/小程序/H5/API)              │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│   第一层：规则引擎 (Python代码)       │
│   — 确定性、可审计、零幻觉           │
│   — 成本：0.001元/次                │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│   第二层：数据层                      │
│   — 神煞表/纳音表/藏干表             │
│   — 调候用神表/案例校准库            │
│   — 用户/订单管理                    │
└──────────────┬──────────────────────┘
               ▼
┌─────────────────────────────────────┐
│   第三层：展示层（可选LLM润色）       │
│   — 规则引擎出JSON → 前端直接渲染     │
│   — 或 JSON → LLM润色成自然语言       │
│   — LLM只做翻译，不做推理             │
└─────────────────────────────────────┘
```

## 技术栈建议

- **后端**：Python FastAPI + SQLite（可切换PG）
- **引擎**：18个模块全部Python代码（现有v5.x ~ v10.x）
- **前端**：SPA（微信小程序或Vue/Nuxt网站）
- **LLM**：仅用于可选润色，不做核心推理

## 对应技能与文件

- bazi-engine-workflow — 引擎工作流（含v5.x确定性规则引擎）
- bazi-master-agent — 总调度Agent（含三引擎架构）
- bazi-platform/ — 全栈平台代码（engine/ + backend/ + frontend/）
- 所有bazi-*-analysis 技能 → 每个对应一个规则引擎模块
