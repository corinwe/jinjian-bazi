解决方案：所有确定性数据从 `family_bazi_data_v1.json` 读取，不再重新计算。

## 使用规则

```yaml
Step 1 — 确认数据源存在
  检查两个路径之一是否存在
  ❌ 不存在 → 先跑引擎生成 → 再追加到数据源

Step 2 — 从数据源读取目标人物
  import json
  with open(data_path) as f:
      db = json.load(f)
  person = [p for p in db if p['姓名'] == name][0]

Step 3 — 验证报告中的数据与数据源一致
  □ 报告八字符 == person['八字']?
  □ 报告身强弱分数 == person['身强弱']['总分']?
  □ 报告财星分数 == person['财星']['总分']?
  □ 报告大运序列 == person['大运']['序列'][:N步]?
  □ 报告起运年龄 == person['大运']['起运年龄']?
  ❌ 任一项不通过 → 阻断输出

Step 4 — 新八字确认后，先追加到数据源
  person = {全部字段}
  db.append(person)
  with open(data_path, 'w') as f: json.dump(db, f, ensure_ascii=False, indent=2)

口诀：
  数据源是第一真理，报告数字必须对齐
  新八字先入库再出报告，老八字拉数据不重算
  八字总分大运起运，五项对比不出错
```
