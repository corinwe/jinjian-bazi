# 跨Agent技能对比吸收方法论

> 编制人：金鉴真人
> 编制时间：2026年06月02日 10:45
> 版本：v1.0
> 场景：当获取到其他Agent的同类技能文件时，执行此流程系统化吸收精华

## 五步吸收流程

```
第1步：全量提取 → 读取对方所有文件（SKILL.md + references/），结构化提取每个文件的
  ├─ 核心知识点（3-5句总结）
  ├─ 独特视角（对方有但我没有的）
  └─ 表达方式（口诀/公式/量化表等比我的好的）

第2步：逐项对比 → 每个知识点标记状态：
  ✅已有   → 我已覆盖且不差
  ⚠️部分  → 我有但不完整，可补充
  ❌缺失  → 我没有，需判断价值
  ✅不同  → 体系不同但等价，不冲突

第3步：价值筛选 → 分三档：
  ⭐必吸  → 确实缺失且有实战价值 → 必须补入
  可吸收  → 有启发但不紧急 → 一并补入
  ❌不吸收 → 已有/不同体系/无价值 → 记录原因不补

第4步：精准补入 → 找到SKILL.md对应位置做局部patch
  ├─ 黄金速查区 → 口诀类
  ├─ 对应章节   → 方法/规则类
  └─ 新子节     → 全新知识点

第5步：交叉验证 → 补入后检查：
  ├─ 九项检查清单（口诀/量化/条件/细节/对应/案例/矛盾/汇总表/组合规则）
  ├─ 冲突排查（新旧知识说法是否一致？如有矛盾以实战校准为准）
  └─ 推知识库（git commit + push）
```

## 九项检查清单

```
每次吸收后对照以下9类逐项检查是否有遗漏：
  □ 口诀类     □ 量化类     □ 条件类（二分法）
  □ 细节类     □ 对应类     □ 案例类
  □ 矛盾类     □ 汇总表类   □ 组合规则类

记忆口诀：口诀量化加条件，细节对应和案例，矛盾汇总组合类，九项查完无遗漏
```

## 关键原则

- **内容吸收 ≠ 架构复制**：只吸收知识点内容，不照搬对方架构设计
- **不重复不冲突**：已有知识点不重复添加，宁缺毋滥
- **自有体系优先**：对方好的表达方式可以用，但知识体系用自己的
- **每次吸收后必须推知识库**，形成可追溯的完整记录
