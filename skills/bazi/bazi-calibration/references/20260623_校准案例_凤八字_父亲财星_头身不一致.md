# 2026-06-23 校准案例三则

## 案例一：凤八字手动计算错误

**错误**：公历1978-12-13 21:00 亥时，手动算日柱为庚戌（❌），实际引擎+官网验证为己酉（✅）。
**八字正确应为**：戊午 甲子 己酉 乙亥（己土日主）
**八字错误曾写为**：戊午 甲子 庚戌 丁亥（庚金日主）

**根因**：
1. 日干支公式手动计算出错，天干6→庚，地支10→戌，但正确应为己酉
2. 没有先跑bazi-engine.py排盘就直接手动算日柱
3. 没有跑bazi-zydx-verify.sh官网验证交叉核对
4. **没有走标准流程：engine排盘→官网验证→对比一致才用**

**修复方案**：
创建 `scripts/bazi-must-verify.sh`，将三步合一为一条命令：
```bash
bash bazi-must-verify.sh 1978 12 13 21 0 0 凤
```
输出：戊午 甲子 己酉 乙亥 ✅（引擎+官网一致）

**教训（已固化为master-agent v9.0）**：
任何八字必须先跑must-verify.sh，不通过就拿不到八字，物理上绕不开。

## 案例二：父亲财星计算错误（己土七杀误算为偏财）

**八字**：己丑 癸酉 癸亥 戊午
**日主**：癸水

**错误**：财星计为19.2分（午中丁火偏财12分 + 午中己土中气7.2分）
**正确**：财星应为12分（午中丁火偏财12分，午中己土是七杀不是财）

**根因**：
- 五行相克关系搞混了：**日主克者为财**
- 癸水克火为财（丙丁巳午），不是克土
- 己土克癸水（土克水），所以己土是七杀，不是偏财
- 误将「己土含财字」和「午火中有财」混在一起，认为午中所有藏干都是财

**口诀**：
日主克者为财，不是被克就是财
癸水克火是财星，土来克水是七杀
财星只分正偏财，劫财虽带财字不是财

**修复**：父亲报告财星从19.2分修正为12分，财富等级从中富下调为小富上段。

## 案例三：凤报告「头改身子没改」

**问题**：凤报告v5.0中，§1表头数据已更新为：
- 己土日主 ✅
- 身弱14.4分 ✅  
- 财星52分 ✅
- 小富 ✅

但§2-§21仍然是旧版从弱格辛金日主的分析：
- 「辛金生于仲冬子月」「从弱格成立」❌
- 「财星精确计算26.4分」❌
- 「从弱50分恒定」「原局0分」❌

**根因**：delegate_task子agent更新报告时，context中只要求更新§1表头，没有要求同步更新§2-§21的分析内容。子agent只改了头，身子没动。主agent也没有做后续验证。

**修复方案（已固化为铁律）**：
每次更新报告后，必须执行一致性校验：
```
grep -c "从弱" §2-§21  # 如果报告非从弱格，§2-§21不应出现"从弱"
grep "财星.*分" §1 vs §8  # §1财星分数必须与§8财星分析一致
```
任何不一致 → 阻断推库，全部修正后重试。

**口诀**：
头改身子也要改，§1§2要一致；delegate只改头，主agent要查身子；不一致就阻断，修正完再推库。
