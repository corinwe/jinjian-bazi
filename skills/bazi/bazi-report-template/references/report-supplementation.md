# 报告补入模块 · 补报告标准化流程

> **适用场景**：用户要求对已完成的八字报告补充特定分析内容（"补XX报告：A+B+C"）
> **核心原则**：不重写整份报告，只对现有§做**定向补强**（修楼不拆楼）

---

## 一、补报告 vs 写报告——核心差异

| 维度 | 写报告（从零） | 补报告（加内容） |
|:-----|:--------------|:----------------|
| **入口** | 引擎JSON + 八字数据 | **已有报告文件**（1400-1800行） |
| **操作** | 从头编写21§ | 读全文 → 定补入位置 → 多个surgical patch |
| **风险** | 内容不完整 | **破坏原有结构**（表格错位、yaml缩进、line numbering变化） |
| **技术** | write_file 一次写入 | **patch** 多次精准替换 |
| **风格** | 统一从头写 | **适配原报告风格**（原有术语体系、emoji风格、格式约定） |

---

## 二、补报告的标准化工作流

### Step 1：读报告——理解结构（必须通读全文）

```yaml
操作：
  read_file(path, offset=1, limit=500)   # 读前半
  read_file(path, offset=500, limit=500) # 读中段
  read_file(path, offset=1000)            # 读尾段

关注点：
  ├─ 标题风格（§编号体系 vs 步进体系 vs 自由标题）
  ├─ 已有§结构（哪个§已覆盖目标内容？哪个§是空白的？）
  ├─ 表格格式（换行符号、对齐方式、emoji体系）
  ├─ yaml/代码块开头结尾风格
  └─ 原有断语/口诀体系（是否有固定格式）
```

### Step 2：定补入点——判断追加 vs 替换 vs 新增§

| 补入策略 | 条件 | 操作 |
|:---------|:-----|:-----|
| **替换** | 目标§已存在但内容不足（如§5.4家里家外已有表但无分析） | patch(old_string=原内容, new_string=增强版) |
| **追加** | 目标§下方的空余行（如§14.2④ 财库只有1行） | patch(old_string=原简短行, new_string=展开版) |
| **新增段** | 目标没有§，需要插入现有§之间 | 找相邻段落的document marker如`---`或`\n---`，在marker前插入 |

### Step 3：写内容——适配原报告风格

**三条铁律：**
1. **术语一致**：原报告用"体用比例"就不要写成"体用比值"
2. **emoji一致**：原报告用🔥就别改成🌟，原报告用⭐就别改成❤️
3. **格式一致**：原报告用yaml块就别改用代码块，原报告用`|`表格就别改markdown原生表格

### Step 4：patch写入——只替换目标字符串

```yaml
使用 patch(mode='replace'):
  必须确保 old_string 在原文件中唯一（包含周围context确保）
  如果原内容已移位（之前patch过），需要重新 read_file 确认最新内容
  多个patch可以依次调用（不互相冲突）
```

### Step 5：验证行数

```yaml
操作：wc -l <file>
检查：行数是否按预期增加（计算新增行数 - 被替换行数 = 净增行数）
```

## 三、三个标准补入模块（v2.1·文颖双时辰案例校准）

> **2026-07-16 文颖案例（1407行，双时辰对比报告）新增细化**：
> 本版本根据文颖（甲木日主·午时 vs 未时双时辰对比报告）的完整工作流，大幅丰富了三个模块的实操模板，特别是**双时辰对比**场景下的双轨分析模式。

### 模块A：家里家外做功分析 + 财官量级评估

**补入位置**：§做功分析中的"家里 vs 外面"子§

**标准结构（双时辰对比版）：**

#### A1 家里分析

```
## 家里（日支子）分析：

【表格】项目 × 内容 × 说明
  行1：日支 → 说明核心体用
  行2：日支与邻支互动（冲/害/合/刑）
  行3：日支与他支合化

【家里做功路径】：核心功描述（如"印星贴身生身"）+ 冲害影响

【家里做功效率判定】：低/中/高 + 百分比 + 原因分析
  ├─ 冲（如子午冲）= 内部消耗大，效率最低（20%）
  ├─ 害（如子未害）= 温和消耗，效率中等（40%）
  └─ 无冲无穿 = 效率高（60%+）

【家里内容五要素】：
  ├─ 日支的十神与体用
  ├─ 日支与邻支的刑冲合害（决定效率的核心）
  ├─ 日支藏干与其他地支的暗合/半合
  ├─ 印星是否被干扰（冲害减弱印星生身效果）
  └─ 家里做功占比
```

#### A2 家外分析

```
## 家外（年/月/时）分析：

【表格】项目 × 内容 × 说明
  行1：年柱体用
  行2：月柱体用（最大用神来源）
  行3：时柱体用（与日主互动方式：克/合/生）

【家外做功路径】：
  1. 体生用路径（如丙食神→生土财）
  2. 用克体/用合体路径（时柱官杀 vs 日主）
  3. 恶性循环 vs 良性循环判断：
     ┌─ 体生用→用克体 = 恶性循环（越努力越被压）
     └─ 体生用→用合体 = 良性循环（越努力越被提拔）

【家外做功效率】：中/中高/高 + 百分比 + 原因
```

#### A3 家里家外分配饼图（ASCII）

```
┌─────────────────────┐
│  家里（X%）          │
│  ├─ 核心功描述      │ ← 效率评价
│  └─ 次要互动        │
│                      │
│  家外（Y%）          │
│  ├─ 主要路径 ✓/✗    │ ← 效率评价
│  ├─ 次要路径 ✓/✗    │
│  └─ 问题标注 ⚠️/✅   │
└─────────────────────┘
```

#### A4 财官量级评估

```
【表格】财官类别 × 分值 × 量级评价（⭐1-5级）

财星合计 → 强/超强/弱
官杀合计 → 官杀性质（攻身型/合身型）+ 量级

【财官综合量级】：⭐⭐⭐⭐ 或 ⭐⭐⭐⭐⭐
  ├─ 财官是否都在家外
  ├─ 财生杀/官的性质（压力型 vs 成就型）
  └─ 身弱是否能胜财官
```

#### A5 双时辰对比总表（双时辰报告特供）

```
【表格】对比维度 × 时辰A × 时辰B
  家里内容、家里互动、家里效率、家里问题
  家外内容、家外效率、家外问题
  财星量级、官杀量级、财官性质
```

**关键判断标准（综合）**：
- 家里效率判据：日支冲>害>合>静（冲最低，静最高）
- 家外效率判据：恶性链<良性链（恶性链中低，良性链中高）
- 财官量级判据：结合身弱看"能否胜"—不能胜则量级再高也是压力
- 双时辰版必须每条路径两时并行分析 + 结尾总表

### 模块B：财官库分析（含五库全量）

**补入位置**：财富分析§或空亡墓库§的"库"段落

**标准结构（双时辰对比版）：**

#### B1 财官库核心定义

```
甲木财库 = 辰（土库，戊土正财之库）
甲木官库 = 戌（金库，辛金正官之库）
         = 丑（金库，辛金正官之库，申金七杀之库）

四墓库原理：
  辰 = 土库（财库）→ 木之财库
  戌 = 金库（官库）→ 木之官库（辛金正官）
  丑 = 金库（官库）→ 木之官库（辛/申/庚）
  未 = 木库（比劫库）→ 木之于自己的库，非财官库
```

#### B2 财库分析

```
## 财库：[地支]（[位置]·家中/家外）

【表格】项目 × 内容
  财库位置、属家内/家外、藏干、力量、来源意义

【财库互动】树形图：
  ├── 冲/破/合 关系1 → 效果
  │    ├── 正面影响
  │    └── 负面影响
  ├── 冲/破/合 关系2 → 效果
  └── 体在用中/用被体困

【财库做功特性】：
  ├─ 冲开 vs 干扰：辰戌冲=大开大合，辰未破=稳定有瑕
  ├─ 位置意义（年/月/日/时对应的财富类型）
  └─ 身弱冲库风险（财库反噬）
```

#### B3 官库分析

```
## 官库：[地支]（[位置]）

【表格】项目 × 内容
  官库位置、藏干、官星状态（透干/藏支）

【官库做功特性】：
  ├─ 官透 vs 官藏：透干=官星掌权，藏支=隐而未发
  ├─ 官杀混杂 vs 正官清透（核心差异）
  └─ 官库被冲/刑/害的影响
```

#### B4 综合总评 + 双时辰对比

```
## 综合总评：
财库 ⭐⭐⭐⭐：... 
官库 ⭐⭐⭐⭐：...

## 双时辰财官库对比总表
【表格】对比维度 × 时辰A × 时辰B
  财库位置、财库力量、财库互动、财库特性、财库总评
  官库位置、官库特性、官库总评、财官关系
```

**辰未相破深度分析模板**（双时辰对比特供）：

```
辰未相破 = 土库之间的内部破坏关系
          辰为水库/土库，未为木库/土库
          同为土而相破，主内部矛盾、隐性问题

对财运影响：
├── 财库（辰）被时柱（未）所破
│   ├── 时柱代表晚年、子女、下属
│   ├── 晚年财运有波折
│   └── 但破坏程度 < 辰戌冲，是"小病"而非"大病"
└── 总体：财库稳定但有瑕疵
```

**关键判断标准：**
- 辰戌冲=大开大合（高风险高回报）；辰未破=稳定有瑕（细水长流）
- 官杀混杂 vs 正官清透：决定事业压力的性质
- 透干的官星 = 官库开库 = 官星掌权（比藏干有力）
- 戌的双重属性：火库(食伤库)+土库(财库/印库) → 需根据全局判定

## 四、常见陷阱

| 陷阱 | 表现 | 解决办法 |
|:-----|:-----|:---------|
| **patch冲突** | 第一次patch后行号变化，第二次old_string找不到 | 每次patch后重新read_file确认最新内容 |
| **old_string非唯一** | 多行相同text，patch替换了错误的 | 增加前后context行确保唯一性 |
| **格式错位** | 新内容用了不同的table/wrapping风格 | 严格复制原报告最近的格式 |
| **新增段破坏原有§编号** | 插入了新内容但后续§的引用编号没更新 | 如果§编号是自动的（如1、2、3），需要在备注中说明 |
| **口诀风格不一致** | 原报告用四言诗，新内容用七言 | 严格匹配原报告的口诀格式 |
