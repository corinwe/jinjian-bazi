# 八字分析报告模板 · 详尽版（收费用户）

> **来源**: 本模板是 `bazi-report-template` 的详尽版本，面向收费用户
> **精简版**（免费用户）: `bazi-report-template/SKILL.md`
> **详尽版**（收费用户）: 本文件
> 
> **区别**: 详尽版每个§增加引擎JSON字段的逐个引用、深度推理过程、交叉验证结论、可执行建议

## §0. 版本选择与加载

### 0.1 双版本选择

| 维度 | 精简版（免费） | 详尽版（收费） |
|:----|:--------------|:---------------|
| **用户** | 免费/首次体验 | 付费/VIP |
| **总字数** | 1,500-2,500字 | 4,000-8,000字 |
| **§数量** | 21§（每个80-200字） | 21§（每个200-500字） |
| **引擎数据** | 摘要引用 | 逐个字段引用+数值 |
| **推理过程** | 结论为主 | 结论+推理路径 |
| **建议** | 通用建议 | 精确到年份/月份 |
| **交叉验证** | 无 | 有（不同规则互证） |

### 0.2 SOP调用规则

```yaml
Phase 4.4:
  IF 用户类型 == "免费" → skill_view('bazi-report-template')        # 精简版
  IF 用户类型 == "付费" → skill_view('bazi-report-template-detailed') # 详尽版
```

## §1. 报告生成全流程

### 1.1 详尽版额外流程

```
① 加载引擎JSON（/tmp/{姓名}_engine.json）
② 逐个§提取JSON字段值（详细版需要逐个字段列出）
③ 加载对应技能规则（skill_view）
④ 写分析内容（结论+推理路径+数据引用）
⑤ 写可执行建议（精确到年份的窗口）
⑥ 交叉验证（用不同规则互证同一结论）
⑦ 完成标记 ✅
```

### 1.2 数据引用格式（详尽版特有）

```yaml
每个数据点必须标注来源JSON字段路径：
  - 身强弱分数: engine["sec_3_shen_qiang_ruo"]["score"] = 64
  - 财星等级: engine["sec_8_wealth"]["cai_star_level"] = "Lv4"
  - 大运列表: engine["sec_17_da_yun_detail"]["list"][0]["gan_zhi"] = "甲申"
```
