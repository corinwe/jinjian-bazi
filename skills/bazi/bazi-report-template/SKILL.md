# 八字分析报告模板 · 21§标准

> **来源**: 本模板基于 bazi-paipan-sop §4.3A 和 bazi-foundation-analysis 各§的定义
>
> **双版本体系**: 
> - 本文件 = **精简版**（免费用户 1,500-2,500字）
> - `references/detailed-template.md` = **详尽版**（收费用户 4,000-8,000字）
> - `references/skill-orchestration-for-21s.md` = **技能编排参考**（各§的skill加载顺序+引擎降级策略+行数基准）
- `references/report-supplementation.md` = **报告补入标准流程**（补报告专用：家里家外/财官库/一生做功标准化模块+patch工作流）
> - SOP Phase 4.4 根据用户类型选择加载

### 1.2 引擎JSON → 报告字段映射规则

```
引擎JSON字段路径 → §X.标题
示例:
  engine_output["ge_ju"]["ge_ju_name"] → §3命格与格局·格局名称
  engine_output["shen_qiang_ruo"]["total_score"] → §3命格与格局·身强弱分数
  engine_output["cai_fu"]["cai_star_level"] → §10财富格局·财星等级
```

### 🚨 1.2a 引擎JSON字段名实不一致陷阱（2026-07-16实战教训）

**致命陷阱**：模板文档中的 `sec_*` 前缀字段路径（如 `sec_3_shen_qiang_ruo`）与金鉴真人bazi-engine.py实际输出的JSON结构**完全不同**。实际引擎输出使用**中文key路径**，非英文 `sec_*` 前缀。`references/detailed-template.md` 中的 `sec_*` 字段名为旧版格式，仅供结构参考，不可直接用于数据提取。

| 文档示例路径（❌ 无效） | 实际引擎路径（✅ 正确） |
|:------------------------|:----------------------|
| `sec_3_shen_qiang_ruo.score` | `身强弱.总分` |
| `sec_2_ge_ju.main` | `四柱.年柱` + 十神分析手工推断格局 |
| `sec_8_wealth.cai_star_level` | 引擎无等级字段，需引用wealth skill手动计算 |
| `sec_17_da_yun_detail.list[0].gan_zhi` | `大运.序列[0].干支` |

**实际引擎JSON结构（从198+份报告验证的核心字段）：**
```yaml
基础: engine["姓名"] / ["八字"] / ["日主"] / ["性别"] / ["日主五行"]
身强弱: engine["身强弱"]["总分"] → float(55.6) | ["等级"] → "中和/身强/身弱"
四柱: engine["四柱"]["年柱/月柱/日柱/时柱"] → "辛卯"
藏干: engine["藏干"]["年支/月支/日支/时支"] → [{"天干":"丙","比例":"100%"},...]
十神: engine["十神"]["年干/月干/日干/时干"] → "正财/正官/日主/正官"
纳音: engine["纳音"]["年柱/月柱/日柱/时柱"] → "松柏木"
大运: engine["大运"]["规则/起运/起运年龄"] | ["序列"][0]["干支/起始年龄/终止年龄/起始年份/终止年份"]
```

**强制前置检查：** 每次写报告前，先 `read_file(path)` 确认引擎JSON的真实字段名。确认使用的是**中文key路径**而非 `sec_*` 路径。

### 1.2b 多技能编排顺序（21§报告生成时强制遵守）

生成21§报告时，需要依次加载对应技能并从中提取规则。**加载顺序 ≠ 报告§顺序**——先加载与数字评分相关的技能（身强弱、财富），再加载定性分析技能（性格、婚姻等）。

```yaml
Step 0 — 通用前置（每次必加载）:
  skill_view('bazi-report-template')         # 本模板
  skill_view('bazi-foundation-analysis')      # 基础规则（藏干/十神/纳音/神煞）

Step 1 — 定量数据层（从引擎JSON提取）:
  skill_view('bazi-fortune-analysis')         # §3·格局+§5·大运+§5·起运

Step 2 — 逐§加载对应深度技能:
  §7 性格分析 → 先load bazi-destiny-analysis（五重人格）
  §8 学业运势 → 先load bazi-education-analysis（年柱印+文昌）
  §10 财富格局 → 先load bazi-wealth-analysis（五路财富+墓库）
  §11 婚姻感情 → 先load bazi-marriage-analysis（夫妻星+宫）
  §12 子女缘分 → 先load bazi-children-analysis（官杀/食伤）
  §14 健康调理 → 先load bazi-health-psychology（五行过三）
  §18 灾祸化解 → 先load bazi-misfortune-analysis（神煞+应灾）
  §19-20 开运建议 → 先load bazi-remission-methods（文昌+补库+风水）

Step 3 — 已知缺失字段的降级策略:
  □ 引擎无"格局"字段 → 月令藏干无透干→手工推断「杂气格」+按全局最显著十神命名
  □ 引擎无"财星等级"字段 → 按wealth skill的§2.2五级表手动计算
  □ 引擎无"神煞"字段 → 从年干/日干手工查文昌/天乙/华盖等
```

### 1.3 字段映射三原则

| 原则 | 内容 |
|:----|:------|
| **原则A** | 每个§必须有且仅有一个引擎JSON字段映射 |
| **原则B** | 禁止凭空捏造数据——无对应字段时标注「引擎数据缺失」 |
| **原则C** | 所有评分/数字直接从JSON提取，LLM禁止自行计算 |

## §2. §内容模板规范

### 2.1 通用模板结构

每个§的内容必须遵循以下三段式：

```
【§N 标题】

[引擎数据摘要]
（从JSON提取的关键数据，1-3句）

[分析内容]
（基于技能规则的分析解读，每个结论必须有证据链：
  规则来源行号 → 八字数据 → 结论）

[提醒/建议/补充]
（针对该§的实用建议）
```

### 2.2 复杂§的子结构

#### §3 命格与格局
```
- 日主五行属性
- 身强弱判定（分数+状态）
- 格局名称+成格条件验证
- 喜用神+忌神
- 综合评价
```

#### §6 流年溯源
```
- 当前流年（2026丙午）各柱分析
- 年柱对年柱影响
- 月柱对流年影响
- 日柱对流年影响
- 时柱对流年影响
- 流年与大运互动
```

#### §10 财富格局
```
- 财星等级（6级）
- 财星位于四柱
- 身强/身弱 vs 财旺/财弱 匹配度
- 财星类型（正财/偏财比例）
- 财星与其他十神互动
- 财富建议
```

### 2.3 禁止使用的模板话术

以下话术**禁止**出现在报告中（每次出报告前检查）：

```
❌ "您的八字显示..."
❌ "根据五行理论..."
❌ "从命理学角度..."
❌ "俗话说..."
❌ "古人云..."
❌ "总体来看..."
❌ "乾坤流转..."
❌ "命运多舛..."
❌ 任何占位符（[姓名]、[年龄]、[具体分析]）
❌ 任何玄学黑话堆砌（太极贵人在哪、孤辰寡宿什么的没有数据支撑不能说）
```

### 2.4 必须使用的话术风格

```
✅ 直接陈述：            "癸水日主生于丑月..."
✅ 数据先行：            "引擎评分64分，身强..."
✅ 证据链完整：          "丑中己土本气克身→月令0分"
✅ 结论具体：            "财星等级Lv4，正偏财比3:1"
✅ 建议可执行：           "流年丙午财星透干，宜主动开源"
```

## §2.5 通用生成规则

### 2.5.1 年龄向上取整

```
年龄计算 = ceil(当前年 - 出生年)
禁止显示小数年龄（如"4.7岁"→"5岁"）
特例：起运年龄保留月精度（如"4岁8个月"）
```

### 2.5.2 大运只列8步至80岁

```
规则：从起运开始最多列8步大运，每步10年
示例：起运4岁 → 4~14岁 / 14~24岁 / ... / 74~84岁（刚好到80+）
起点：起运年龄向上取整到整数
终点：第8步大运结束（覆盖至80岁以上）
```

### 2.5.3 双模式选择

| 模式 | 覆盖§ | 适用场景 |
|:----|:------|:---------|
| **完整版21§** | §1-§21全部 | 正式分析报告（默认） |
| **精简版7板块** | 排盘+格局+事业财运+婚姻+健康+流年+总结 | 快速参考/朋友圈 |

### 2.5.4 4项必含专项内容

每个报告中以下4项必须有内容（§25字段映射保证数据完整性）：

```
① 性格分析（§7）— 五重人格至少3维
② 补财库方案（§20）— 可操作的具体建议
③ 文昌方案（§8学业）— 若有求学子女
④ 婚姻子女事件表（§11+§12）— 关键时间点
```

## §4. 格式标准

### 4.1 排盘格式（§2）

```
八字排盘

乾造：     年      月      日      时
天干：     乙      己      癸      戊
地支：     卯      丑      未      午
藏干：     乙    己癸辛  己丁乙   丁己
十神:    食神    七杀    日主    正官
纳音：   大溪水  霹雳火  杨柳木  天上火

大运: 戊子 丁亥 丙戌 乙酉 甲申 癸未 壬午
起运: 4岁8个月
```

### 4.2 评分展示格式

```
身强弱: 64分 · 身强
格局: 七杀格
喜用神: 金(印) > 水(比劫)
忌神: 土(官杀) > 火(财)
```

### 4.3 段间距和排版

- 每个§之间 **空1行**
- 子标题使用 `###` 三级标题
- 关键数据使用 **加粗**
- 列表项使用 `-` 短横线（不要用*）
- 禁止使用表格（除排盘外）——移动端兼容性差

## §5. 输出格式

### 5.1 完整报告JSON Schema

```json
{
  "report_id": "str",
  "generated_at": "ISO datetime",
  "basic_info": {
    "name": "str",
    "gender": "str",
    "birthday": "str",

  },
  "sections": [
    {
      "section": 1,
      "title": "命主基本信息",
      "content": "str"
    },
    ...
    {
      "section": 21,
      "title": "综合总结",
      "content": "str"
    }
  ],

    "shen_qiang_ruo": {...},
```

### 5.2 输出要求

- 每个§的内容必须是**完整段落**（非要点列表）

- 禁止出现 `[数据缺失]` 占位符——无数据则跳过该§
- 不要使用 `§` 符号本身作为章节标题显示

## §6. 证据链检查清单

每次使用此模板生成报告后，检查：

- [ ] 每个§的内容是否有引擎JSON数据支撑
- [ ] 所有数字评分是否从JSON直接提取
- [ ] 是否有任何「通用占位符」或「模板话术」
- [ ] §3格局判定是否包含成格条件+身强弱验证
- [ ] §10财富分析是否包含财星等级+身财匹配度

- [ ] 总行数是否≥600行（通过 `wc -l <file>` 验证）
- [ ] 总字数是否在2800-5000字范围内（字数和行数不直接相等，但607行≈46KB为达标基准）

- [ ] 段落格式是否移动端友好
- [ ] 最终输出是否为有效的JSON格式
