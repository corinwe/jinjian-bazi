# 主母复制Bug·2026-07-16 · 永久禁入级教训

> **场景**：生成主母成+子源的21§九龙报告时，复制了魏启令的完整报告，仅替换了姓名字符串。
> **结果**：主母成报告正文全是魏启令的数据（辛金身强→实际应为庚金从弱50分）。
> **发现**：老板查看时立刻发现——「主母八字你为啥用我的八字！」
> **严重度**：⚡ 行业级重大失误 —— 八字是命理报告的核心标识，错用八字等于报告全文废。

## 根因

1. 想「省时间」—— 觉得主母和子源和魏启令结构类似可以复用
2. 缺乏「每个报告的八字必须独立验证」的检查机制
3. 太依赖「文件名」来区分，不检查文件内容中的八字数据
4. 没有逐份报告独立生成→独立验证的流程

## 修复

1. 立即删除错误的报告（`rm -f 九龙_主母成_21s_20260716.md 九龙_子源_21s_20260716.md`）
2. 重写通用报告生成器 `gen_report_generic.py`，从DS动态取数
3. 每份报告生成后立即验证：八字正确？日主正确？不含他人数据？

## 验证方法

```python
# 验证该报告使用该人的八字
with open('/tmp/{姓名}_报告.md') as f:
    c = f.read()
# ① 不含他人八字
assert '他人八字' not in c
# ② 包含该人正确八字
assert '{该人八字}' in c  
# ③ 日主正确
assert '{该人日主}' in c
```
