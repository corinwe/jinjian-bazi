# 八字数据源架构（老板校准版·2026-07-16）

## 核心原则

1. **唯一数据源** — 所有报告模块从同一份 `/tmp/{姓名}_datasource.json` 取数
2. **物理约束** — BAZI_DATASOURCE环境变量 + pre_tool_call hook + pre_llm_call auto-set
3. **不分家** — 盲派和九龙体系共享同一份数据源，只是规则不同

## 🚨 数据源字段命名约定（2026-07-16致命教训）

**字段访问只能用字段名，禁止用数组索引：**

```
✅ 正确：DS['年干'], DS['月支'], DS['身强弱']['等级']
❌ 错误：B8[0], B8[2], B8[4], B8[6]  — 索引顺序容易搞混
❌ 错误：DS['八字段'][0]  — 字段名"八字段"不存在（应为"8字段"）
```

**已验证的优先字段路径：**
```
├─ DS['年干']              → 年干天干
├─ DS['月支']              → 月支地支
├─ DS['日主']              → 日主
├─ DS['身强弱']['总分']    → 身强弱分数
├─ DS['身强弱']['等级']    → 身强弱标签(身强/中和/身弱/从弱)
├─ DS['藏干十神']['年支']  → 年支所有藏干十神
├─ DS['十神']['月']        → 月干十神
└─ DS['大运'][0]['干支']   → 第一步大运干支
```

**根因**：8字段列表是[年干,月干,日干,时干,年支,月支,日支,时支]
错误索引 `G=[B8[0],B8[2],B8[4],B8[6]]` 取成了[年干,日干,年支,日支]→月令变成时干
正确：`G=B8[0:4]`

## 物理约束链（必须按顺序）

```
① pre_llm_call (inject-context.sh) → 自动检测并设置BAZI_DATASOURCE
② pre_tool_call (precheck.sh→precheck.py) → 写文件前检查:
   a) BAZI_DATASOURCE已设置？
   b) /tmp/.bazi_verified 已touch？
   c) 任一不满足 → BLOCK
③ report-generator.py → 所有模块函数签名 def module(DS)
   没有DS参数 → 拒绝执行
④ post_tool_call / git pre-commit → 报告中数字 vs DS对齐验证
```

## 大运规则

- 只出8步大运（到80岁），不出10步
- 顺逆由年干阴阳+性别决定（非日干）：
  - 年干阳男/阴女 → 顺排
  - 年干阴男/阳女 → 逆排
- 大运真假：大运天干在原局藏干（本气/中气/余气）存在即为真
  - 藏干余气(30%)也算"有"——申中戊余气=大运戊为真

## 体用铁律

- 体 = 日主 + 印 + 比劫 + 食伤（食神+伤官全算体）
- 用 = 财 + 官（正财偏财正官七杀，藏干财官一律算用）
- 天干财官（年干8分/月时干12分）必须计入用
