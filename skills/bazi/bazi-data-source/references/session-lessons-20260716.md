# Session Lessons — 2026-07-16 全量Bug修复记录

> 本次session共修复5个Bug，全部由老板亲自指出。每个Bug都反映了系统缺乏物理约束的问题。

## Bug① — 十神阴阳规则倒置（最严重的Bug）

**发现时间**：老板逐句检查子源报告时指出"年支乙木对丙火应为正印，不是偏印"
**根因**：`calc_shishen()` 中规则写反了——同阴阳应为偏印/七杀，代码却写了异阴阳
**影响范围**：全部6份数据源的藏干十神全部算错
**修复**：`bazi-data-source.py`中两个条件反转
**教训**：十神计算是八字命理的基本功，不应该错。错在没有验证用例。

## Bug② — 8字段索引错误

**发现时间**：主母报告显示"生于壬月"（应为未月）
**根因**：`G = [B8[0], B8[2], B8[4], B8[6]]` → 取成了[年干,日干,年支,日支] ❌
**正确**：`G = B8[0:4]; Z = B8[4:8]`
**影响范围**：所有使用这个索引的报告，报告的天干地支数据全乱
**教训**：索引切片是基础操作不应该错。应该用断言验证 `Z[1] == 月支`

## Bug③ — 复制模板用错八字

**发现时间**：老板比较主母报告发现数据全是魏启令的
**根因**：生成多份报告时复制了魏启令的模板只改名字没换数据
**教训**：每份报告必须从各自数据源独立生成，永远不能复制

## Bug④ — 从弱→事业分析误判"中和"

**发现时间**：老板检查主母报告发现从弱被写成了中和
**根因**：条件分支只覆盖`身强`和`身弱`，`从弱`漏了，掉入else→中和
**教训**：条件分支必须覆盖所有可能值

## Bug⑤ — 天干财官漏算

**发现时间**：体用分析时子源用只算了14.4分（仅藏干财）
**正确**：天干财官32分+藏干财官14.4分=46.4分
**教训**：体用穷举必须同时列出天干和藏干成分
