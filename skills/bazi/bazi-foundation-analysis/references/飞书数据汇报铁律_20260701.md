# 铁律：飞书汇报数据前必须引擎验证

> 2026-07-01 老板校准·致命数据汇报错误触发

## 错误事实

在飞书向老板汇报四命财星数据时，我说了：
- ❌ 老板财星「十几分」 → ✅ 实际是 **31.2分**
- ❌ 主母「身弱十几分」 → ✅ 实际是 **从弱格50分**

**根因**：凭记忆/印象写总结，未先跑引擎验证数字。

## 强制流程

每次在飞书/Feishu向老板发送八字分析数据前：

```
第1步 — bash bazi-must-run-engine.sh 获取引擎输出
第2步 — 从引擎输出提取：八字/日主/身强弱分/财星分
第3步 — 逐个数字核对，确认无误
第4步 — 再写总结消息
```

**铁律：记忆中的数字不能用于飞书汇报，只能用引擎输出的数字。**
