# 盲派体用定义修正历史（2026-07-16 老板三次校准）

> 🚨 **本文件记录体用定义的修正过程，防止未来子代理重复犯错。**

## 三次迭代

### 迭代①（❌错）
**体 = 日主 + 印 + 比劫**
**用 = 财 + 官 + 食伤**
错误原因：食神和伤官被归入「用」，但盲派定义食伤是才华/技术（工具包），属于体。

### 迭代②（❌仍错）
**体 = 日主 + 印 + 比劫 + 食神**（缺伤官）
**用 = 财 + 官 + 伤官**（伤官仍在用中）
错误原因：食神归体了但伤官没归。泉师兄原典「五大制用结构」中有「伤官去官」=伤官(体)去制官(用)，证明伤官在体。

### 迭代③（✅正确·锁定）
**体 = 日主 + 印 + 比劫 + 食伤（食神+伤官全算体）**
**用 = 财 + 官（正财偏财正官七杀，藏干财官一律算用）**

## 后续衍生错误

### 错误A：藏干财官不算用（子代理发明）
**现象**：子代理在子源报告中写入「藏干财星不算入用」「用=天干透出的财官」
**根因**：context中体用定义写了但没强调藏干算用，子代理自行「优化」
**修复**：§3A.9铁律①「藏干财官一律算用，禁止排除」
**口诀**：藏干财官全算用，天干藏干一样重

### 错误B：藏干余气不算用（子代理发明）
**现象**：立报告中写「巳中庚不算——藏干余气不单独计入用时之能量」
**根因**：context中写了错误指令「Notice:巳中庚金不在用中」
**修复**：§3A.9铁律①+撤消错误上下文
**口诀**：余气中气都算用，申中一庚也是官

## 锁定机制

1. §3A.9铁律①：藏干财官算用，不可排除
2. §3A.9铁律②：子代理不准发明规则
3. Phase 4.1引擎检查：ENGINE['藏干']存在方可写报告
4. Phase 5.1数据对齐：报告中数字必须能在ENGINE中找到来源
5. verify-format.py自动扫描「不算用」违规模式
