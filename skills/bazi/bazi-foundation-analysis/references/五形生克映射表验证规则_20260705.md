# 手动五行映射表双验证规则（2026-07-05 引擎CAI_WUXING_MAP Bug教训沉淀）

## 问题回顾

2026-07-05 引擎审计发现 `liu_nian_v2.py` 第931行的 `CAI_WUXING_MAP` 将「金日主」的财星映射为「火」—— 正确值应为「木」。

```python
# ❌ 错误（已修复）
CAI_WUXING_MAP = {"木": "土", "火": "金", "土": "水", "金": "火", "水": "火"}
# ✅ 正确
CAI_WUXING_MAP = {"木": "土", "火": "金", "土": "水", "金": "木", "水": "火"}
```

## 根因

手动编写五行生克映射表时，`金→木` 这条规则被写错为 `金→火`。可能是在心智模型中将「火克金」反向关联为「金→火」，而正确应为「金克木→财=木」。

## 双验证规则

**所有手动编写的五行映射表（财星/官杀/食伤/印星等）必须通过以下两关验证：**

### 第一关：口诀验证

用五行生克口诀逐条检查：

```python
# 我克者=财（我克制的五行）
口诀：金克木 → 财=木
口诀：木克土 → 财=土
口诀：土克水 → 财=水
口诀：水克火 → 财=火
口诀：火克金 → 财=金

# 克我者=官杀（克制我的五行）
口诀：火克金 → 官杀=火
口诀：金克木 → 官杀=金
...

# 生我者=印（生我的五行）
口诀：土生金 → 印=土
口诀：金生水 → 印=金
...

# 我生者=食伤（我生的五行）
口诀：金生水 → 食伤=水
口诀：水生木 → 食伤=木
...
```

### 第二关：对称性验证

映射表必须是**双射**（一一对应）：

```python
# CAI_WUXING_MAP 验证：
assert CAI_WUXING_MAP["木"] == "土"  # 木克土
assert CAI_WUXING_MAP["火"] == "金"  # 火克金
assert CAI_WUXING_MAP["土"] == "水"  # 土克水
assert CAI_WUXING_MAP["金"] == "木"  # 金克木 ✅
assert CAI_WUXING_MAP["水"] == "火"  # 水克火
```

### 标准五行生克验算表

| 日主五行 | 我克（财） | 克我（官杀） | 生我（印） | 我生（食伤） | 同我（比劫） |
|:--------:|:----------:|:------------:|:----------:|:------------:|:------------:|
| 木 | 土 | 金 | 水 | 火 | 木 |
| 火 | 金 | 水 | 木 | 土 | 火 |
| 土 | 水 | 木 | 火 | 金 | 土 |
| 金 | **木** | **火** | **土** | **水** | **金** |
| 水 | 火 | 土 | 金 | 木 | 水 |

**特别关注「金」这一行**（最容易出错）：
- 💰 金克木 → 金日主的财 = 木（不是火！）
- 🗡️ 火克金 → 金日主的官杀 = 火
- 🌱 土生金 → 金日主的印 = 土
- 💧 金生水 → 金日主的食伤 = 水

## 检查清单

当在任何引擎模块中添加/修改五行映射表时：

- [ ] 口诀验证：每一条都用生克口诀查过
- [ ] 对称性验证：五行之间形成闭合环
- [ ] 特别检查「金」行：金克木，不是金被火克
- [ ] 用真实八字跑一次输出：确认输出结果与预期一致
