# 新课精读吸收 → 全量知识提取 → 技能整合 → 冲突审计 完整流程

> 用于从老师讲课的逐字记录（docx原文）中提取所有通用知识规则，整合至skill体系，归档知识库，检查与已有体系的冲突。

## 🚨 核心原则（老板亲授·永不可违）

### 铁律一：完整性高于一切（「千万别漏」）
```
❌ 挑重点提取 → 会漏，老板不接受
✅ 全量提取 → 宁可多不可少，每条通用规则都必须记录
✅ 至少20-30条起，实际常有30-50条
✅ 提取后用数字确认：共提取XX条，覆盖范围包括...
```

### 铁律二：提取「通用普适性规则」，不是原文复述
```
✅ 「元辰查法：阳男阴女子见未、丑见申…」→ 通用规则
✅ 「灾煞查法：三合中神所冲之字即为灾煞」→ 通用规则
❌ 「老师说有个命主怎么怎么了…」→ 案例不重要，规则重要
```

### 铁律三：必须进行冲突审计
```
每次整合新知识后，必须对照已有SKILL.md全文检查：
  ① 新规则是否与已有规则矛盾？
  ② 新规则是否过于简化而取代了已有精细体系？
  ③ 跨领域知识（姓名学/风水）是否被误用于命理判断？
  
如有冲突 → 加注⚠️说明，以已有精细体系为准
```

## 前置条件

- 原文docx文件（位于 cache/documents/ 下）
- 已建立的 skill 体系（bazi-fortune-analysis SKILL.md + references/）
- 已建立的 knowledge base（corinwe/weiwuji-knowledge-base）

## 流程总览（7步）

```
docx原文 → ①读取提取 → ②并行全面萃取 → ③核对补漏
→ ④整合至skill → ⑤冲突审计+加注 → ⑥归档知识库 → ⑦Git推送
```

## 第一步：docx读取 + 文本提取

用Python读取docx，提取文本保存到 cache/lectures/ 下。
验证指标：原文通常含 "说话人 1 00:00:00" 时间戳，字数通常在2万-8万。

## 第二步：并行全面萃取

使用 delegate_task 并行处理每个文档。要求：
- 每条以「✅ 规则：」开头
- 至少20-25条（实际常达30-50条）
- 按主题归类和命名

## 第三步：核对补漏（千万别漏）

检查清单：
□ 提取条数是否≥20条？
□ 每条是否以「✅ 规则：」开头？
□ 是否覆盖了所有明显的知识板块？
□ 规则是否通用、可复用到其他八字？

## 第四步：整合至skill

| 规则类型 | 归属位置 |
|:---------|:---------|
| 八字命理通用规则 | 附录J 实战补充 |
| 神煞体系（元辰/灾煞/天罗地网） | 附录H 牢狱之灾断事体系 |
| 空亡相关 | 附录I 空亡新解 |
| 风水砂水 | 附录K 风水砂水体系 |
| 姓名学 | 附录L 姓名学实战小妙招 |
| 从业经验 | 附录M 命理师从业五步法 |

整合方式：使用 skill_manage(action='patch') 修改 SKILL.md

## 第五步：冲突审计（强制步骤）

检查清单：
```
🔴 能量倍数体系是否一致？（六合/三合/三会/自刑/三刑/冲）
🔴 身强弱判定标准是否一致？（40/60分界线不可变）
🔴 从弱格规则是否冲突？（简化说法必须加注裁决说明）
🔴 跨领域知识是否被误用？（姓名学不可取代八字法）
🔴 印星规则、用神规则是否一致？
```

## 冲突裁决六原则

```yaml
原则① 精细 > 简化：100/60/30藏干评分 > "只看本气"
原则② 已有 > 新增：已有覆盖的作为补充而非替代
原则③ 实战 > 教学：教学版简化不可取代实战版
原则④ 体系独立：风水/姓名学独立体系不混入命理
原则⑤ 加注不删改：冲突时加⚠️说明，不删除新知识
原则⑥ 通知不沉默：发现冲突主动报告
```

## 第六步：知识库归档

原文文件归档到：07-国学哲学/八字命格/01-理论体系/金鉴课程/YYYY-MM-DD/

## 第七步：Git推送

```bash
cd /root/weiwuji-knowledge-base
git add -A && git commit -m "YYYY-MM-DD N篇新课吸收" && git push
```
