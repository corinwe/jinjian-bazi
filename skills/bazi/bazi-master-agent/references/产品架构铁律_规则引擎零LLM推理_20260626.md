# 产品架构铁律：规则引擎零LLM推理（2026-06-26老板确认）

> **老板原话：「你的报告质量还是不行，一会儿稳定一会儿不稳定」**
> **老板原话：「我们是商业化产品，必须任何一个八字进来都能按照标准格式以及实际规则，匹配客户八字的各方面信息，逐一对比得出报告结论」**
> **老板结论：规则引擎（确定性代码）+ 网站/API，不是LLM一问一答**

## 核心铁律

### LLM零推理
- 规则引擎做所有确定性计算（排盘/身强弱/财星/格局/喜用神/大运/8维度/21§）
- LLM仅做可选润色翻译（JSON→自然语言）
- ❌ 禁止LLM自行计算任何命理数据
- ❌ 禁止LLM直接写完整报告
- ❌ 禁止subagent delegate_task出报告

### Subagent不可靠（老板原话）
- ✅ 仅当数据在context中封闭传递，subagent只做翻译时可用
- ❌ 禁止subagent自行计算八字/分数/大运年份
- ❌ 禁止subagent自行对规则做判断

### 产品形式
```
用户输入 → 规则引擎(Python代码) → 结构化JSON → 前端渲染
                                              → (可选)LLM润色
```
