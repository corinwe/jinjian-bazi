# 外部文章/公众号知识提取与技能整合五步法

> 编制人：金鉴真人
> 编制时间：2026-06-25
> 版本：v1.0

## 核心原则

每一句断言式的话都是一个可沉淀的知识点——不走马观花，不概括提炼，逐字逐句过，每一句都可能是一条规则。

## 五步流程

### Step ① — 全文提取
获取文章完整内容 → 确认完整性 → 保存到 /tmp/ 待处理

### Step ② — 逐句遍历，全量记录（🚨 核心步骤）

**⛔ 禁止：比对式阅读（先对比已有体系，只记缺失的）**
**✅ 正确：逐字逐句精读（每句话都当规则记录，先全记再分类）**

判断标准（每句话都要问）：
- 叙述性文字 → 记录为一条规则
- 断言式判断 → 记录为一条规则
- 条件式规则 → 记录为一条规则
- 对比性语句 → 至少两条规则
- 案例/例子 → 记录为一条规则
- 否定式语句 → 特别重要！记录为一条规则
- 总结性口诀 → 记录为一条规则

口诀：每个句子都是规则，逐句记录不筛选；先全记来再分类，不是对比找缺失。

### Step ③ — 逐条比对已有体系

上一步完成的「全部规则清单」→ 逐条对比现有SKILL.md：
- ✅ 完全一致 → 标记「已有」
- ⚠️ 部分覆盖但表述不同 → 标记「需细化」→ 补充到现有章节
- 🆕 完全不存在 → 标记「新增」→ 写入对应skill的对应§章节

⚠️ 已有规则也要记录——它们代表「被验证可保留」的信号。

### Step ④ — 沉淀入skill

按domain写入对应skill：
- 基础十神规则 → bazi-foundation-analysis
- 子女相关 → bazi-children-analysis
- 灾祸相关 → bazi-misfortune-analysis
- 事业相关 → bazi-career-analysis
- 风水/化解 → bazi-remission-methods

每个新增规则必须标注来源「公众号原文·实战待验」

### Step ⑤ — 推库

git add/commit/push + 同步技能包到知识库

## 与旧版「防遗漏地毯式扫描」的关系

旧版：先对比→找遗漏→补缺（**比对法·已废弃**）
新版：先全记→再分类→补缺（**逐字逐句法·当前标准**）

**核心变更：禁止跳过已有规则直接找缺失！**
