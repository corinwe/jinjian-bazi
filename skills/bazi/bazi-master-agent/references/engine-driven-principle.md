# 引擎驱动原则（2026-07-16 老板校准）

## 核心原则

> **引擎JSON只加载一次，所有章节从此派生，无二次计算。**

## 工作流程

```
第1步 — 确认引擎JSON存在
  路径：/tmp/{姓名}_{时辰}_engine.json
  不存在→调python3 bazi-engine.py {年} {月} {日} {时} {分} {性别} {姓名} --json > /tmp/{name}_engine.json

第2步 — 一次性加载
  ENGINE = json.load(open('/tmp/{name}_engine.json'))
  BAZI = ENGINE['八字']
  RIZHU = ENGINE['日主']
  SHEN = ENGINE['身强弱']['总分']
  CANG = ENGINE['藏干']   # {'年支':[...], '月支':[...], '日支':[...], '时支':[...]}
  SHISHEN = ENGINE['十神']  # {'年干':'劫财','月干':'食神','日干':'日主','时干':'比肩'}
  DAYUN = ENGINE['大运']['序列']  # [{序号,干支,起始年龄,终止年龄,...}, ...]
  NAYIN = ENGINE['纳音']   # {'年柱':'石榴木',...}
  ZHUS = ENGINE['四柱']    # {'年柱':'庚申','月柱':'癸未','日柱':'辛亥','时柱':'辛卯'}

第3步 — 所有章节只读ENGINE变量（禁止重新计算）
  报告中每个数字标记来源，如：
  「身强弱{SHEN}分 → {SHEN_LV}（ENGINE['身强弱']）」
  「卯乙偏财12分（ENGINE['藏干']['时支'][0]）」

第4步 — 写完后校验（Phase 5.1）
  □ 报告中身强弱得分=ENGINE['身强弱']['总分']？
  □ 报告中大运年龄=ENGINE['大运']['序列'][i]['起始年龄']？
  □ 报告中引用的藏干在ENGINE['藏干']中存在？
```

## 为什么必须这样做

| 错误 | 根因 | 引擎驱动后 |
|:-----|:------|:----------|
| 体用算术错误(12+7.2+12+24=40.8) | 手算不校验 | ENGINE数据一次性算好，无从手算 |
| 藏干财不算用 | 子代理自行发明 | 藏干从ENGINE读取，分类已定 |
| 大运真假误判 | 不查藏干 | ENGINE['藏干']里有所有藏干，自动检 |
