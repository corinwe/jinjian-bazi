# 双模式工作流：引擎模式 vs 手工模式（2026-06-27新增）

> **用户2026-06-27指令**：「不要用你写的那个engine...你就直接按照之前原始的方式，用sub agent去写报告」
> **注意**：同一位用户此前也说过「千万别让sub agent干」，两种模式都是用户在不同场景下的需求。核心是**按用户当前指令切换**。

## 模式选择规则

```yaml
模式A — 引擎模式（默认·批量/快速）
  触发条件：用户没有明确说「不要用引擎」
  流程：gate.py → engine.py排盘评分 → narrative函数润色 → report输出
  优点：确定性、可重复、零幻觉
  适用：API产品、前端展示、快速批量

模式B — 手工Deep模式（用户明确要求时）
  触发条件：用户说「不要用engine」「用sub agent」「原始方式写报告」
  流程：准备数据 → delegate_task子agent写21§ → 验证 → 推库
  优点：深度高（≥1700行）、可个性化、灵活
  适用：知识库深度报告、用户要求深度分析时

🚨 切换规则：
  ✅ 每次用户要求写报告时，先判断用户说的是哪种模式
  ✅ 如果用户没说「不要用engine」→ 默认引擎模式
  ✅ 如果用户说了「不要用engine」→ 切换到手工Deep模式
  ❌ 禁止：用户说「不要用engine」时仍坚持用引擎
  ❌ 禁止：用户没说不让用引擎时，擅自切换到手写模式（耗时10x）
```

## 手工Deep模式委托context模板

```yaml
【context必须包含的字段】

姓名：{姓名}
性别：{男/女}（{顺逆排}·{配偶星}）
出生：公历{YYYY-MM-DD HH:MM}（农历描述）
八字：{年柱} {月柱} {日柱} {时柱}
日主：{五行}（{阴阳}）
纳音：{4个纳音}
空亡：{日柱空亡}

【核心命理】
身强弱：{分数}分 {等级}（{计分说明}）
从弱排查：{✅/❌} {原因}
格局：{格局名}（{判定条件}）
喜用神：{五行排序}
忌神：{五行排序}
调候：{说明}
财星：{分数}分（{明细}）
财富等级：{等级}
文昌：{说明}
学历：{说明}

【大运序列】
{顺排/逆排} 起运{年龄}岁≈{年份}
1. {干支}({年份}){十神} {评分}
...（全部8步）

【输出要求】
- 21§全覆盖（§1-§21）
- ≥1700行
- 保存到：{知识库路径}
- §16事件表≥70行
- 附录含完整数据支持
- 文昌规则：出生≥2001→§11.5+§21.7强制含文昌
- 学历保守判断
- 不使用engine JSON输出

【⚠️ 文件命名铁律——子agent必须用 _NEW_ 后缀】
- 子agent生成的文件名格式：`{姓名}_完整八字命理深析报告_vX.0_标准格式_NEW_20260627.md`
- 手动写的文件用 `_ALLNEW_` 后缀以区分来源
- 父agent在搜索子agent输出时，必须同时搜索 `_NEW_` 和 `_ALLNEW_` 两种模式
- 坑：直接搜版本号（如 v24/v27）可能找不到子agent文件
- 口诀：子agent写 `_NEW_`，自己写 `_ALLNEW_`，找文件两种都要搜
```

## 双模式对比

| 维度 | 引擎模式 | 手工Deep模式 |
|:----|:--------|:-------------|
| 行数 | ~380行 | 1700-2200行 |
| 时间 | <1分钟 | ~5分钟/份 |
| 可靠性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| 幻觉风险 | 零 | 取决于子agent |
| 适用场景 | API/前端/批量 | 知识库存档/深度报告 |
| 用户偏好 | 默认 | 用户明确要求时 |
| 批次并行 | ✓ | ✓（最多4批） |
