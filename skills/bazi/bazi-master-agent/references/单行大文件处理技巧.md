# 单行大文件处理技巧（2026-06-05踩坑记录）

> 适用于AI语音转写文档（如.docx提取的.txt文件），这类文件常表现为超大单行文本（22K~54K字符），read_file按行读取无法分页。

## 问题现象

- 文件 `total_lines=0`（实际是单行，行计数失灵）
- `read_file` 只返回前~100K字符
- `search_files` 返回截断后的结果
- 实际上整个文件是一个巨大的单行文本

## 解决方案

### 方案A：用 patch 的 replace_all 模式插入换行符

```python
import zipfile, xml.etree.ElementTree as ET

# 第一步：从docx提取txt（如果只有docx）
with zipfile.ZipFile(f) as z:
    tree = ET.parse(z.open('word/document.xml'))
    texts = []
    for t in tree.iter('{http://schemas.openxmlformats.org/wordprocessingml/2006/main}t'):
        if t.text:
            texts.append(t.text)
    content = ''.join(texts)
    with open(output_name, 'w', encoding='utf-8') as out:
        out.write(content)
```

关键：docx提取后，用 `patch` 的 `replace_all` 模式在时间标记前插入换行：

```yaml
# 在工具调用中使用：
old_string: '说话人 1'
new_string: '\n说话人 1'
replace_all: true
```

这样就把单行文件按说话人时间戳拆分成多行，然后就可以用 `read_file` 正常分页读取了。

### 方案B：用Python直接在终端拆分

```bash
python3 -c "
with open('input.txt', 'r') as f:
    content = f.read()
content = content.replace('说话人 1', '\n说话人 1')
with open('output_split.txt', 'w') as f:
    f.write(content)
"
```

## 用于搜索关键词确认内容存在

当文件太大无法完整读取时，用 `search_files` 的 count 模式确认关键词存在：

```yaml
search_files:
  pattern: '目标关键词'
  path: /path/to/file
  # 返回匹配行数和上下文片段
```

## 经验总结

1. 转写文档（docx→txt）通常有50000+字符，不要期望一次性读完
2. 先用 `search_files` 确认关键知识点存在
3. 再用 `patch` 拆分后逐段读取
4. 子代理使用 `delegate_task` 并行处理多份文档时，每个子代理独立处理自己的拆分
