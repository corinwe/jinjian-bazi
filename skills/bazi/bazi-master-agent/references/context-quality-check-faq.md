# 派发子代理context质量检查（2026-07-16新增·藏干财官教训触发）

> **本场教训**：我在context中写了「巳中庚不在用中」的错误指令，子代理忠诚执行后产生了9处理论违规。
> 这暴露了系统漏洞：**Master-Agent传错context → 子代理忠实地写错报告 → 无人校验**

## 铁律：派发前context质量检查五步

每次用 delegate_task 派发子代理前，必须执行以下5项检查。**缺一不可**。

```
□ ① 体用定义正确性检查
   体=日主+印+比劫+食伤（食神+伤官全算体）
   用=财+官（正财偏财正官七杀·藏干财官一律算用）
   如context中有体用定义的叙述 → 核对是否与上述一致

□ ② 排除规则检查
   在context中全文搜索「不算」「排除」「不纳入」「不计入」「不应计」
   找到任何一个 → ❌ 必须删除
   铁律：藏干财官全部算用，没有任何排除规则

□ ③ 算术自检
   context中提供的用分数、体分数，逐项加和是否等于总分
   常见错误：12+7.2+12+24=40.8 ❌ 正确=55.2
   口诀：自写算式逐项加，确保总分不出差

□ ④ 口诀条件校验
   context中引用的泉师兄口诀/断语 → 条件是否满足本八字？
   例：「四柱皆带财」→ 逐柱检查是否有财星
   例：「身弱不担财」→ 检查身强弱分数
   条件不满足 → 不准引用，须注明差异

□ ⑤ 技能版本确认
   context要求子代理加载的技能 → 以skill_view验证是否有§3A.9铁律
   确认体用定义是最新版（体含食伤、用只财官、藏干算用）
```

## 常见陷阱速查表

| 陷阱 | 错误写法（我犯过的） | 正确写法 |
|:-----|:-------------------|:---------|
| 藏干排除 | 「巳中庚不在用中」 | 「巳中庚七杀12分算用（铁律：藏干财官一律算用）」 |
| 用计算错误 | 「用=40.8分(12+7.2+12+24)」 | 「用=55.2分(12+7.2+12+24)」 |
| 口诀条件不符 | 「四柱皆有财」 → 年柱无财 | 「三柱有财（月日时），年柱无财（庚申劫财）」 |
| 大运真假 | 「戊子 戊❌无戊→假」 | 「戊子 戊✅藏申→真」（申藏戊土余气30%） |
| 同音字混用 | 「官→财→食伤/库→路」 | 「官→财→食伤/库→禄」 |
| 遗漏藏干财官 | 用只算天干32分 | 用=天干32+藏干22.8=54.8分 |

## 口诀

```
派发之前五步查，体用算术别漏下
排除规则不能有，条件校验不能假
藏干财官都算用，铁律牢记不能忘
context错了子跟错，源头把关是第一
```

## 验证脚本（手动执行）

派发前运行以下命令检查context是否干净：
```bash
# 检查排除规则
echo "检查排除规则:"
grep -n "不算\|排除\|不纳入\|不单独计入" <(echo "$CONTEXT_TEXT") 2>/dev/null || echo " ✅ 干净"

# 检查体用定义
echo "检查体用定义:"
echo "$CONTEXT_TEXT" | grep -q "体.*=.*食伤" && echo " ✅ 体含食伤" || echo " ❌ 体缺食伤"
echo "$CONTEXT_TEXT" | grep -q "用.*=.*财.*官" && echo " ✅ 用只财官" || echo " ❌ 用定义不符"
```
