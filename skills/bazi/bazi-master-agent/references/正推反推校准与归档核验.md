# 正推→反推校准方法论 & 文件归档三重核验

## 来源
2026-06-15 主母成八字校准会话。老板亲授。

## 正推→反推校准铁律

每次分析八字时必须**先做纯理论正推**（完全不动用用户的实证），得出完整的纯理论结论后，再用用户提供的实证反推校准。绝不能先入为主用实证凑理论。

### 正推检查清单（每项必须过）

```
□ 排盘验证（引擎+五鼠遁+五虎遁）
□ 身强弱（得令+月令印+有根+透干+合局）
□ 喜用神/忌神
□ 格局判定（正八格+特殊格+空亡+墓库）
□ 原局合局检查 ⚠️易漏（六合/三合/三会，特别是月日相邻）
□ 空亡检查 ⚠️易漏（大运地支是否空亡）
□ 神煞检查 ⚠️易漏（天罗地网/元辰/灾煞/天乙贵人/华盖等）
□ 十二长生（大运所在周期）
□ 化气格检查（天干五合是否紧邻+有根+月令生）
□ 辰戌丑未变性检查
□ 大运逐运综合评估（表面十神+合局+空亡+神煞+十二长生五维合一）
□ 逐流年关键年排查
```

### 反推校准流程

1. 纯理论结论先完整输出
2. 再逐条对比用户提供的实证
3. 有差异 → 找根因（是漏了某个检查点？还是理论用错了？）
4. 根因找到了 → 沉淀到对应skill中，防止再犯

### 口诀

先纯理论正推，再实证反推校准
理论推导不动用用户给的人生事实
对比差异找根因，漏了什么补什么
这才是真正提升能力

## 文件归档三重核验

保存任何报告到知识库前，必须做以下三重核验：

### 第一重：人物对应核验
- 报告的命主是谁？目标目录对应谁？两者是否一致？
- 例：主母成的报告 → 02-主母-成/ ❌ 不是 11-家主母亲/

### 第二重：八字年龄核验
- 报告中的八字出生年份 → 与目标目录的人物年龄是否匹配？
- 例：丁卯丁未庚午壬午(1987年生) → 年龄约39岁
  → 放入02-主母-成/ ✅ 放入11-家主母亲/ ❌（1952年生不符）

### 第三重：目录ls核验
- 放文件前先 `ls` 目标目录，确认目录存在且内容合理
- 不要 mkdir -p 创建新目录，不要靠记忆假设目录名

### 口诀

报告归档三重验，人物对应第一关
八字年龄要匹配，ls看目录再动手

## 实战案例

主母成（丁卯丁未庚午壬午·1987-07-20午时）：
- 之前6个遗漏：午未六合/空亡/天罗/元辰/天乙/最佳大运判错
- 根因：正推时未逐项过检查清单，凭经验跳步
- 修正后：正推检查清单已建立，先过清单再出结论
