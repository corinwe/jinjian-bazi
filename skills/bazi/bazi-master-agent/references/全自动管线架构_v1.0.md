# 金鉴真人·全自动分析管线设计文档 v1.0

## 架构总览

```
用户发八字请求
    ↓
┌─────────────────────┐
│  bazi-master-agent  │ ← 总调度层（路由+管线编排）
│  自动检测请求类型   │
└─────────┬───────────┘
          ↓
┌─────────────────────┐
│  scripts/bazi-     │ ← 管线脚本（可手动/自动调用）
│  pipeline.sh       │
└─────────┬───────────┘
          ↓
┌─────────────────────┐
│  4步管线执行        │
│  Step1: 排盘验证     │
│  Step2: 基础包(§23)  │
│  Step3: 全量10子skill│
│  Step4: 模板输出12板块│
└─────────────────────┘
```

## 管线触发方式

### 方式A：AI引擎自动触发（默认）
任何涉及「八字」「排盘」「算命」「分析」的用户请求
→ master-agent自动检测
→ 全量加载10个子skill
→ 按v2.3模板输出

### 方式B：脚本手动触发
```bash
bash scripts/bazi-pipeline.sh 2011 5 31 9 0 5 男 子源 成都
```

## 三大铁律

1. **任何八字=全量分析**：不再默认只加载foundation+fortune
2. **排盘验证门禁**：引擎验证不通过=不分析
3. **模板强制**：12大板块缺一不可，身材外貌强制300字+

- 知识库路径：07-国学哲学/八字命格/02-人物档案/{对应目录}/
