# 金鉴真人·音频知识处理管线 v1.0

> 2026-07-04 实战提炼（10份音频~19h处理经验）

## 一、总流程

```
收到音频
  │
  ├─ 磁盘预检 → 空间不足先清理(apt/pip/tmp/journal)
  │
  ├─ 批处理转写(background+notify_on_complete)
  │   按文件大小从小到大
  │   每份约1倍实时(2核CPU: ~19h/10份)
  │
  ├─ 主题扫描(head-20行识别)
  │
  ├─ 并行delegate_task精读(每批≤4份)
  │   每份：读全稿→提取知识点→对比已有skill
  │
  ├─ 冲突审计(逐条标记覆盖状态)
  │
  └─ 三系统沉淀：
       ① 知识库文档(01-理论体系/)
       ② 技能(SKILL.md + references/ + scripts/)
       ③ 自动化(检查脚本/面诊工具)
       ④ git推库
```

## 二、关键命令

```bash
# 磁盘检查
df -h /
apt-get clean -y
pip cache purge
journalctl --vacuum-time=3d

# 单份转写
python3 audio_pipeline.py <音频文件> --name "课程名"

# 批量转写(按大小排序)
ls -S *.m4a | while read f; do python3 audio_pipeline.py "$f" --name "课_$f"; done

# 主题快速识别
head -20 逐字转写.txt | grep -v "^#" | grep -v "^\-\-\-" | head -5
```

## 四、常见问题

| 问题 | 处理方法 |
|:----|:---------|
| 磁盘100% | 清理apt/pip缓存+journal日志，空间<1G时进程被kill |
| Whisper进程中断 | 检查目录是否已有原始音频存档→重启转写即可 |
| 第三方讲师内容 | 非九龙道长课程→标记备用，不纳入主体系 |
| 转写质量差(粤语/口音) | 备注说明，知识提取以可识别内容为准 |
