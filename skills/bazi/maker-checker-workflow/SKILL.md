# Maker/Checker 工作流

> **来源**: SOUL.md Sub-Agent分工体系 + HERMES.md Maker/Checker铁律
> 用于SOP Phase 4.5 — 确保分析报告内容质量

## §0. 分析深度铁律（2026-07-18校准）

**每个§必须有真实命理分析，禁止只输出数据不输出判断。**

| 命理模块 | ❌ 浅层（数据复读） | ✅ 深层（真实分析） |
|:---------|:------------------|:------------------|
| 婚姻 | "配偶宫为伤官" | "伤官在配偶宫：才华过人但也挑剔苛刻。对配偶要求高，容易因小事不满。" |
| 子女 | "时柱卯辛" | "时支偏财→子女经济头脑好。时干比肩→与子女如朋友。" |
| 六亲 | "年柱劫财→祖上" | "年柱庚申五行与日主同→性格相近。月柱克耗日主→关系有张力。" |
| 三决断 | "一决身：身强" | "一决身：身强→主动出击型。敢于挑战担责。注意固执。" |
| 置业 | "宜选水木火相关数字" | "宜选：3、8、2、7、1、6。忌选：4、9、5、0。" |

**检验标准**：每段分析应包含"数据→解释→判断→建议"四个层级，缺一不可。

## §1. 核心原则

```
Maker = 内容撰写人（写分析的人）
Checker = 对抗性审查人（找漏洞的人）
两人必须不同 — 同一个人不能既写又审
```

## §2. 检查维度（五轴）

| 维度 | 检查项 | 方法 |
|:----|:-------|:-----|
| **正确性** | 格局判定、身强弱、喜用神 | 对比引擎JSON + 规则行号 |
| **一致性** | 数据与引擎一致，无自创 | grep提取数字对比JSON |
| **完整性** | 5项必含内容全部存在 | 逐项过checklist |
| **可读性** | 语言通顺、无模板话术 | 眼扫一遍 |
| **格式** | 排盘格式正确、段落间距一致 | 移动端预览 |

## §3. Maker/Checker 执行流程

```
Maker:
  ① 加载所有技能（Phase 4.1）
  ② 逐§写分析（按Phase 4.3A映射）
  ③ 每写完一§打勾✅

Checker（Maker之后）:
  ① 从报告中提取所有数字/十神/五行数据
  ② 对比引擎JSON中的对应字段
  ③ 逐项确认一致
  ④ 不一致 → 标记并退回Maker改
  ⑤ 全部一致 → 标记✅

完成条件 → Phase 5发布前校验
```

## §4. 常见Checker发现的问题

| 问题 | 严重度 | 处理方法 |
|:-----|:------|:---------|
| 分数写错（记忆偏差） | 🔴 | 以JSON为准重写 |
| 格局命名错误 | 🔴 | 查fortune-analysis §6 7步法重判 |
| 缺少强制内容 | 🔴 | 补充后重审 |
| 模板话术 | 🟡 | 替换为具体数据描述 |
| 排版不一致 | 🟢 | 调整格式 |
