# 八字学习资料·提取与知识库归档工作流

> 2026-06-01 更新：新增 pdftotext/pandoc 方案 + 穷举式提取质量验证

用于提取八字命理PDF/文档内容并存入知识库。
核心原则：**要么全提取，要么就别提取——遗漏知识点会被用户退回重修。**

## 一、环境准备

```bash
# 方案A：pymupdf（最快 — 适合文本型PDF）
pip install pymupdf

# 方案B：pdftotext（poppler — 适合图片型/扫描型PDF，图文兼顾）
apt-get install -y poppler-utils

# 方案C：pandoc（万能转换 — 适合docx转markdown）
apt-get install -y pandoc

# 方案D：OCR（仅当以上全部失败时才用）
apt-get install -y tesseract-ocr tesseract-ocr-chi-sim
pip install pytesseract
```

## 二、提取技术选择决策树

```yaml
收到文档后，按优先级尝试：

收到 .txt → 直接 read_file
收到 .docx → ✅ 优先使用 pandoc（最清晰，保留结构）
收到 .pdf → 先尝试 read_file（判断是否为文本型PDF）
  ├─ 能读出中文 → 文本型PDF ✅ 直接提取
  └─ 全是乱码/二进制（如 %PDF-1.7...）→ 图片型PDF
      ├─ 优先找同内容的 docx 版本（用户可能有！）
      │   └─ 有 docx → pandoc 转 markdown（最清晰）
      └─ 无 docx → pdftotext 提取
          └─ pdftotext -layout 保留排版
```

### 实战命令

```bash
# PDF→文本（pdftotext, 推荐-layout保留表格格式）
pdftotext -layout input.pdf output.txt

# DOCX→Markdown（pandoc）
pandoc input.docx -t markdown --wrap=none -o output.txt
```

## 三、⚠️ 穷举式提取质量验证（金鉴铁律）

> **这是本工作流最重要的环节。用户会逐句检查你的提取是否遗漏。**
> 提取完成后，必须执行以下验证步骤，不可跳过：

### 3.1 第一步：首轮提取

```
逐行阅读全文 → 识别所有知识点
按类别归类：疾病/财运/格局/十神/流年/神煞等
对照现有技能体系 → 区分「已有」和「新增」
```

### 3.2 第二步：反向扫描（防遗漏）

```
提取完成后，重新从第一行开始逐行扫描：
  对每个段落/句子问自己：「这个知识点提取了吗？」
  如果是 → 具体在哪个文件哪个章节？
  如果否 → 为什么没提取？是不重要还是漏了？
  
⚠️ 常见的遗漏类型：
  ├─ 散落在闲聊/问答中的知识点（最容易漏）
  ├─ 一句话带过的口诀（看似不重要，实际是精华）
  ├─ 案例中的断病/断事逻辑（比理论更重要）
  └─ 用户互动中的反馈/印证（实战校准数据）
```

### 3.3 第三步：对比自查清单

```
□ 所有案例的断语逻辑都提取了吗？
□ 所有口诀/金句都提取了吗（哪怕只有一句话）？
□ 所有五行对应关系（木=毛发/身高等）都提取了吗？
□ 所有调理方法（颜色/石头/位置）都提取了吗？
□ 所有反常识的知识点（如金多火灭/藏干不流通）都提取了吗？

如果以上有任何一项「否」→ 回到第二步重新扫描
```

### 3.4 第四步：还原验证

```yaml
还原验证：只看提取后的知识点，能否还原原文档的完整框架？
  ① 把提取的知识点按原文档结构排列
  ② 看是否有「断层」— 原文档中间某段内容在提取后消失了
  ③ 如果有 → 缺的段落大概率有遗漏知识点
  
实例（本课程教训）：
  用户问「文中还有未提取的知识点吧」
  → 逐行重新扫描发现遗漏了14项
  → 包括口诀、性格区分、辨时辰法等
```

## 四、提取后处理

1. 阅读提取内容，理解整体结构
2. 如果有多份文档（如网络班+实地班），合并为一份完整文件
3. 在文件头部添加元数据（来源、页数、提取日期、内容覆盖范围）
4. **按四步走流程执行**（升级skill → 更新理论体系 → 建自动机制 → 推知识库）
5. 存入知识库的理论目录：`07-国学哲学/八字命格/`
