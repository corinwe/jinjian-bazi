# daily-context-manager — 跨会话上下文恢复体系

## 核心理念

不建平行系统，用好已有的四层架构：

```
layer 1: memory()           → 稳定事实跨会话永久保留（15% usage）
layer 2: session_search()   → 我主动搜过去对话，不让你重复
layer 3: skills              → 踩过的坑沉淀为可复用流程
layer 4: 当前上下文          → 会话内跟踪，结束即清
```

## 行为纪律（三铁律）

### 铁律一：有怀疑 → 先搜不问

当以下情况出现时，不等老板开口，主动 session_search：

- 老板提到一个之前讨论过的话题，印象模糊
- 某项数据/结论感觉之前出现过
- 涉及跨多个会话的连续性任务
- 回答中带有不确定性词（"好像"、"我记得"、"应该是"）

**执行：**
```
# 不要问老板"之前说过吗？"或"在哪说的？"
# 直接搜
session_search(query="关键词", limit=3)
```

### 铁律二：搜不到 → 老板说"搜一下" → 我翻历史

如果主动搜了但没找到，老板会说"搜一下"。此时：

- 换关键词角度再搜一次
- 缩小时间范围
- 搜到后告诉老板"找到了"
- 没搜到则如实说"没找到，您方便再发一下？"

### 铁律三：零重复错误

每次执行重要操作前（八字排盘、写报告、改系统配置等）：

- 回想该类型任务曾犯过的错误
- 有则读对应的 skill 中的 pitfalls 部分
- 如果连续两次犯同类错误 → 写进 skill 固化

四层架构不动，照常运行。
三铁律严格执行。
