# Sunday Preview Execution — 2026-05-24

## Context
- Saturday (2026-05-23) cron already produced a full weekly recap covering Mon-Fri May 18-22
- Sunday (2026-05-24) cron fired at 8:00 AM Beijing — same Friday close data, no new market events
- Challenge: produce a distinct report that adds value without rehashing Saturday's content

## Solution: "Sunday Preview" format

### Key differences from Saturday weekly recap

| Dimension | Saturday Recap | Sunday Preview |
|-----------|---------------|----------------|
| Title framing | "周末特辑 · 本周全市场深度复盘" | "明日开市 · 全周前瞻 — [key themes]" |
| Deep-dive format | Week-in-review (Mon-Fri events) | Forward-looking themes (4-6 numbered, 2-4 sentences each) |
| Holdings table | "周度总览" with weekly change column | "前瞻速览" with extra emphasis on next-week catalysts |
| Character count | 1600-2000 | 2000-2500 (more forward analysis) |
| 7-role picks | Any stocks with next-week catalysts | Same criteria, but add "X date catalyst" emphasis in header |
| Catalyst table | Same format | Same format, update event descriptions for Mon-Fri |

### Deep-dive structure for Sunday
Instead of a single narrative paragraph, use numbered themes:

```
## 🧠 行业深度洞察：周度前瞻 — 未来一周的五条主线

1️⃣ **[Theme 1]** — [Catalyst event → impacted positions → scenario analysis]
2️⃣ **[Theme 2]** — [...]
...
5️⃣ **[Theme 5]** — [...]
```

Each theme should:
- Name a specific catalyst or trend for the COMING week (not the past)
- Name the affected portfolio positions
- Describe 2 scenarios (bull/bear or what-if)
- Keep each to 2-4 sentences

### Pitfalls
- Do NOT use "周末特辑" or "周度复盘" labels if Saturday already used them
- Do NOT re-run Task D (industry deep-dive) via delegate_task — compose the forward-looking themes yourself from collected data
- The holdings data is the same as Saturday (Friday close) — the analytical framing is what differentiates the report
- Character count ~2,700 is acceptable for Sunday (more forward-looking themes to cover)
