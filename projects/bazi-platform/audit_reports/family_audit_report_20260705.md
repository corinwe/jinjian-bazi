### ❌ 检查点①：六亲宫位对应

**Skill规则：**
```yaml
年干 = 父系家族
年支 = 母系家族
月干 = 父亲本人/兄姐
月令 = 母亲本人/弟妹
日支 = 配偶及家族（夫妻宫）[婚姻模块处理]
时干 = 长子/长女 [子女模块处理]
时支 = 次子及后续子女 [子女模块处理]
```

**代码现状（family.py:31-34, 59-82）：**
```python
year_shi_shen = get_shi_shen_for_gan(year_gan, ri_zhu)   # 只查年干十神
month_shi_shen = get_shi_shen_for_gan(month_gan, ri_zhu)  # 只查月干十神
```

**差距：**
- ✅ 正确：月干=父亲本人（通过 month_gan 的十神体现）
- ❌ **缺失**：年支=母系家族 — 代码只分析年干(year_gan)的十神，未分析年支藏干
- ❌ **缺失**：年干=父系家族 — 代码取了 year_gan 十神但 desc_map 描述不精确（见检查点③）
- ❌ **缺失**：月令=母亲本人/弟妹 — 代码只分析月干(month_gan)，未分析月支(month_zhi)藏干
- ❌ **缺失**：无"年干父系/年支母系"分区段输出

**修复建议：** 对年支/月支的藏干也计算十神，输出时区分「年干父系」「年支母系」「月干父亲」「月支母亲」四个子段。

### ❌ 检查点②：父母离异/家庭变故信号

**Skill规则：**
```yaml
⚠️ 月柱（月干+月支）代表父母宫。
月柱被合/冲/刑/害 → 父母关系出问题的最直接信号。

【月干被合 → 父亲角色变化】
  月干=父亲本人被合走 → 父亲角色缺失

【月支被冲 → 母亲受冲击】
  月支为母位，被冲则母亲有变故

【区分：合年干 vs 合月干】
  合年干（祖上宫）→ 家族层面的变动，非父母直接
  合月干（父母宫）→ 父母关系的直接冲击 ⭐ 更强信号
```

**代码现状（family.py:41-56）：**
```python
# 年支被冲 → 早年离家/祖业不守
if rel["冲"]:  # 检查年支与月支相冲
    family_notes.append("年支与月支相冲→早年离家或父母关系紧张")

# 年支月支相害 → 父母关系不睦
if rel["害"]:
    family_notes.append("年支与月支相害→祖上/父母关系暗藏矛盾")

# 年支月支相刑 → 家庭内部矛盾多
if rel["刑"]:
    for x in rel["刑"]:
        family_notes.append(f"年支与月支{x['type']}→家庭内部矛盾多")

# 年支月支六合 → 家庭和睦
if rel["六合"]:
    family_notes.append("年支月支六合→家庭关系和睦")
```

**差距：**
- ❌ **缺失**：**月柱被合**（最关键信号！）— 只检查了年支月支之间的地支关系，但skill明确指出"月柱被合=父母关系出问题的最直接信号"。这里的"合"应包括天干五合对月干的作用
- ❌ **缺失**：**月干被合→父亲角色变化** — 没有天干五合分析（需要使用v2的`天干五合`）
- ❌ **缺失**：**月支被冲→母亲受冲击** — 当前只检查"年支与月支相冲"，但skill说的是"月支被冲"即月支被任何因素冲（非仅年支冲月支）
- ❌ **缺失**：**合年干 vs 合月干区分** — 无祖上宫 vs 父母宫的差异化分析
- ❌ **缺失**：代码只用 year_zhi 和 month_zhi 两个地支检查关系，但六亲分析应扩展为四柱全量地支关系检查（包含日支/时支对年月的影响）
- ⚠️ rel["冲"] 只检查 year_zhi vs month_zhi，如果这两个地支无冲但月支被其他柱冲（如日支冲月支），则完全遗漏

**修复建议：**
1. 月柱分析应同时做天干五合（用check_all_relations_v2的`天干五合`结果）
2. 新增月干被合→父亲角色变化
3. 新增月支单独受冲→母亲受冲击（而非仅year_zhi vs month_zhi互冲）
4. 新增合年干 vs 合月干的差异化断语

**差距：**
- ❌ **致命缺失**：代码**完全不区分男命/女命**。`analyze_nian_yue` 没有 gender 参数，所有十神描述都是性别中立的

2. 重构 desc_map 为 gender-aware 双表（男版/女版）
3. 每个十神输出对应性别下的实际六亲角色标注
