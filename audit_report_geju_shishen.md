## 一、格局（§4/§12）审计

### A. 致命错误：报告生成器瞎编格局排名

**理论依据**：`九龙道长_八字最好的格局排名_20260627.md` 明确规定了顶级大富大贵格局5种（食神制杀、伤官配印、财官双美、从官杀、从财）和上等富贵格局5种（官印相生、食伤生财、五行流通、专旺格、木火通明），共10大格局。

**报告实现（report_generator_simple.py 第1151行）**：
```
'参考金鉴真人十大格局排名：正官第一，七杀第二，正印第三，偏印第四，正财第五，偏财第六，食神第七，伤官第八，比肩第九，劫财第十。'
```

**错误性质**：❌ **完全瞎编**。理论文件说的是10种**特定格局组合**（食神制杀格、伤官配印格等），报告将其篡改为正八格+比劫的简单排序，与理论毫无关系。这是编造理论。

---

### B. 引擎 get_ge_ju() 缺少格局分级/评分

**理论要求**：十大格局需要分级，顶级格局有特殊断语（如"食神制杀→聪明绝顶、胆大心细、易掌大权"）。

**引擎实现（bazi_engine.py 第537-591行）**：
- 只返回正八格名称（"食神格""伤官格""正财格"等）
- 没有格局分级（顶级/上等）
- 没有格局排名
- 没有格局组合判定（如"食神制杀"需要食神+七杀同时出现）

**错误性质**：❌ **严重缺失**。引擎的格局判定完全没有反映理论文件的十大格局排名体系。返回的"食神格"和理论中的"食神制杀格（顶级格局）"是完全不同的概念。

---

### C. 专旺格判定过于宽松

**理论要求（SKILL.md 第1029-1038行）**：专旺格要求"八字满盘印比 + 无财官食伤破格"，即全局几乎都是同一五行。

**引擎实现（bazi_engine.py 第1363-1378行）**：
```python
if ri_pct >= 70 and sqr_score >= 80:
    name = ZHUAN_WANG_NAMES.get(ri_wx, "")
```

**错误性质**：⚠️ 用"某五行占比>70%"代替"满盘印比、无财官食伤破格"，条件过于宽松。一个八字某五行占比>70%但不满足"无破格"条件时，不应判定为专旺格。

---

### D. 化气格缺"有根"条件

**理论要求（SKILL.md 第1057-1060行）**：
```
化气格 = 天干五合 + 合化后五行在月令得令有根
```

**引擎实现（bazi_engine.py 第1390-1409行）**：
```python
if WX_SHENG[yue_wx] == hua_wx or yue_wx == hua_wx:
    return {"is_hua_qi": True, ...}
```

**错误性质**：⚠️ 引擎只检查了"月令生助化气五行"，缺少"有根"条件。月令生助但无根时不应算化气成功。

---

### E. 五行流通格和木火通明格完全未实现

**理论要求**：九龙道长格局排名中第8名为"五行流通格"，第10名为"木火通明格"。SKILL.md也标注为"需要新增"。

**引擎/报告实现**：完全不支持这两种格局。

**错误性质**：❌ **缺失**。两个理论明确的格局完全未实现。

---

### F. 从弱格在格局判定中被忽略

**理论要求**：从官杀格（第4名）、从财格（第5名）是顶级格局。

**引擎实现**：`get_ge_ju()`只返回正八格名称，不从格判定。从格信息隐含在身强弱判定（"从弱"级别）中，但没有作为格局输出。

**错误性质**：⚠️ 从格是知识库中明确的格局类型，引擎的正八格判定无法反映从格。

## 二、十神（§7/十神分析）审计

### A. 致命错误：十神吉凶分类完全混乱（报告生成器§12）

**理论依据（十神信息课程核心知识点总结 第6行）**：
> 4吉神（正印、食神、正官、正财）、4恶神（七杀、偏印/枭神、伤官、劫财）、2中性（比肩、偏财）

**报告生成器实现（report_generator_simple.py 第715-717行）**：
```python
ji_shen_list = ['正官', '正印', '偏印', '正财', '食神']  # 5个"吉神"
e_shen_list = ['七杀', '劫财', '伤官', '偏财']            # 偏财被列为恶神!
zhong_xing_list = ['比肩']
```

**错误性质**：❌ **理论严重错误**：
1. **偏印被误列为吉神**：理论明确说偏印/枭神是四大恶神之一，报告将其归为吉神
2. **偏财被误列为恶神**：理论明确说偏财是中性（平神），报告将其归为恶神
3. **正官被列为吉神但多了一个"偏印"**，导致吉神5个（理应是4个）

---

### B. 引擎完全忽略"偏印≠枭神"的区分

**理论要求（SKILL.md 第1445-1446行）**：
> 偏印 ≠ 枭神：命局中无食神时叫偏印（中性偏凶），有食神时叫枭神（凶神，枭神夺食）

**引擎实现（bazi_engine.py 第109-125行）**：`shi_shen()`函数固定返回"偏印"，没有"枭神"状态。

**错误性质**：❌ **缺失关键区分**。当命局中有食神时，偏印应变为"枭神"，这是枭神夺食断事的基础。

---

### C. 缺乏十神缺失的完整分析

**理论要求（文件2 第162-168行）**：详细列出了缺每种十神的性格缺陷：
- 无正印：不爱学习、责任心差
- 无偏印：粗心大意
- 无食神：性格闷、不会享受
- 无七杀：容易被欺负
- 无正官：难当领导
- 无正财：财来财去

**报告实现**：性格分析（§4）有220行，但缺乏对十神缺失的系统分析。

**错误性质**：⚠️ **部分缺失**。理论有完整"十神缺失论"，报告未体现。

---

### D. 七杀/正官在四柱的详细断语未实现

**理论依据（九龙道长_七杀在哪个柱上最好.md）**：详细描述了正官和七杀在年柱、月柱、日柱、时柱各位置的断事规则（共约80条）。

**报告实现**：完全没有使用这些详细断语。

**错误性质**：⚠️ **缺失**。理论中关于十神位置断事的丰富内容未在报告中体现。

## 五、跨领域问题汇总

### A. 藏干权重体系不一致

| 来源 | 规则 | 
|------|------|
| **引擎**（line 26） | 本气100%, 中气60%, 余气30% |
| **九龙道长Part1**（知识点3） | 得令=40分, 透干=12分, 通根叠加 |
| **SKILL.md**（line 765-772） | 本气根+30分, 中气/余气根+15分 |

三种规则完全不同。引擎的权重用于藏干计分（如财星评分），与九龙道长的40分/12分体系不兼容。

---

### B. 身强弱评分与九龙道长40分法则不兼容

- 引擎使用0-100分的通用评分体系
- 九龙道长使用"40分墓库法则"（≥40分=库，<40分=墓）
- 两种体系的分值含义不同，引擎没有将身强弱评分映射到墓库法则

---

### C. §12格局分析中"辅格"概念纯属虚构

**报告生成器在第12.2节引入了"辅格"概念**，判断逻辑是看天干透出的非月令本气十神。但是**所有理论文件中都没有"辅格"这个概念**。传统命理中有"兼格"概念，但报告中的"辅格"评分系统（吉神格+吉神格=顺遂，占85分等）完全是编造的。

**错误性质**：❌ **理论虚构**。
